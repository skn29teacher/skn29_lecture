# Windows용 Nginx 리버스 프록시 연동

본 단계에서는 Windows 개발 환경에서 사용할 수 있는 Windows용 Nginx 웹 서버를 다운로드하여 구성하고, WAS(Waitress)와 TCP 소켓 통신을 매개로 삼아 리버스 프록시(Reverse Proxy)로 연동하는 방법을 학습합니다.

---

## 1. Windows Nginx의 특징과 통신 제약

Nginx는 본래 Linux/UNIX 환경에 적합하도록 구현되어 대규모 트래픽 처리에 압도적인 성능을 보입니다. Windows 버전의 Nginx 역시 배포되어 개발 및 간단한 테스트에 널리 사용되지만, 몇 가지 플랫폼 고유의 한계와 제약이 있습니다.

- **UNIX 도메인 소켓 지원 불가**: Windows 운영체제 커널의 구조적 특성상, 파일 상의 소켓인 UNIX 소켓 통신을 Nginx Windows 포팅 버전에서 사용할 수 없습니다.
- **TCP 루프백 통신 대체**: 따라서 Windows 환경에서 Nginx와 WAS(Waitress)를 중계하려면 반드시 로컬 호스트 IP인 `127.0.0.1`의 특정 TCP 포트(예: 8000)를 사용해 상호 통신해야 합니다.

---

## 2. Nginx와 WAS(Waitress) 연동 흐름

```text
[클라이언트 브라우저]
       │  (HTTP: localhost)
       ▼
 [Nginx (Windows)]  <─── 포트 80에서 대기
       │
       │  (TCP Proxy: 127.0.0.1:8000)
       ▼
[Waitress (Windows)] <─── 포트 8000에서 대기 (Django 실행)
```

---

## 3. 단계별 구성 가이드

### 1단계: Nginx 구성 디렉터리 구성
프로젝트 루트 폴더 옆에 `nginx/` 폴더를 생성하고 설정 파일 `nginx.conf`를 작성합니다.

```text
django_project/       # Django 웹 소스 코드 및 Waitress 실행 코드
nginx/
└── nginx.conf        # Windows용 Nginx 전체 설정 파일
```

### 2단계: Nginx 설정 파일 구현
Nginx 기동을 위해 기존 기본 설정을 완전히 대체할 수 있는 전체 설정 규격을 정의합니다. 80번 포트로 들어오는 HTTP 동적 요청을 동일 시스템 내 8000번 포트로 떠 있는 Waitress로 패스하는 TCP 프록시 옵션을 정의합니다.

- 파일명: `nginx/nginx.conf`
```nginx
worker_processes  1;

events {
    worker_connections  1024;
}

http {
    include       mime.types;
    default_type  application/octet-stream;

    sendfile        on;
    keepalive_timeout  65;

    # upstream 정의: Windows 환경에서 기동 중인 Waitress TCP 주소(127.0.0.1:8000)를 연결합니다.
    upstream django_app {
        server 127.0.0.1:8000;
    }

    server {
        listen 80;
        server_name localhost;

        # 파일 업로드 크기 제한 해제 (50MB)
        client_max_body_size 50M;

        # 로깅 파일 설정 (Windows Nginx의 기본 logs 폴더 내에 저장)
        access_log logs/django_access.log;
        error_log logs/django_error.log;

        # 모든 일반 동적 요청을 Waitress WAS로 프록싱
        location / {
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            proxy_set_header Host $http_host;
            
            # 리다이렉트 충돌 방지
            proxy_redirect off;
            
            # 정의된 TCP upstream으로 패스
            proxy_pass http://django_app;
        }
    }
}
```

### 3단계: Waitress 실행 스크립트 확인
4단계에서 고도화한 `run_waitress.py` 코드가 그대로 유지되는지 점검합니다. Nginx가 아직 정적 파일을 직접 처리하지 않으므로, Waitress가 정적/미디어 서빙을 겸할 수 있도록 래핑된 핸들러를 사용합니다.

- 파일명: `django_project/run_waitress.py`
```python
import os
from urllib.parse import urlparse
from waitress import serve
from django.core.wsgi import get_wsgi_application
from django.contrib.staticfiles.handlers import StaticFilesHandler
from django.views.static import serve as django_static_serve
from django.conf import settings

# 중요: Django 설정 모듈(Settings)을 가리키는 환경 변수를 최상단에서 기본값으로 지정합니다.
os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'todoboard.settings.production')

class StaticAndMediaFilesHandler(StaticFilesHandler):
    def __init__(self, application):
        super().__init__(application)
        self.media_url = urlparse(settings.MEDIA_URL)

    def _should_handle(self, path):
        return (path.startswith(self.base_url.path) and not self.base_url.netloc) or \
               (path.startswith(self.media_url.path) and not self.media_url.netloc)

    def serve(self, request):
        if request.path.startswith(self.media_url.path):
            path = request.path[len(self.media_url.path):]
            return django_static_serve(request, path, document_root=settings.MEDIA_ROOT)
        return super().serve(request)

application = StaticAndMediaFilesHandler(get_wsgi_application())

if __name__ == '__main__':
    print("Starting Waitress WSGI server on http://127.0.0.1:8000")
    serve(application, host='127.0.0.1', port=8000, threads=4)
```

## 4. Windows에서 Nginx 기동 및 연동 테스트

Windows 환경에서 Nginx를 다운로드하여 설정 파일을 적용하고 구동하는 절차는 다음과 같습니다.

1. **Nginx Windows 버전 다운로드**: [Nginx 공식 홈페이지](https://nginx.org/en/download.html)의 Stable version 메뉴에서 `nginx/Windows-x.xx.x.zip` 압축 파일을 다운로드하여 원하는 위치(예: `C:\nginx`)에 압축을 해제합니다.
2. **설정 파일 덮어쓰기**: 위에서 작성한 `nginx/nginx.conf` 파일 내용을 복사하여 다운로드한 Nginx의 `conf/nginx.conf` 파일 내용 전체와 대체(덮어쓰기)합니다. (기본 nginx.conf 전체 내용을 대체해야 구조적 에러 없이 리버스 프록시가 작동합니다.)
3. **Waitress 및 Nginx 구동**:
   ```cmd
    :: 1. 먼저 PostgreSQL 데이터베이스에 필요한 마이그레이션을 적용합니다.
    python manage.py migrate --settings=todoboard.settings.production

    :: 2. Django 가상환경 터미널에서 Waitress 기동 (기본적으로 production 설정 적용)
    python run_waitress.py

    :: 3. Nginx 압축 해제 폴더로 이동하여 Nginx 실행
    cd C:\nginx
    start nginx
   ```
4. **연동 종료 및 제어**:
   ```cmd
   :: Nginx 안전하게 종료
   nginx -s quit
   
   :: Nginx 강제 종료
   nginx -s stop
   ```

브라우저에서 `http://localhost` 주소로 포트 번호 없이 접속했을 때, Nginx가 요청을 수락하여 포트 8000번의 Waitress 서버로 동적 페이로드를 전달하여 Django 대시보드 화면이 에러 없이 노출된다면 성공입니다.
