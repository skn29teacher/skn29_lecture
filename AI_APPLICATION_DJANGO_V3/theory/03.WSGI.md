## 1. WSGI(Web Server Gateway Interface) 어플리케이션 서버
웹 서버(Nginx, Apach)와 python 어플리케이션(Django, Flask) 연결하는 표준 인터페이스
- 브라우져(HTTP요청) -> Nginx(웹서버) -[WSGI(Gunicorn)]  > Django ->PostgreSQL
- Django 의 runserver는 내장 서버(단일 쓰레드기반)이고 내부적으로 당연히 WSGI 역활을 하는 모듈이 있다.wsgi.py

## 2. 웹 서버(Nginx)
클라이언트와 직접 대면해서 정적파일(html, css, js, 이미지)를 직접 빠르게 접근, 동적연산을 위해서 WAS에 중계
## 3. WAS(Web Application Server)  Gunicorn
웹 서버로부터 넘겨받은 동적요청을 파이썬 코드로 실행해서 처리한뒤 그 결과를 다시 웹 서버로 반환

## 4. gunicorn을 requiremnts.txt에 추가
- pip install -r requirements.txt
설정파일을 참고해서 Django WSGI 어플리케이션 가동()
- gunicorn -c gunicorn.config.py todobard.wsgi:application
