# Windows Nginx SSL HTTPS 설정 및 Django 보안 강화

본 단계에서는 Windows 개발 환경에서 OpenSSL을 활용하여 개발용 자가 서명 인증서를 발급하고, Windows용 Nginx에 SSL/TLS 설정을 적용하여 HTTPS 보안 채널을 개시함과 동시에 Django 애플리케이션의 세션 및 쿠키 보안 강화를 적용하는 방법을 학습합니다.

---

## 1. SSL/TLS 인증서 개요 및 Windows 발급 방식

HTTPS 통신을 개시하기 위해서는 웹 서버(Nginx)에 공개키와 인증 기관의 서명이 담긴 인증서 파일(`*.crt`) 및 이에 대응하는 비밀키 파일(`*.key`)이 배치되어야 합니다.

운영 서버에서는 Let's Encrypt 등 공인 인증서 기관을 활용하지만, 로컬 Windows 개발 및 테스트 환경에서는 자가 서명(Self-signed) 인증서를 발급하여 연동합니다.

---

## 2. 윈도우 환경 OpenSSL을 이용한 인증서 생성

Windows 터미널(Git Bash, PowerShell 또는 CMD)에서 아래 명령어를 실행하여 1년 동안 유효한 테스트용 키 세트를 발급받을 수 있습니다.

```bash
# Nginx 구성 디렉터리 내에 certs 폴더를 생성하고 발급 진행
mkdir -p certs
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout certs/django.key \
  -out certs/django.crt \
  -subj "/C=KR/ST=Seoul/L=Seoul/O=Playdata/CN=localhost"
```

## 3. 단계별 구성 가이드

### 1단계: Nginx 구성 디렉터리 구성
프로젝트 루트 폴더 옆에 `nginx/` 폴더를 생성하고 설정 파일 `nginx.conf`를 작성합니다.

```text
django_project/       # Django 웹 소스 코드 및 Waitress 실행 코드
nginx/
└── nginx.conf        # Windows용 Nginx 전체 설정 파일
```

### 2단계: Nginx 설정 파일 구현
Nginx 기동을 위해 기존 기본 설정을 완전히 대체할 수 있는 전체 설정 규격을 정의합니다. 443 ssl 수신 서버 블록을 추가하고 발급받은 인증서 경로를 Windows 절대 경로로 기재합니다.

- 파일명: `nginx/nginx.conf`
```nginx
worker_processes  1;

events {
    worker_connections  1024;
}

http {
    include       mime.types;
    default_type  application/octet-stream;

    sendfile        on;
    keepalive_timeout  65;

    upstream django_app {
        server 127.0.0.1:8000;
    }

    # 1. HTTP(80) 요청을 HTTPS(443)로 강제 전환
    server {
        listen 80;
        server_name localhost;
        return 301 https://$host$request_uri;
    }

    # 2. HTTPS(443) 보안 서버 (Windows 경로 구성)
    server {
        listen 443 ssl;
        server_name localhost;

        # SSL 인증서 매핑 (Windows 경로 형식 적용)
        ssl_certificate C:/Users/Playdata/Downloads/skn29_4_lecture/skn29_4_lecture/day07/07_ssl_security_hardening_windows/nginx/certs/django.crt;
        ssl_certificate_key C:/Users/Playdata/Downloads/skn29_4_lecture/skn29_4_lecture/day07/07_ssl_security_hardening_windows/nginx/certs/django.key;

        # SSL 암호화 사양 최적화
        ssl_protocols TLSv1.2 TLSv1.3;
        ssl_ciphers HIGH:!aNULL:!MD5;
        ssl_prefer_server_ciphers on;

        client_max_body_size 50M;

        gzip on;
        gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;

        # 보안 헤더 추가
        add_header X-Frame-Options "SAMEORIGIN" always;
        add_header X-Content-Type-Options "nosniff" always;
        add_header X-XSS-Protection "1; mode=block" always;
        add_header Content-Security-Policy "default-src 'self' http: https: data: blob: 'unsafe-inline'" always;

        # 정적 리소스 직접 서빙 (Windows 경로 예시)
        location /static/ {
            alias C:/Users/Playdata/Downloads/skn29_4_lecture/skn29_4_lecture/day07/07_ssl_security_hardening_windows/django_project/staticfiles/;
            expires 30d;
            add_header Cache-Control "public, no-transform";
            access_log off;
        }

        # 미디어 리소스 직접 서빙 (Windows 경로 예시)
        location /media/ {
            alias C:/Users/Playdata/Downloads/skn29_4_lecture/skn29_4_lecture/day07/07_ssl_security_hardening_windows/django_project/media/;
            expires 7d;
            add_header Cache-Control "public, no-transform";
        }

        # WAS Waitress 중계 연동
        location / {
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            
            # 중요: Django 측에 HTTPS 연결 상태를 알리기 위해 포워딩 헤더 설정
            proxy_set_header X-Forwarded-Proto https;
            
            proxy_set_header Host $http_host;
            proxy_redirect off;
            proxy_pass http://django_app;
        }
    }
}
```

### 2단계: Django 운영 설정 내 보안 필터 추가
`production.py` 설정 파일 하단에 SSL 프록시 감지 헤더 정의 및 브라우저 쿠키 암호화 정책을 추가합니다.

- 파일명: `todoboard/settings/production.py`
```python
from .base import *

# 운영 환경용 설정
DEBUG = False

# 운영 환경의 허용 호스트 설정
ALLOWED_HOSTS = env.list('ALLOWED_HOSTS', default=['localhost', '127.0.0.1'])

# PostgreSQL 데이터베이스 설정
DATABASES = {
    'default': env.db('DATABASE_URL')
}

# 운영 환경용 정적 파일 수집 경로 설정
STATIC_ROOT = BASE_DIR / 'staticfiles'

# ==============================================================================
# 보안 강화 설정 (HTTPS 및 쿠키 보안)
# ==============================================================================

# Nginx 리버스 프록시로부터 전달받은 SSL 헤더 검증 설정
SECURE_PROXY_SSL_HEADER = ('HTTP_X_FORWARDED_PROTO', 'https')

# HTTPS 접속이 아닐 경우 자동으로 HTTPS로 리다이렉션 수행
SECURE_SSL_REDIRECT = True

# 세션 및 CSRF 쿠키의 Secure 플래그 활성화 (HTTPS를 통해서만 전송되도록 강제)
SESSION_COOKIE_SECURE = True
CSRF_COOKIE_SECURE = True

# HTTP Strict Transport Security (HSTS) 활성화 (브라우저가 1년간 HTTPS로만 접속하도록 지시)
SECURE_HSTS_SECONDS = 31536000
SECURE_HSTS_INCLUDE_SUBDOMAINS = True
SECURE_HSTS_PRELOAD = True

# 브라우저 MIME Sniffing 방어 필터 적용
SECURE_CONTENT_TYPE_NOSNIFF = True
```

### 3단계: Waitress 구동용 스크립트 적용
정적 및 미디어 파일이 HTTPS 프로토콜 내에서도 정상적으로 동작하도록 감싼 `run_waitress.py` 코드를 유지합니다.

- 파일명: `django_project/run_waitress.py`
```python
import os
from urllib.parse import urlparse
from waitress import serve
from django.core.wsgi import get_wsgi_application
from django.contrib.staticfiles.handlers import StaticFilesHandler
from django.views.static import serve as django_static_serve
from django.conf import settings

# 중요: Django 설정 모듈(Settings)을 가리키는 환경 변수를 최상단에서 기본값으로 지정합니다.
os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'todoboard.settings.production')

class StaticAndMediaFilesHandler(StaticFilesHandler):
    def __init__(self, application):
        super().__init__(application)
        self.media_url = urlparse(settings.MEDIA_URL)

    def _should_handle(self, path):
        return (path.startswith(self.base_url.path) and not self.base_url.netloc) or \
               (path.startswith(self.media_url.path) and not self.media_url.netloc)

    def serve(self, request):
        if request.path.startswith(self.media_url.path):
            path = request.path[len(self.media_url.path):]
            return django_static_serve(request, path, document_root=settings.MEDIA_ROOT)
        return super().serve(request)

application = StaticAndMediaFilesHandler(get_wsgi_application())

if __name__ == '__main__':
    print("Starting Waitress WSGI server on http://127.0.0.1:8000")
    serve(application, host='127.0.0.1', port=8000, threads=4)
```

## 4. 구동 테스트 방법

```cmd
:: 1. 운영용 데이터베이스 마이그레이션 적용
python manage.py migrate --settings=todoboard.settings.production

:: 2. 운영(production) 환경을 기준으로 정적 파일을 수집합니다.
python manage.py collectstatic --settings=todoboard.settings.production --noinput

:: 3. Django Waitress 서버 기동 (기본적으로 production 설정 적용)
python run_waitress.py

:: 4. Windows Nginx 기동 및 재로드
nginx -s reload
```

웹 브라우저를 켜고 `https://localhost` 주소로 접속을 테스트합니다. 자가 서명 인증서이므로 브라우저에 경고 창(보안 에러)이 뜰 수 있지만, 상세 정보 보기를 눌러 접속을 강행하면 HTTPS 프로토콜 상에서 안전하게 대시보드 데이터 송수신이 이루어집니다.
