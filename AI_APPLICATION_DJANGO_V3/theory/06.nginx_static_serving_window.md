# Nginx를 통한 윈도우 환경 정적 파일 직접 서빙

본 단계에서는 Windows 개발 환경의 파일 시스템 내에 수집되어 있는 정적 파일(Static) 및 업로드 미디어 파일(Media)을 Nginx가 가로채어 직접 웹 브라우저에 고속으로 서빙하도록 경로 매핑 설정을 진행하고 최적화 기법을 적용하는 방법을 학습합니다.

---

## 1. Windows Nginx의 정적 경로 설정 규칙 및 주의사항

Nginx 설정 파일(`nginx.conf`)에서 디렉터리 경로를 지정할 때 Windows 운영체제 고유의 규칙 때문에 종종 구동 에러를 겪습니다. 이를 예방하기 위해 아래 규칙을 준수해야 합니다.

- **슬래시(/) 사용 권장**: Windows의 기본 경로 구분 기호는 백슬래시(`\`)이지만, Nginx에서는 이를 확장 이스케이프 문자(예: `\n`, `\t` 등)로 해석할 수 있어 인식이 불가능할 수 있습니다. 따라서 경로 구분자는 반드시 **슬래시(`/`)**를 사용해야 합니다.
  - 잘못된 예: `alias C:\project\staticfiles\;`
  - 올바른 예: `alias C:/project/staticfiles/;`
- **끝 슬래시 일치**: `location /static/` 과 같이 슬래시로 매핑을 끝냈다면, 내부의 `alias` 지시어 경로 끝에도 반드시 `/` 슬래시를 붙여 매핑을 맞춰주어야 매칭 에러가 나지 않습니다.

---

## 2. 정적 파일 직접 서빙의 성능상 이점

Django는 동적인 비즈니스 로직(데이터 조회, 로그인 연산 등)을 실행하도록 최적화된 프레임워크입니다. 이미지, CSS, JS와 같은 변하지 않는 정적 파일을 파이썬 인터프리터 수준에서 읽어서 전송하는 것은 CPU와 메모리 자원을 크게 낭비하는 일입니다.

반면 Nginx는 C언어로 구현된 비동기 이벤트 기반 서버로, OS 커널 수준의 `sendfile` 시스템 콜을 사용하여 파일을 디스크에서 네트워크 소켓으로 직접 버퍼 복사 없이 고속 전송할 수 있습니다. 

따라서 정적 및 미디어 요청을 Nginx가 직접 처리하도록 프록시 우회 설정을 하면 다음 효과를 얻습니다.
- Waitress의 작업 스레드들이 정적 서빙 업무에서 해방되어 동적 요청 처리에 더 집중할 수 있습니다.
- 웹 페이지 리소스 로딩 속도가 획기적으로 향상됩니다.

## 3. 단계별 구성 가이드

### 1단계: Nginx 구성 디렉터리 구성
프로젝트 루트 폴더 옆에 `nginx/` 폴더를 생성하고 설정 파일 `nginx.conf`를 작성합니다.

```text
django_project/       # Django 웹 소스 코드 및 Waitress 실행 코드
nginx/
└── nginx.conf        # Windows용 Nginx 전체 설정 파일
```

### 2단계: Nginx 설정 파일 구현
Nginx 기동을 위해 기존 기본 설정을 완전히 대체할 수 있는 전체 설정 규격을 정의합니다. 정적 파일 및 미디어 파일의 Windows 실제 절대 경로를 alias로 연결하고 gzip 전송을 적용합니다.

- 파일명: `nginx/nginx.conf`
```nginx
worker_processes  1;

events {
    worker_connections  1024;
}

http {
    include       mime.types;
    default_type  application/octet-stream;

    sendfile        on;
    keepalive_timeout  65;

    upstream django_app {
        server 127.0.0.1:8000;
    }

    server {
        listen 80;
        server_name localhost;

        # 파일 업로드 크기 제한 해제 (50MB)
        client_max_body_size 50M;

        # Gzip 압축 전송 설정
        gzip on;
        gzip_comp_level 5;
        gzip_min_length 1024;
        gzip_proxied any;
        gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;

        # 로깅 파일 설정
        access_log logs/django_access.log;
        error_log logs/django_error.log;

        # 1. 정적 파일 직접 서빙 (Windows 경로 예시)
        # Windows Nginx에서는 파일 경로 구분자로 슬래시(/)를 사용하는 것이 오류를 막는 가장 안전한 방법입니다.
        location /static/ {
            alias C:/Users/Playdata/Downloads/skn29_4_lecture/skn29_4_lecture/day07/06_nginx_static_serving_windows/django_project/staticfiles/;
            expires 30d;
            add_header Cache-Control "public, no-transform";
            access_log off;
        }

        # 2. 미디어 파일 직접 서빙 (Windows 경로 예시)
        location /media/ {
            alias C:/Users/Playdata/Downloads/skn29_4_lecture/skn29_4_lecture/day07/06_nginx_static_serving_windows/django_project/media/;
            expires 7d;
            add_header Cache-Control "public, no-transform";
        }

        # 3. Waitress WAS로 동적 요청 전달
        location / {
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            proxy_set_header Host $http_host;
            proxy_redirect off;
            proxy_pass http://django_app;
        }
    }
}
```

### 2단계: Waitress 실행 코드 적용
Windows 로컬 테스트 안정성을 확보하기 위해 static/media 서빙 래핑 핸들러가 적용된 `run_waitress.py` 코드를 제공합니다. Nginx가 오프라인이거나 개별 테스트 시에도 문제없이 로드됩니다.

- 파일명: `django_project/run_waitress.py`
```python
import os
from urllib.parse import urlparse
from waitress import serve
from django.core.wsgi import get_wsgi_application
from django.contrib.staticfiles.handlers import StaticFilesHandler
from django.views.static import serve as django_static_serve
from django.conf import settings

# 중요: Django 설정 모듈(Settings)을 가리키는 환경 변수를 최상단에서 기본값으로 지정합니다.
os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'todoboard.settings.production')

class StaticAndMediaFilesHandler(StaticFilesHandler):
    def __init__(self, application):
        super().__init__(application)
        self.media_url = urlparse(settings.MEDIA_URL)

    def _should_handle(self, path):
        return (path.startswith(self.base_url.path) and not self.base_url.netloc) or \
               (path.startswith(self.media_url.path) and not self.media_url.netloc)

    def serve(self, request):
        if request.path.startswith(self.media_url.path):
            path = request.path[len(self.media_url.path):]
            return django_static_serve(request, path, document_root=settings.MEDIA_ROOT)
        return super().serve(request)

application = StaticAndMediaFilesHandler(get_wsgi_application())

if __name__ == '__main__':
    print("Starting Waitress WSGI server on http://127.0.0.1:8000")
    serve(application, host='127.0.0.1', port=8000, threads=4)
```

## 4. 구동 테스트 및 효과 확인

Nginx 설정 파일을 갱신한 후 Django 정적 자원을 수집하고 Nginx 설정을 재로드하여 최종 접속 효과를 확인합니다.

```cmd
:: 1. 운영용 데이터베이스 마이그레이션 적용
python manage.py migrate --settings=todoboard.settings.production

:: 2. 운영(production) 환경을 기준으로 정적 파일을 하나의 물리 디렉터리에 수집합니다.
python manage.py collectstatic --settings=todoboard.settings.production --noinput

:: 3. Waitress 및 Nginx 구동 후 Nginx 설정 재로드 적용
nginx -s reload
```

웹 페이지에 재접속한 뒤 개발자 도구(F12)의 Network 탭을 통해 CSS, JS 등의 자원을 확인했을 때, 아래 지표가 정상 조회되는지 확인합니다.
- **Content-Encoding: gzip**: Nginx가 정적 리소스를 gzip 알고리즘으로 압축하여 저용량 전송 중임을 의미합니다.
- **Cache-Control: public, no-transform**: 브라우저 로컬 저장소 캐시를 정상 획득했음을 의미하며 페이지 새로고침 속도가 극대화됩니다.
