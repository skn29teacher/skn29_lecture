# 환경 설정 및 Django 프로젝트 구조 이해

Django 개발 환경을 구축하고 프로젝트와 애플리케이션을 직접 생성하는 방법부터 생성된 모든 구성 파일의 상세한 역할까지 알아봅니다. 이 매뉴얼을 따라 하며 새로운 디렉토리에서 프로젝트를 직접 구성할 수 있습니다.

---

## Django 개발 환경 설정 및 프로젝트 생성 절차

Django 프로젝트를 처음부터 구축할 때 실행하는 실무 명령어와 수행 절차입니다.

### 가상환경 활성화
의존성 충돌을 방지하기 위해 지정된 가상환경을 활성화하고 작업을 진행합니다.
```bash
conda activate agents
```

### Django 프로젝트 생성
프로젝트를 생성할 상위 폴더를 준비하고, 해당 폴더 내에서 장고 어드민 도구를 이용해 프로젝트 골격을 생성합니다.
```bash
# 1. 프로젝트 폴더 생성 및 이동 (예시)
mkdir todoboard
# (폴더 이동 생략 - 현재 위치한 프로젝트 루트 경로에서 실행)

# 2. Django 프로젝트 초기 설정 생성
django-admin startproject todoboard .
```
- 뒤에 붙는 마침표(`.`)는 현재 명령어를 실행 중인 디렉토리를 장고 프로젝트의 루트 디렉토리로 사용하겠다는 의미입니다.
- 이 마침표가 누락되면 `todoboard`라는 폴더가 중첩으로 생성되어 경로 관리가 까다로워집니다.

### 애플리케이션 생성
Django는 전체 웹서비스를 기능 단위로 쪼갠 앱(Application) 구조로 개발합니다. 프로젝트 루트 디렉토리(manage.py가 있는 위치)에서 다음 명령을 실행합니다.
```bash
python manage.py startapp todos
```
- `todos`는 할일 관리 기능을 전담할 애플리케이션의 이름입니다.

## 프로젝트 루트 폴더 내 구성 요소 상세 설명

프로젝트 생성 시 설계되는 폴더와 파일들의 세부 구조 및 동작 원리입니다.

### manage.py (프로젝트 관리 스크립트)
장고 프로젝트 관리를 위해 가장 빈번하게 실행하는 파이썬 스크립트입니다. 개발 서버 실행, 데이터베이스 관리, 슈퍼유저 생성 등 다양한 명령을 내장하고 있습니다.
- `python manage.py runserver`: 로컬 웹 서버(기본 8000 포트)를 실행합니다.
- `python manage.py startapp <앱이름>`: 새로운 기능별 앱 디렉토리를 생성합니다.
- `python manage.py makemigrations`: 모델의 변경 사항을 추적하여 마이그레이션 설계 도면(파이썬 파일)을 작성합니다.
- `python manage.py migrate`: 마이그레이션 도면을 기반으로 실제 데이터베이스 테이블을 구축하거나 변경합니다.
- `python manage.py createsuperuser`: 관리자 사이트(Admin)에 접속할 수 있는 최고 권한 계정을 만듭니다.

## 프로젝트 설정 디렉토리 (todoboard/todoboard/) 내 파일 분석

`startproject` 명령 시 생성되는 설정 패키지 내부 파일입니다. 프로젝트 전체에 영향을 주는 전역 속성을 조절합니다.

### __init__.py
- 역할: 현재 폴더가 파격적인 단순 디렉토리가 아닌, 파이썬이 인식 가능한 패키지(import가 가능한 단위)임을 선언하는 빈 파일입니다.

### settings.py (전역 설정 파일)
프로젝트 전반에 걸친 보안 정책, 데이터베이스 접속 정보, 연동 애플리케이션 등을 관리하는 장고의 심장부입니다.
- `BASE_DIR`: 프로젝트의 최상위 루트 디렉토리 경로를 계산하여 담아두는 변수입니다. 파일 경로를 동적으로 읽어올 때 기준점이 됩니다.
- `SECRET_KEY`: 암호화 서명에 사용되는 비밀 키입니다. 세션 관리, 쿠키 보안 등에 쓰이며 운영 환경에서는 외부에 유출되지 않도록 엄격히 은폐해야 합니다.
- `DEBUG`: 개발 모드(`True`)와 운영 모드(`False`)를 분기합니다. 개발 모드에서는 상세한 에러 페이지와 디버그 정보를 제공하지만, 운영 환경에서는 보안을 위해 반드시 `False`로 바꿔야 합니다.
- `ALLOWED_HOSTS`: 해당 장고 서버가 통신을 수락할 호스트 주소(도메인 또는 IP)를 지정하는 보안 필터 리스트입니다.
- `INSTALLED_APPS`: 프로젝트에 포함되어 작동할 모든 애플리케이션 리스트입니다. 우리가 `startapp`으로 만든 앱도 반드시 여기에 추가해야 장고가 모델, 뷰 등을 인식합니다.
- `MIDDLEWARE`: HTTP 요청이 뷰에 도달하기 전과 뷰가 응답을 반환한 후에 순차적으로 거치는 일련의 보안 및 처리 필터들입니다. (예: 세션 처리, CSRF 방어, 공통 Common 처리 등)
- `ROOT_URLCONF`: 장고가 사용자의 웹 요청 주소를 최초로 분석하기 시작할 기준 `urls.py` 주소를 정의합니다.
- `TEMPLATES`: HTML 화면을 가공할 템플릿 엔진의 종류와 템플릿 검색 경로(`DIRS`)를 세팅합니다.
- `DATABASES`: 사용할 데이터베이스 종류 및 커넥션 세팅 정보입니다. 기본값으로는 파일 기반 경량 데이터베이스인 SQLite3가 지정되어 있습니다.
- `LANGUAGE_CODE`: 다국어 설정을 조율합니다. 한국어 표시를 위해 `'ko-kr'`로 값을 수정합니다.
- `TIME_ZONE`: 날짜 시간 처리 기준시를 지정합니다. 국내 기준인 `'Asia/Seoul'`로 지정합니다.
- `AUTH_USER_MODEL`: 프로젝트 내에서 기본으로 채택할 회원 모델을 설정합니다. (기본값은 `'auth.User'`)

### urls.py (메인 URL 라우팅 파일)
- 역할: 사용자가 웹 브라우저 주소창에 친 URL 주소와 이를 처리할 뷰(View) 함수를 1대1로 연결해 주는 이정표 파일입니다. 주소를 규칙적으로 매칭하기 위해 `path()` 함수를 사용합니다.

### wsgi.py / asgi.py (게이트웨이 파일)
- 역할: 장고 웹 애플리케이션이 실제 운영 서버 환경(Nginx, Apache, Gunicorn 등)과 통신하기 위해 준수해야 하는 표준 규격 인터페이스 진입 파일입니다. `wsgi`는 동기식 방식, `asgi`는 실시간 소켓 통신 등을 지원하는 비동기식 방식 인터페이스를 담당합니다.

## 애플리케이션 디렉토리 (todoboard/todos/) 내 파일 분석

`startapp` 명령 시 생성되며, 실질적인 비즈니스 로직과 서비스 기능이 들어가는 코드 폴더입니다.

### migrations/ (마이그레이션 역사 보관 폴더)
- 역할: `models.py`에 정의한 데이터 모델의 역사가 고스란히 저장되는 폴더입니다. 데이터베이스 테이블 구조의 변경 이력서들이 파이썬 스크립트 파일 형태로 자동 축적됩니다.

### admin.py (관리자 설정 파일)
- 역할: 장고가 자랑하는 웹 기반 내장 관리자 화면(Admin)에 등록할 모델과 상세 관리 페이지의 레이아웃 커스텀 설정을 지정하는 장소입니다.

### apps.py (앱 설정 파일)
- 역할: 해당 애플리케이션의 메타데이터 정보(시스템 상 고유 명칭 등)가 정의되는 설정 파일로, 수정할 일이 거의 없습니다.

### models.py (데이터 설계 도면)
- 역할: 데이터베이스의 테이블 스키마를 파이썬 클래스 형태로 설계하는 장소입니다. SQL을 몰라도 이곳에 변수와 타입을 적으면 장고 ORM이 이를 테이블 구조로 자동 변환해 줍니다.

### tests.py (테스트 작성 파일)
- 역할: 구현한 코드의 신뢰성을 보장하고 향후 리팩토링 시 사이드 이펙트를 방지하기 위해 자동화된 기능 테스트 코드를 적어 두는 파일입니다.

### views.py (비즈니스 컨트롤러)
- 역할: 사용자의 요청을 직접 넘겨받아 DB에서 값을 읽거나 연산을 처리한 뒤, 최적의 HTML 페이지나 JSON 형식 등의 데이터(응답)를 생성해 반환해 주는 가장 중요한 컨트롤러 로직 작성 파일입니다.

## 실무적인 초기 설정 변경 가이드

프로젝트와 앱을 생성한 직후 진행해야 하는 핵심 필수 설정들입니다.

### 1. Custom User 모델 정의 (todos/models.py)
유저 관련 스키마 확장을 용이하게 하기 위해 개발 첫 단추에 바로 Custom User 모델을 생성합니다.
```python
from django.db import models
from django.contrib.auth.models import AbstractUser

class CustomUser(AbstractUser):
    nickname = models.CharField(max_length=50, blank=True)

    def __str__(self):
        return self.username
```

### 2. settings.py 수정 (todoboard/settings.py)
새로 생성한 애플리케이션을 등록하고, 다국어 시간 설정 및 Custom User 모델을 세팅합니다.
```python
# 1. 앱 목록에 todos 등록
INSTALLED_APPS = [
    ...
    'todos',
]

# 2. 다국어 및 시간대를 한국으로 교체
LANGUAGE_CODE = 'ko-kr'
TIME_ZONE = 'Asia/Seoul'

# 3. Custom User 선언 정보 반영 (맨 아래쪽에 작성 추천)
AUTH_USER_MODEL = 'todos.CustomUser'
```

### 3. 마이그레이션 실행 및 데이터베이스 생성
설정이 완료되면 모델 정보를 데이터베이스에 굽는 절차를 통해 실습을 정상 구동할 수 있습니다.
```bash
python manage.py makemigrations
python manage.py migrate
```
- 정상 수행되면 프로젝트 루트 폴더에 `db.sqlite3` 파일이 자동 생성됩니다.

## Django 개발 서버 실행 및 접속 확인 방법

프로젝트와 데이터베이스 설정이 완료된 후, 로컬 개발 환경에서 웹 서비스를 가동하고 접속하는 세부 절차입니다.

### 1. 개발 서버 구동 명령어
`manage.py`가 있는 프로젝트 루트 디렉토리에서 다음 명령을 실행합니다.
```bash
python manage.py runserver
```
- 기본적으로 장고는 로컬 호스트(`127.0.0.1`)의 `8000`번 포트를 통해 웹서비스를 가동합니다.
- 만약 다른 포트번호(예: 8080)로 구동하고 싶다면 명령어 뒤에 포트번호를 지정합니다: `python manage.py runserver 8080`

### 2. 브라우저 접속 확인
서버 구동 메시지가 터미널에 정상 출력되면, 웹 브라우저를 열고 주소창에 아래의 주소를 입력하여 접속을 확인합니다.
- 서비스 메인 주소: `http://127.0.0.1:8000/` 또는 `http://localhost:8000/`
- 장고 초기 환영 페이지 또는 설정한 루트 URL 뷰가 정상적으로 로드되는지 확인합니다.

### 3. 개발 서버 종료
가동 중인 개발 서버를 중지하려면 터미널 창에서 다음 단축키를 입력합니다.
- `Ctrl + C`
