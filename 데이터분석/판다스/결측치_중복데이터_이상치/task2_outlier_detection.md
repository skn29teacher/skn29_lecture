# 📘 Task 2 — 이상치(Outlier) 탐지 및 처리

## 1. 이상치란?

**이상치(Outlier)**는 데이터 분포에서 극단적으로 벗어난 값을 말합니다.

| 유형 | 설명 | 예시 |
|------|------|------|
| 자연 이상치 | 실제로 발생한 극단값 | 억만장자의 소득 |
| 입력 오류 | 데이터 입력 실수 | 나이 = 300, 급여 = -50,000 |
| 측정 오류 | 센서·장비 오작동 | 체온 = 200°C |
| 표본 오류 | 잘못된 모집단 혼입 | 성인 데이터에 영아 포함 |

>  **왜 중요한가?**
> - 평균·분산이 왜곡됩니다 (평균이 이상치 방향으로 끌려감)
> - 회귀 모델의 기울기가 크게 변합니다
> - 클러스터링·분류 성능이 저하됩니다

---

## 2. IQR(사분위 범위) 방식

### 원리

```
Q1 = 25번째 백분위수
Q3 = 75번째 백분위수
IQR = Q3 - Q1

하한(Lower Bound) = Q1 - 1.5 × IQR
상한(Upper Bound) = Q3 + 1.5 × IQR

→ 하한 미만 또는 상한 초과 = 이상치
```

### 코드

```python
def detect_outliers_iqr(data, column):
    Q1 = data[column].quantile(0.25)
    Q3 = data[column].quantile(0.75)
    IQR = Q3 - Q1
    lower = Q1 - 1.5 * IQR
    upper = Q3 + 1.5 * IQR
    outlier_mask = (data[column] < lower) | (data[column] > upper)
    return outlier_mask, lower, upper

# 이상치 제거
mask, lower, upper = detect_outliers_iqr(df, 'salary')
df_clean = df[~mask]
```

### 특징

| 장점 | 단점 |
|------|------|
| 정규분포를 가정하지 않음 | 1.5 배수가 항상 최적은 아님 |
| 중앙값 기반이라 극단값에 강건 | 다변량 이상치를 탐지하지 못함 |
| 구현이 간단하고 직관적 | 데이터가 매우 편향되면 오탐 가능 |

## 3. Z-score 방식

### 원리

```
Z = (X - μ) / σ

|Z| > 3 이면 이상치로 판단 (일반적 기준)
→ 평균에서 3 표준편차 이상 떨어진 값
```

### 코드

```python
from scipy import stats

z_scores = np.abs(stats.zscore(df['salary']))
outlier_mask = z_scores > 3
df_clean = df[~outlier_mask]
```

### 특징

| 장점 | 단점 |
|------|------|
| 통계적 근거가 명확 (3σ 규칙) | **정규분포를 가정** |
| 표준화된 기준으로 비교 용이 | 이상치 자체가 평균·표준편차를 왜곡 |
| 임계값 조절 가능 (2, 2.5, 3 등) | 표본 크기가 작으면 Z-score가 불안정 |

---

## 4. IQR vs Z-score 비교

```
                IQR 방식                  Z-score 방식
─────────────────────────────────────────────────────────
분포 가정    없음 (비모수적)             정규분포 가정
기준점       Q1, Q3 (중앙값 기반)        평균, 표준편차
이상치에      강건                        취약 (왜곡됨)
  대한 저항
민감도       보통 (1.5×IQR)              높음 (3σ)
적합한 경우  편향된/비정규 분포           정규분포에 가까운 데이터
```

### 실습 결과 비교

| 방식 | 원본 행 수 | 제거 후 행 수 | 제거 행 수 | 제거 비율 |
|------|:--------:|:----------:|:--------:|:-------:|
| IQR | 112 | 102 | 10 | 8.93% |
| Z-score | 112 | 106 | 6 | 5.36% |
| Capping | 112 | 112 | 0 | 0% |

>  **IQR이 더 많은 이상치를 탐지한 이유**  
> Z-score 방식은 이상치 자체가 평균과 표준편차를 끌어올려, 극단값마저도 Z-score가 3 이내에 들어오게 만듭니다. 이를 **"마스킹 효과(Masking Effect)"**라 합니다.

---

## 5. 정규분포를 따르지 않을 때의 주의점

실무 데이터는 대부분 정규분포를 따르지 않습니다.

1. **Z-score의 한계**: 3σ 규칙은 정규분포에서만 "99.7%가 ±3σ 안에 있다"는 의미입니다. 비정규 분포에서는 이 기준이 무의미할 수 있습니다.

2. **IQR이 더 안전한 선택**: IQR은 분포 가정이 없으므로, 편향된 데이터(소득, 부동산 가격 등)에 더 적합합니다.

3. **로그 변환 후 Z-score**: 오른쪽으로 치우친(양의 편향) 데이터라면 `np.log1p()` 변환 후 Z-score를 적용하면 효과적입니다.

4. **Modified Z-score**: 평균 대신 중앙값, 표준편차 대신 MAD(Median Absolute Deviation)를 사용하는 변형 Z-score는 비정규 분포에서도 안정적입니다.

```python
# Modified Z-score
from scipy.stats import median_abs_deviation
median = df['salary'].median()
mad = median_abs_deviation(df['salary'])
modified_z = 0.6745 * (df['salary'] - median) / mad
outliers = abs(modified_z) > 3.5
```

## 6. 이상치 처리 전략

| 전략 | 설명 | 적합한 경우 |
|------|------|------------|
| **삭제** | 이상치가 포함된 행 제거 | 입력 오류가 확실한 경우 |
| **Capping** | 상/하한 경계값으로 대치 | 데이터 손실을 방지하고 싶을 때 |
| **변환** | 로그·제곱근 변환 | 편향된 분포 정규화 |
| **별도 분석** | 이상치만 따로 분석 | 이상치 자체가 분석 대상일 때 |
