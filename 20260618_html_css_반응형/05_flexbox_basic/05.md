# Flexbox 레이아웃 기초

## 1. 학습 목표
* 과거 테이블 마크업 및 float 기반 레이아웃 기법의 한계점과 부작용을 이해하고, 이를 극복하기 위해 등장한 1차원 레이아웃 모델 Flexbox의 출현 배경을 파악합니다.
* Flex 컨테이너(부모)와 Flex 아이템(자식) 간의 위계적 구조 관계를 이해합니다.
* 주축(Main Axis)과 교차축(Cross Axis)의 개념을 이해하고, `flex-direction`에 따라 두 축의 정렬 흐름이 동적으로 전환되는 원리를 파악합니다.
* 컨테이너 정렬 핵심 속성(`justify-content`, `align-items`, `flex-wrap`, `gap`)과 아이템 제어 속성(`flex-grow`, `flex-shrink`, `flex-basis`, `align-self`)의 의미와 역할을 숙지합니다.

---

## 2. Flexbox 등장 배경 (float 방식과의 비교)
기존에 웹 페이지의 요소를 가로 정렬하기 위해 주로 활용되던 방법은 `float: left;` 또는 `float: right;` 였습니다. 

* **기존 float 방식의 문제점**: 
  1. float 속성을 가진 자식 요소를 부모 요소가 제대로 인식하지 못해 **부모 영역 높이가 0으로 주저앉는 버그**가 지속적으로 발생했습니다. 이를 해결하기 위해 부모에 `clearfix`라는 가짜 해제용 가상 요소를 덧붙이거나 `overflow: hidden;`을 임시방편으로 걸어야만 했습니다.
  2. 세로 정렬(Vertical Center) 구현이 매우 까다로워 수동 픽셀값이나 `transform: translateY` 계산을 동원해야 했습니다.
  3. 형제 요소들 간의 높이를 동일하게 맞추는 등의 1차원 비율 분배 배치가 불가능에 가까웠습니다.

* **Flexbox의 혁신**:
  `display: flex;` 한 줄만 선언하면, 부모-자식 높이가 즉시 연동되고, 상하좌우 모든 정렬 및 간격 조절이 선언형 코드로 깔끔하게 완성됩니다.

---

## 3. 주축(Main Axis)과 교차축(Cross Axis) 개념
Flexbox 레이아웃 모델은 **주축(Main Axis)**과 **교차축(Cross Axis)**이라는 직교하는 두 축을 기준으로 삼습니다.

```
[flex-direction: row 인 경우 (기본값)]

           주축 (Main Axis): 좌 -> 우 (justify-content가 정렬 제어)
        ┌─────────────────────────────────────────────────────────┐
        │  [Item 1]    [Item 2]    [Item 3]                       │
  교차축│                                                         │
  (Cross│                                                         │
  Axis) │                                                         │
   ↓    │                                                         │
        └─────────────────────────────────────────────────────────┘
        (align-items 가 교차축 정렬 제어)
```

```
[flex-direction: column 인 경우]

        주축 (Main Axis): 위 -> 아래 (justify-content가 세로 정렬 제어)
        ┌──────────────────┐
        │   [Item 1]       │
        │                  │
        │   [Item 2]       │
        │                  │
  교차축│   [Item 3]       │ (Cross Axis: 좌 -> 우, align-items가 가로 정렬 제어)
   →    │                  │
        └──────────────────┘
```

> [!IMPORTANT]
> `flex-direction: column`으로 전환되면 주축이 세로로 바뀌고 교차축이 가로로 회전합니다. 이에 따라 `justify-content`는 세로(주축) 방향 정렬을, `align-items`는 가로(교차축) 방향 정렬을 조작하게 되므로 헷갈리지 않게 반드시 숙지해야 합니다.

## 4. Flex 컨테이너(부모)에 쓰는 핵심 속성

### 1) `flex-direction` (정렬 방향 설정)
* `row` (기본값): 왼쪽에서 오른쪽으로 가로 배치
* `row-reverse`: 오른쪽에서 왼쪽으로 가로 배치
* `column`: 위에서 아래로 세로 배치
* `column-reverse`: 아래에서 위로 세로 배치

### 2) `justify-content` (주축 정렬 방식)
* `flex-start` (기본값): 시작점 기준으로 요소를 몰아서 정렬
* `flex-end`: 끝점 기준으로 요소를 몰아서 정렬
* `center`: 주축 한가운데로 요소를 모아 정렬
* `space-between`: 첫 요소와 끝 요소를 양끝 벽에 고정하고 남은 요소들 간격을 동일하게 자동 배치
* `space-around`: 각 요소의 좌우에 동일한 마진 여백 부여 (요소 간 여백이 양끝 벽 여백의 2배가 됨)
* `space-evenly`: 양끝 벽과 요소들 사이의 모든 빈 여백 간격을 균등하게 배분

### 3) `align-items` (교차축 정렬 방식 - 한 줄 기준)
* `stretch` (기본값): 아이템에 높이(또는 너비) 지정이 없을 때 교차축 크기만큼 100% 늘려서 채움
* `flex-start`: 교차축 시작점(위쪽)에 배치
* `flex-end`: 교차축 끝점(아래쪽)에 배치
* `center`: 교차축 세로 한가운데 정렬 (수직 중앙 정렬의 해법)
* `baseline`: 폰트의 밑줄 기준선(Base Line)에 맞춰 텍스트 크기가 달라도 정렬

### 4) `flex-wrap` (줄바꿈 여부)
* `nowrap` (기본값): 부모 너비가 모자라도 찌그러져서 한 줄에 욱여넣어짐 (줄바꿈 안 됨)
* `wrap`: 부모 너비가 부족하면 자식 요소가 아래 줄로 자동으로 밀려 내려감
* `wrap-reverse`: 줄바꿈 시 위쪽 줄로 밀려 올라감

### 5) `gap` (아이템 간의 간격)
* `gap: 15px;` 형태로 선언하며, 아이템과 아이템 사이에만 고유한 여백을 만듭니다. (마진처럼 첫 요소나 끝 요소의 벽 쪽에는 불필요한 마진이 튀어나오지 않아 레이아웃 계산에 극히 편리함)

## 5. Flex 아이템(자식)에 쓰는 개별 제어 속성

### 1) `flex-grow` (팽창 비율)
* **정의**: 부모 컨테이너 내부에 남는 잉여 여백이 있을 때, 해당 자식이 여백을 나누어 먹으며 늘어날 비율을 의미합니다.
* **기본값**: `0` (여백이 남아도 스스로 늘어나지 않음)
* **비율**: A 아이템이 `1`, B 아이템이 `2`이면 남는 잉여 여백을 1:2의 지분 비율로 가져가 팽창합니다.

### 2) `flex-shrink` (수축 비율)
* **정의**: 컨테이너의 크기가 자식들의 총합보다 작아져 공간이 부족할 때, 자식을 찌그러뜨려 축소할 비율을 설정합니다.
* **기본값**: `1` (공간 부족 시 균등 비율로 축소됨)
* **설정**: `0`으로 지정하면 부모 창이 좁아져도 자신의 절대 크기 크기를 훼손당하지 않고 원래 크기를 유지합니다.

### 3) `flex-basis` (기본 시작 크기)
* **정의**: `flex-grow`나 `flex-shrink`에 의해 팽창/수축하기 전, 요소가 가지는 고유의 기본 시작 픽셀 크기 값을 가리킵니다. (`width`와 유사한 역할이지만 Flexbox 환경에서는 `flex-basis`가 우선 적용됨)
* **기본값**: `auto`

### 4) `flex` 축약형 속성
* `flex: grow shrink basis;` 의 형태로 한 번에 축약 선언합니다.
* **예시**: `flex: 1;` -> `flex-grow: 1; flex-shrink: 1; flex-basis: 0%;` 를 생략 선언한 것으로, 남는 모든 여백 공간을 다른 형제와 균등하게 나눠 가지며 부피를 꽉 채우겠다는 핵심 실무 관용구입니다.

### 5) `align-self` (개별 정렬)
* 부모가 정해준 전체 `align-items` 규칙을 무시하고, **해당 개별 자식 아이템만** 홀로 교차축에서 다른 방향으로 탈선하여 정렬되도록 지정합니다.

---

## 6. 핵심 키워드 정리
* **주축/교차축**: `flex-direction`에 종속되어 정렬의 척도를 가르는 교차선.
* **justify-content vs align-items**: 각각 주축 방향 정렬과 교차축 방향 정렬을 처리하는 속성.
* **flex-wrap**: 자식 요소가 부모 범위를 탈출하려 할 때 아래 라인으로 자동 갱신할지 여부.
