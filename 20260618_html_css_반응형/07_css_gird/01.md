# CSS Grid 레이아웃

---

## 학습 목표



- CSS Grid의 핵심 개념(컨테이너, 아이템, 트랙, 셀, 갭)을 설명할 수 있다.
- `grid-template-columns`, `grid-template-rows`로 격자를 자유롭게 정의할 수 있다.
- `fr`, `repeat()`, `minmax()`, `auto-fit/auto-fill`을 활용한 유연한 레이아웃을 구성할 수 있다.
- `grid-template-areas`로 직관적인 대시보드 레이아웃을 설계할 수 있다.
- `grid-column`, `grid-row`로 아이템을 특정 셀에 배치하거나 병합할 수 있다.
- Flexbox와 Grid의 차이를 이해하고 상황에 맞게 선택할 수 있다.

---

## 1. Flexbox(1차원) vs Grid(2차원)

CSS 레이아웃의 양대 산맥인 Flexbox와 Grid는 **적용 차원**이 다릅니다.  
"무엇을 쓸까?" 헷갈릴 때 아래 표를 기준으로 판단하세요.

| 비교 항목 | Flexbox | Grid |
|---|---|---|
| **차원** | 1차원 (행 또는 열 중 하나) | 2차원 (행과 열 동시) |
| **배치 방향** | 단방향 (가로 또는 세로) | 양방향 (가로 + 세로) |
| **주요 사용처** | 네비게이션 메뉴, 카드 정렬, 버튼 그룹 | 페이지 레이아웃, 대시보드, 갤러리 |
| **아이템 크기 제어** | 콘텐츠 기반 (flex-grow/shrink) | 명시적 트랙 기반 (fr, px, %) |
| **정렬 기준** | 주축(main axis) / 교차축(cross axis) | 행 축 / 열 축 |
| **겹침 배치** | 불가 (기본) | 가능 (grid-column/row 으로) |
| **브라우저 지원** | IE 10+ (구형도 가능) | IE 11+ (일부 제한) |

###  선택 기준 공식

```
컴포넌트 단위의 1방향 배열    → Flexbox
페이지 전체 혹은 복잡한 2D 배치 → Grid
두 기술을 함께 쓰는 것이 가장 강력합니다 (Grid 외곽 + Flex 내부)
```

## 2. Grid 기본 개념

### 2.1 용어 정리

```
┌──────────────────────────────────────┐
│          Grid Container              │
│  ┌────────┬────────┬────────┐        │
│  │  Cell  │  Cell  │  Cell  │ ← Row  │
│  ├────────┼────────┼────────┤  Track │
│  │  Cell  │  Cell  │  Cell  │        │
│  └────────┴────────┴────────┘        │
│    ↑ Column Track                    │
│         ↕ Gap (row-gap)              │
│       ↔ Gap (column-gap)             │
└──────────────────────────────────────┘
```

| 용어 | 설명 |
|---|---|
| **Grid Container** | `display: grid`가 선언된 부모 요소 |
| **Grid Item** | 컨테이너의 직계 자식 요소 |
| **Grid Line** | 격자를 나누는 선 (번호로 참조: 1, 2, 3...) |
| **Grid Cell** | 행선과 열선으로 둘러싸인 하나의 격자 칸 |
| **Grid Track** | 두 선 사이의 공간 (행 트랙 또는 열 트랙) |
| **Grid Area** | 하나 이상의 셀로 이루어진 직사각형 영역 |
| **Gap** | 트랙과 트랙 사이의 여백 |

### 2.2 그리드 라인 번호 시스템

```
    1    2    3    4   ← Column Lines
  ┌────┬────┬────┐
1 │    │    │    │
  ├────┼────┼────┤
2 │    │    │    │
  ├────┼────┼────┤
3 │    │    │    │
  └────┴────┴────┘
↑ Row Lines
```

- 열 라인: 왼쪽부터 1, 2, 3, 4 (n열이면 n+1개)
- 행 라인: 위쪽부터 1, 2, 3, 4 (n행이면 n+1개)
- 음수 인덱스: -1(마지막), -2(끝에서 두 번째)...

## 3. Grid 컨테이너 만들기

```css
.container {
  display: grid;        /* 블록 레벨 그리드 */
  display: inline-grid; /* 인라인 레벨 그리드 */
}
```

`display: grid`를 선언하는 순간 직계 자식들이 **Grid Item**이 됩니다.

## 4. grid-template-columns / grid-template-rows

격자의 열과 행 크기를 정의하는 핵심 속성입니다.

### 4.1 단위 종류

| 단위 | 설명 | 예시 |
|---|---|---|
| `px` | 고정 픽셀 크기 | `200px` |
| `%` | 컨테이너 너비 기준 비율 | `50%` |
| `fr` | 남은 공간의 비율 (**fraction**) | `1fr 2fr` |
| `auto` | 콘텐츠 크기에 맞게 자동 조절 | `auto` |
| `min-content` | 콘텐츠의 최소 크기 | `min-content` |
| `max-content` | 콘텐츠의 최대 크기 | `max-content` |

#### `fr` 단위 동작 원리

```css
.container {
  grid-template-columns: 1fr 2fr 1fr;
  /* 전체 너비를 4등분 (1+2+1=4)
     첫 번째 열: 1/4
     두 번째 열: 2/4 (가장 넓음)
     세 번째 열: 1/4         */
}
```

```
┌────────┬────────────────┬────────┐
│  1fr   │      2fr       │  1fr   │
│ (25%)  │     (50%)      │ (25%)  │
└────────┴────────────────┴────────┘
```

> **💡 `fr` vs `%` 차이**  
> `%`는 gap을 무시하고 계산, `fr`은 gap을 제외한 남은 공간을 기준으로 계산합니다.  
> 따라서 gap이 있는 경우 `fr`이 더 정확합니다.

### 4.2 repeat() 함수

같은 패턴을 반복할 때 사용합니다.

```css
/* 기본 문법 */
grid-template-columns: repeat(횟수, 크기);

/* 예시들 */
grid-template-columns: repeat(3, 1fr);
/* 동일: 1fr 1fr 1fr */

grid-template-columns: repeat(4, 200px);
/* 동일: 200px 200px 200px 200px */

grid-template-columns: repeat(3, 1fr 2fr);
/* 동일: 1fr 2fr 1fr 2fr 1fr 2fr → 6열 */

/* 혼합 사용 */
grid-template-columns: 250px repeat(3, 1fr);
/* 첫 열 250px 고정, 나머지 3열 균등 분배 */
```

### 4.3 minmax() 함수

트랙의 **최소**와 **최대** 크기를 동시에 지정합니다.

```css
/* 문법 */
minmax(최솟값, 최댓값)

/* 예시 */
grid-template-columns: minmax(200px, 1fr);
/* 최소 200px, 최대는 남은 공간 전체 */

grid-template-columns: repeat(3, minmax(150px, 1fr));
/* 3열, 각 열은 최소 150px ~ 최대 균등 */

/* 행에도 적용 가능 */
grid-template-rows: minmax(100px, auto);
/* 최소 100px, 콘텐츠가 많으면 자동 확장 */
```

### 4.4 auto-fit vs auto-fill

`repeat()`와 함께 사용하여 **컨테이너 너비에 따라 열 수를 자동으로 결정**합니다.

```css
/* auto-fill: 빈 트랙도 공간 차지 */
grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));

/* auto-fit: 빈 트랙을 접어서 아이템이 늘어남 */
grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
```

#### 시각적 차이 (아이템 3개, 컨테이너 넓을 때)

```
auto-fill (빈 칸 유지):
┌─────┬─────┬─────┬─────┬─────┐
│  1  │  2  │  3  │     │     │  ← 빈 트랙 존재
└─────┴─────┴─────┴─────┴─────┘

auto-fit (빈 칸 제거):
┌───────────┬───────────┬───────────┐
│     1     │     2     │     3     │  ← 아이템이 공간 채움
└───────────┴───────────┴───────────┘
```

| 상황 | 권장 값 |
|---|---|
| 아이템 고정 크기 유지 | `auto-fill` |
| 아이템이 공간을 채워야 할 때 | `auto-fit` |
| 카드 그리드 (일반적) | `auto-fit` + `minmax()` |

## 5. gap (여백)

```css
.container {
  gap: 20px;             /* row-gap과 column-gap 동시 설정 */
  gap: 20px 40px;        /* row-gap: 20px, column-gap: 40px */
  
  row-gap: 20px;         /* 행 사이 여백 */
  column-gap: 40px;      /* 열 사이 여백 */
}
```

> **⚠️ 구형 속성명**  
> 과거에는 `grid-gap`, `grid-row-gap`, `grid-column-gap`을 사용했으나  
> 현재 표준은 `gap`, `row-gap`, `column-gap`입니다.

## 6. grid-template-areas — 영역 이름으로 레이아웃 설계

가장 직관적인 레이아웃 방법입니다. CSS 안에서 레이아웃을 **ASCII 아트**처럼 그릴 수 있습니다.

### 6.1 기본 문법

```css
.container {
  display: grid;
  grid-template-areas:
    "영역명1 영역명2 영역명3"  /* 1행 */
    "영역명4 영역명5 영역명6"  /* 2행 */
    "영역명7 영역명8 영역명9"; /* 3행 */
}
```

### 6.2 대시보드 레이아웃 예제

#### 레이아웃 설계도

```
┌─────────────────────────────────┐
│            HEADER               │
├──────────┬──────────┬───────────┤
│          │          │           │
│   NAV    │   MAIN   │   ASIDE   │
│ (사이드바) │ (콘텐츠) │  (위젯)   │
│          │          │           │
├──────────┴──────────┴───────────┤
│            FOOTER               │
└─────────────────────────────────┘
```

#### CSS 코드

```css
.dashboard {
  display: grid;
  grid-template-columns: 200px 1fr 250px;
  grid-template-rows: 60px 1fr 50px;
  grid-template-areas:
    "header header header"
    "nav    main   aside"
    "footer footer footer";
  min-height: 100vh;
  gap: 10px;
}

/* 각 아이템에 grid-area 할당 */
.header { grid-area: header; }
.nav    { grid-area: nav;    }
.main   { grid-area: main;   }
.aside  { grid-area: aside;  }
.footer { grid-area: footer; }
```

#### HTML 구조

```html
<div class="dashboard">
  <header class="header">헤더</header>
  <nav class="nav">사이드 내비게이션</nav>
  <main class="main">메인 콘텐츠</main>
  <aside class="aside">위젯 영역</aside>
  <footer class="footer">푸터</footer>
</div>
```

### 6.3 영역 병합하기

같은 이름을 반복하면 자동으로 병합됩니다.

```css
grid-template-areas:
  "header header header"  /* header가 3칸 차지 */
  "nav    main   main"    /* main이 2칸 차지 */
  "nav    footer footer"; /* nav가 2행 차지 */
```

```
┌──────┬──────┬──────┐
│         header      │
├──────┬──────┬──────┤
│      │             │
│ nav  │    main     │
│      ├──────┬──────┤
│      │    footer   │
└──────┴──────┴──────┘
```

> **규칙**: 같은 이름의 셀은 반드시 **직사각형** 모양을 이루어야 합니다.  
> 빈 칸으로 두려면 `.` (마침표)을 사용합니다.

## 7. grid-area로 아이템 배치

### 7.1 이름으로 배치 (grid-template-areas와 함께)

```css
.item { grid-area: main; } /* 'main' 영역에 배치 */
```

### 7.2 라인 번호로 배치 (축약형)

```css
/* grid-area: 행시작 / 열시작 / 행끝 / 열끝 */
.item {
  grid-area: 1 / 2 / 3 / 4;
  /* row-start: 1, column-start: 2
     row-end: 3,   column-end: 4  */
}
```

## 8. grid-column / grid-row — 개별 아이템 배치

### 8.1 기본 문법

```css
.item {
  /* grid-column: 시작라인 / 끝라인 */
  grid-column: 1 / 3;   /* 1번 열선 ~ 3번 열선 (2열 차지) */
  grid-row: 1 / 2;      /* 1번 행선 ~ 2번 행선 (1행 차지) */
}
```

### 8.2 span 키워드

```css
.item {
  grid-column: 1 / span 2;  /* 1번 열선부터 2칸 차지 */
  grid-column: span 3;       /* 현재 위치에서 3칸 차지 */
  
  grid-row: 2 / span 3;     /* 2번 행선부터 3행 차지 */
}
```

### 8.3 음수 라인 번호

```css
.item {
  grid-column: 1 / -1;   /* 첫 열부터 마지막 열까지 (전체 너비) */
  grid-row: -3 / -1;     /* 끝에서 세 번째 행부터 마지막 행까지 */
}
```

### 8.4 배치 예시

```
3×3 그리드에서:
┌────┬────┬────┐
│ A  │    B   │  A: grid-column: 1/2
├────┼────┼────┤  B: grid-column: 2/4
│         │ D │  C: grid-column: 1/3, grid-row: 2/4
│    C    ├────┤  D: grid-column: 3/4, grid-row: 2/3
│         │ E │  E: grid-column: 3/4, grid-row: 3/4
└─────────┴────┘
```

```css
.A { grid-column: 1 / 2; grid-row: 1 / 2; }
.B { grid-column: 2 / 4; grid-row: 1 / 2; }
.C { grid-column: 1 / 3; grid-row: 2 / 4; }
.D { grid-column: 3 / 4; grid-row: 2 / 3; }
.E { grid-column: 3 / 4; grid-row: 3 / 4; }
```

## 9. 정렬 속성

### 9.1 컨테이너 기준 (모든 아이템에 적용)

| 속성 | 기본값 | 설명 |
|---|---|---|
| `justify-items` | `stretch` | 아이템을 **열 축(수평)** 기준으로 정렬 |
| `align-items` | `stretch` | 아이템을 **행 축(수직)** 기준으로 정렬 |
| `place-items` | `stretch` | `align-items` / `justify-items` 축약형 |

```css
.container {
  justify-items: start | end | center | stretch;
  align-items:   start | end | center | stretch;
  
  /* place-items: align justify; */
  place-items: center;         /* 가로세로 모두 가운데 */
  place-items: start end;      /* 세로: start, 가로: end */
}
```

### 9.2 개별 아이템 기준 (특정 아이템에만 적용)

| 속성 | 설명 |
|---|---|
| `justify-self` | 해당 아이템만 수평 정렬 |
| `align-self` | 해당 아이템만 수직 정렬 |
| `place-self` | 축약형 |

```css
.special-item {
  justify-self: end;     /* 이 아이템만 오른쪽 정렬 */
  align-self: center;    /* 이 아이템만 수직 가운데 */
  place-self: center end;
}
```

### 9.3 justify-content / align-content (그리드 전체 정렬)

그리드 트랙 전체가 컨테이너보다 작을 때 트랙 묶음 전체를 정렬합니다.

```css
.container {
  justify-content: start | end | center | space-between | space-around | space-evenly;
  align-content:   start | end | center | space-between | space-around | space-evenly;
}
```

## 10. 자동 배치 (Implicit Grid)

명시적으로 지정하지 않은 아이템은 **암묵적 그리드**에 자동 배치됩니다.

```css
.container {
  grid-auto-flow: row;    /* 기본: 행 방향으로 자동 배치 */
  grid-auto-flow: column; /* 열 방향으로 자동 배치 */
  grid-auto-flow: dense;  /* 빈 공간을 채우는 밀집 배치 */
  
  grid-auto-rows: 150px;     /* 암묵적 행의 기본 크기 */
  grid-auto-columns: 100px;  /* 암묵적 열의 기본 크기 */
}
```

## 11. 전체 속성 요약표

### 컨테이너 속성

| 속성 | 역할 |
|---|---|
| `display: grid` | 그리드 컨테이너 선언 |
| `grid-template-columns` | 열 트랙 정의 |
| `grid-template-rows` | 행 트랙 정의 |
| `grid-template-areas` | 영역 이름으로 레이아웃 정의 |
| `grid-template` | 위 세 가지 축약형 |
| `gap` | 행·열 간격 |
| `row-gap` | 행 간격 |
| `column-gap` | 열 간격 |
| `justify-items` | 모든 아이템 수평 정렬 |
| `align-items` | 모든 아이템 수직 정렬 |
| `place-items` | 축약형 |
| `justify-content` | 트랙 전체 수평 정렬 |
| `align-content` | 트랙 전체 수직 정렬 |
| `grid-auto-flow` | 자동 배치 방향 |
| `grid-auto-rows` | 암묵적 행 크기 |
| `grid-auto-columns` | 암묵적 열 크기 |

### 아이템 속성

| 속성 | 역할 |
|---|---|
| `grid-column` | 열 방향 배치 (start / end) |
| `grid-row` | 행 방향 배치 (start / end) |
| `grid-area` | 영역 이름 또는 축약형 배치 |
| `justify-self` | 개별 수평 정렬 |
| `align-self` | 개별 수직 정렬 |
| `place-self` | 축약형 |
