# Flexbox 실전 레이아웃 구현

## 1. 학습 목표
* 헤더, 가변형 카드 목록, 멀티 컬럼 푸터 등 현대 웹 페이지의 빈출 레이아웃 구성요소를 Flexbox로 완전히 조작할 수 있습니다.
* 부모도 Flexbox이고, 그 내부의 자식 요소 역시 Flexbox 컨테이너가 되는 **Flexbox 중첩(Nesting)** 설계의 원리를 파악합니다.
* `flex: 1` 선언이 가지는 속성값의 기하학적 의미(`flex-grow:1`, `flex-shrink:1`, `flex-basis:0%`)를 이해하고, 이를 이용해 남은 높이/너비를 꽉 채우거나 버튼을 박스 하단에 밀어붙이는 실무 패턴을 적용합니다.
* `flex-wrap: wrap`과 퍼센티지 너비 또는 `flex-basis`를 이용해 화면 크기 감소 시 자연스럽게 카드가 다음 줄로 넘어가는 반응형 기초 배열을 흉내 냅니다.

---

## 2. 실무 레이아웃의 3대 핵심 패턴

### 1) 헤더 (GNB 양끝 배치)
* **특징**: 왼쪽의 로고와 오른쪽의 내비게이션 메뉴 묶음을 양끝단 벽에 완전히 밀어붙이고 세로를 한가운데 맞추는 패턴입니다.
* **해법**:
  ```css
  .header-container {
    display: flex;
    justify-content: space-between;
    align-items: center;
  }
  ```

### 2) 가로 세로 비율 고정 카드 갤러리 (Card Grid)
* **특징**: 여러 장의 카드를 가로 방향으로 나열하다가 화면 가로 폭이 부족하면 카드를 깨뜨리지 않고 다음 라인으로 줄바꿈합니다.
* **해법**:
  ```css
  .card-container {
    display: flex;
    flex-wrap: wrap; /* 줄바꿈 허용 */
    gap: 20px; /* 카드간 고정 여백 */
  }
  .card-item {
    flex: 1 1 calc(33.33% - 20px); /* 3열 배치 기본 시작, 최소 부피 보장 */
    min-width: 280px; /* 모바일에서 찌그러지지 않고 1열로 떨어지게 하는 방어막 */
  }
  ```

### 3) 3컬럼 푸터 (Multi-column Footer)
* **특징**: 하단 꼬리말 영역에서 회사 소개, 고객 센터 링크, 소셜 미디어 링크 등 3개 이상의 기둥(Column)을 가로로 정렬하며, 모바일 화면에서는 세로로 차곡차곡 쌓이게 만듭니다.
* **해법**: `flex-direction: row`에서 모바일 미디어 진입 시 `flex-direction: column`으로 전환하는 기본 발상을 활용합니다.

## 3. Flexbox 중첩 (Nesting) 패턴
Flexbox 레이아웃을 완성도 높게 짜기 위해서는 중첩 기법을 이해해야 합니다.

```
┌─────────────────────────────────────────────────────────────────┐ <- 1차 부모 (display: flex)
│  ┌──────────────────────┐  ┌──────────────────────┐             │
│  │ [Card 1 (Flex Item)] │  │ [Card 2 (Flex Item)] │             │
│  │                      │  │                      │             │
│  │  ┌────────────────┐  │  │                      │             │
│  │  │  [Card Image]  │  │  │                      │             │
│  │  └────────────────┘  │  │                      │             │
│  │                      │  │                      │             │
│  │  ┌────────────────┐  │  │                      │             │
│  │  │  [Card Title]  │  │  │                      │             │
│  │  └────────────────┘  │  │                      │             │
│  │                      │  │                      │             │
│  │  ┌────────────────┐  │  │                      │             │
│  │  │  [Card Button] │  │  │                      │             │ <- 2차 자식이자
│  │  └────────────────┘  │  │                      │             │    세부 부모 (display: flex,
│  └──────────────────────┘  └──────────────────────┘             │    flex-direction: column)
└─────────────────────────────────────────────────────────────────┘
```

* **바깥쪽 부모**: 카드 리스트 전체를 정렬하기 위해 `display: flex;` 및 `flex-wrap: wrap;`을 부여합니다.
* **안쪽 자식 (카드 개별 박스)**: 그 자체는 바깥 부모의 아이템이지만, 카드 내부의 이미지, 텍스트, 하단 고정 버튼을 세로 일직선 상에 균일한 비율로 배치하기 위해 내부에도 `display: flex;` 및 `flex-direction: column;`을 부여합니다.

## 4. 자주 하는 실수 및 실무 팁

### 1) `flex: 1`의 정확한 정체
* `flex: 1;`은 `flex-grow: 1; flex-shrink: 1; flex-basis: 0%;`와 동일합니다.
* 여기서 `flex-basis: 0%;`로 셋팅되기 때문에 자식 요소의 기존 콘텐츠 고유 크기(`width`)를 완전히 무시하고, **오로지 부모가 가진 전체 남는 가로폭 여백만을 수학적으로 1:1 정확히 나누어 가지게 만듭니다.**
* 만약 `flex-grow: 1;`만 지정하고 `flex-basis: auto;`로 두면 각 자식이 담고 있는 글자 길이에 비례해 카드 가로 크기가 짝짝이가 되는 버그가 일어납니다. 따라서 균등 너비 레이아웃에는 반드시 축약형인 `flex: 1;`을 사용합니다.

### 2) 버튼을 카드의 맨 아래로 완벽히 붙이기 (`margin-top: auto`)
* 카드 내부의 소개글 텍스트 분량이 카드마다 다른 경우, 카드 전체의 높이는 똑같아지더라도 버튼의 위치가 글자 줄 수에 맞춰 삐뚤빼뚤 춤추게 됩니다.
* **해결법**: 카드 개별 박스를 `flex-direction: column`으로 선언하고, 가장 밑에 있는 버튼 요소에 **`margin-top: auto;`**를 선언해줍니다.
* **원리**: Flexbox 환경에서 특정 자식에 `auto` 마진을 주면, 주축 방향의 남은 모든 빈 여백을 해당 마진이 혼자서 전부 흡수해버립니다. 따라서 버튼 위의 빈 공간을 최대로 벌려서 버튼이 카드의 가장 아래쪽 바닥선에 자석처럼 찰칵 고정되도록 만들어 줍니다.

---

## 5. 핵심 키워드 정리
* **flex: 1**: 콘텐츠 너비를 무시하고 남은 공간을 균등 지분율로 채워 가로폭을 분할하는 축약 명령.
* **margin-top: auto (Flexbox 내)**: 세로형 Flex 내부에서 특정 아이템을 바닥 끝단으로 강제 이동시키는 실무 테크닉.
* **Flex 중첩**: 레이아웃 전체 틀과 개별 상세 정보 영역을 각각 1차원 Flex로 상호 제어하는 방식.
