# 박스 모델, 텍스트·배경 스타일링

## 1. 학습 목표
* CSS 박스 모델(Box Model)의 4대 요소인 콘텐츠, 패딩, 테두리, 마진의 개념을 명확히 정의합니다.
* `box-sizing` 속성의 `content-box`와 `border-box` 차이를 픽셀 크기 계산 수식으로 증명하고, 현업에서 `border-box`를 기본 리셋으로 설정하는 명확한 당위성을 파악합니다.
* `display` 속성에 따른 블록(Block), 인라인(Inline), 인라인 블록(Inline-block), 그리고 감추기(None)의 물리적인 렌더링 성격을 비교 이해합니다.
* px(절대 단위), %(상대 단위), em/rem(폰트 비례 단위), vw/vh(뷰포트 비례 단위) 등 CSS 핵심 단위를 용도에 맞게 적용합니다.
* 타이포그래피(글꼴 스택, 굵기, 줄간격 등) 및 배경 제어(background-image, size, position), 박스 입체감 처리(border-radius, box-shadow) 기법을 습득합니다.

## 2. CSS 박스 모델 (Box Model) 구조
브라우저는 모든 HTML 요소를 사각형 모양의 상자(Box)로 규정하여 렌더링 영역을 계산합니다.

```
+------------------------------------------+  <- 바깥 여백 (Margin)
|  +------------------------------------+  |  <- 테두리 선 (Border)
|  |  +------------------------------+  |  |  <- 안쪽 여백 (Padding)
|  |  |                              |  |  |
|  |  |       콘텐츠 (Content)       |  |  |  <- 실제 텍스트나 이미지 영역
|  |  |                              |  |  |
|  |  +------------------------------+  |  |
|  +------------------------------------+  |
+------------------------------------------+
```

* **콘텐츠 (Content)**: 글자나 이미지 등 요소의 실제 내용이 위치하는 공간입니다. (`width`, `height`로 제어)
* **패딩 (Padding)**: 콘텐츠와 테두리 선(Border) 사이의 안쪽 여백입니다. 패딩을 주면 박스가 내부적으로 팽창하며 요소의 배경색이 패딩 영역까지 칠해집니다.
* **테두리 (Border)**: 안쪽 여백과 바깥 여백을 가르는 경계선입니다. 선의 두께, 스타일, 색상을 가집니다.
* **마진 (Margin)**: 다른 인접 요소들과의 간격을 벌리기 위한 테두리 밖의 순수 바깥 여백입니다. 다른 요소와 배경 영역에 영향을 주지 않는 빈 공간입니다.

## 3. box-sizing 속성비교 (박스 크기 계산법)

어떤 요소를 가로 300px, 패딩 20px, 테두리 5px로 설정했을 때 브라우저가 차지하는 실제 가로 폭은 `box-sizing` 설정에 따라 완전히 달라집니다.

### 1) `content-box` (기본값)
* **정의**: `width` 속성으로 지정한 크기가 오직 **콘텐츠 영역**에만 한정됩니다.
* **실제 계산 가로 크기**: `width` + `left padding` + `right padding` + `left border` + `right border`
  * 예시 계산: $300\text{px} + 20\text{px}\times 2 + 5\text{px}\times 2 = 350\text{px}$
* **단점**: 패딩이나 테두리 두께를 수정할 때마다 전체 레이아웃 가로 너비가 비정상적으로 깨져 매번 계산을 다시 해야 합니다.

### 2) `border-box` (실무 표준 권장)
* **정의**: `width` 속성으로 지정한 크기 안에 **콘텐츠, 패딩, 테두리가 모두 포함**됩니다.
* **실제 계산 가로 크기**: 무조건 `width`로 고정됩니다. (콘텐츠 영역 크기가 패딩/테두리 공간만큼 알아서 축소됨)
  * 예시 계산: 콘텐츠 너비($250\text{px}$) + 패딩($40\text{px}$) + 테두리($10\text{px}$) = 딱 $300\text{px}$
* **장점**: 디자이너가 요청한 가이드라인 크기 그대로 수치를 지정하고 안쪽 여백을 자유롭게 조절해도 레이아웃의 전체 구조가 전혀 흐트러지지 않습니다.

## 4. display 속성: block vs inline vs inline-block

| 속성값 | 개행 여부 | 가로 너비 (width) | 마진/패딩 영향 | 주요 태그 |
|:---|:---|:---|:---|:---|
| **`block`** | 자동으로 줄바꿈됨 (한 줄 독점) | 부모 너비의 100%를 기본 차지 | 상하좌우 마진/패딩 모두 정상 작동 | `div`, `p`, `h1~h6`, `ul`, `li` |
| **`inline`** | 줄바꿈 없이 가로 배치 | 콘텐츠 글자 크기만큼만 가로폭 차지 | **상하 마진/패딩이 레이아웃에 적용되지 않음** (가로 방향 패딩/마진만 가능) | `span`, `a`, `strong`, `em` |
| **`inline-block`** | 줄바꿈 없이 가로 배치 (inline의 흐름) | 원하는 대로 가로/세로 폭 지정 가능 | 상하좌우 마진/패딩 모두 정상 작동 | `input`, `button`, `select` (기본값) |
| **`none`** | 화면에서 완전히 삭제 | - | 공간 자체를 차지하지 않고 흔적 없이 사라짐 | (스크립트 제어용 혹은 모바일 메뉴 감춤 등) |

## 5. CSS 단위 체계 정리

* **`px` (Pixels)**: 절대적인 화소 단위로 크기가 항상 일정합니다. 해상도 왜곡이 없는 정밀 정렬에 유용하지만 사용자가 브라우저 글꼴 크기를 키워도 글씨가 커지지 않는 접근성 한계가 있습니다.
* **`%` (Percent)**: 부모 요소의 크기를 기준으로 계산되는 비례 단위입니다.
* **`em`**: **부모 요소의 폰트 크기**를 $1\text{em}$으로 계산하는 상대 단위입니다. (중첩 구조에서 폰트 크기 계산이 복잡해질 위험 존재)
* **`rem` (Root em)**: 최상위 `<html>` 요소의 폰트 크기(기본 브라우저 $16\text{px}$)를 $1\text{rem}$으로 고정 계산합니다. 
  * 예: $1.5\text{rem} = 16\text{px} \times 1.5 = 24\text{px}$
  * *권장*: 반응형 레이아웃 및 텍스트는 브라우저 글꼴 확대 옵션에 호응하기 위해 **`rem`**을 기본 단위로 사용합니다.
* **`vw` / `vh` (Viewport Width / Viewport Height)**: 사용자가 보는 웹 브라우저의 현재 창 크기를 100등분한 단위입니다. (예: `100vh`는 화면 세로 높이의 $100\%$ 꽉 차는 영역을 뜻함)

## 6. 타이포그래피 & 데코레이션 스타일

### 1) 텍스트 스타일 핵심 속성
* `font-family`: 글꼴 목록 설정. 마지막에는 항상 시스템 대체 글꼴 계열(`sans-serif` 등)을 선언해야 안심할 수 있습니다.
* `font-size`: 글씨 크기 (rem 권장).
* `font-weight`: 글씨 두께 (100 ~ 900 단위 또는 `normal`, `bold`).
* `line-height`: 줄간격. 단순히 `px`보다 글씨 크기 대비 배수(예: `1.5` ~ `1.6`)로 적는 것이 텍스트 겹침을 방지하는 표준 기법입니다.
* `text-align`: 텍스트 정렬 (`left`, `center`, `right`, `justify` 양끝정렬).
* `letter-spacing`: 글자 간의 간격(자간). 가독성 튜닝에 필수적입니다.

### 2) 배경 및 꾸밈
* `background-size`: 배경 이미지를 맞추는 기준입니다.
  * `cover`: 이미지 비율을 유지하며 요소 공간을 빈틈없이 꽉 채웁니다.
  * `contain`: 이미지가 잘리지 않는 선에서 요소 공간 내에 최대한 크게 맞춥니다.
* `border-radius`: 테두리 모서리를 둥글게 깎아줍니다. 둥근 사각형이나 원형(`50%`)을 만들 때 활용됩니다.
* `box-shadow`: 요소 섀도우(그림자) 효과를 줍니다.
  * `box-shadow: horizontal vertical blur spread color;` (예: `box-shadow: 2px 4px 10px rgba(0,0,0,0.15);`)

---

## 7. 핵심 키워드 정리
* **box-sizing**: 테두리 두께와 내부 안쪽 여백을 총 가로너비 계산에 편입시킬지 규정하는 속성.
* **rem/em 차이**: 기준이 문서 최상위 폰트크기(`html`)인지, 해당 요소 직속 부모 폰트크기인지에 따른 차이.
* **line-height**: 텍스트의 가독성 있는 행간 확보를 위한 높이 비율 속성.
