# 반응형 웹 디자인 종합 프로젝트

---

## 학습 목표


- 반응형 웹 디자인의 필요성과 핵심 원칙을 설명할 수 있다.
- `meta viewport` 태그의 역할을 이해하고 정확히 작성할 수 있다.
- `@media` 쿼리 문법을 완벽히 이해하고 조건식을 자유롭게 작성할 수 있다.
- 모바일 퍼스트 전략의 이점을 설명하고 실제 코드에 적용할 수 있다.
- 브레이크포인트에 따라 Grid 열 수, Flex 방향, 폰트 크기를 반응형으로 변환할 수 있다.
- Grid와 Flexbox를 혼합하여 복잡한 반응형 레이아웃을 설계할 수 있다.
- 완성형 랜딩 페이지를 구현할 수 있다.

## 1. 반응형 웹 디자인의 필요성

### 1.1 모바일 트래픽 현황

| 연도 | 모바일 웹 트래픽 비율 | 데스크탑 비율 |
|------|----------------------|---------------|
| 2015 | 35.1% | 64.9% |
| 2018 | 52.4% | 47.6% |
| 2021 | 57.0% | 43.0% |
| 2024 | **60% 이상** | 40% 미만 |

> ** 결론**: 전 세계 웹 트래픽의 60% 이상이 모바일에서 발생합니다.  
> 고정 너비 레이아웃은 모바일에서 가로 스크롤이 생기거나 글자가 너무 작아집니다.

### 1.2 반응형 디자인 핵심 원칙 3가지

```
1. 유동 그리드(Fluid Grid)    → px 대신 %, fr, em, rem 사용
2. 유연한 이미지(Flexible Image) → max-width: 100%로 이미지 제한
3. 미디어 쿼리(Media Query)   → 화면 크기에 따른 CSS 규칙 분기
```

## 2. meta viewport 완전 이해

### 2.1 viewport란?

**뷰포트(Viewport)**는 웹 페이지가 표시되는 **브라우저의 보이는 영역**입니다.

```
데스크탑: viewport 너비 ≈ 1200~1920px
태블릿:   viewport 너비 ≈ 768~1024px
모바일:   viewport 너비 ≈ 320~480px
```

### 2.2 viewport 메타 태그

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

| 속성값 | 의미 |
|--------|------|
| `width=device-width` | 뷰포트 너비를 디바이스의 실제 화면 너비로 설정 |
| `initial-scale=1.0` | 페이지 로드 시 초기 확대/축소 배율을 100%로 설정 |

> **⚠️ 이 태그가 없으면?**  
> 모바일 브라우저는 페이지를 데스크탑용(약 980px)으로 렌더링한 후 축소합니다.  
> 결과적으로 글씨가 매우 작아지고 미디어 쿼리도 제대로 작동하지 않습니다.

### 2.3 추가 옵션들

```html
<!-- 사용자 확대/축소 비허용 (접근성 이슈로 비권장) -->
<meta name="viewport" content="width=device-width, initial-scale=1.0, user-scalable=no">

<!-- 최대 확대 배율 제한 -->
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=2.0">
```

## 3. 미디어 쿼리 문법 완전 정리

### 3.1 기본 문법

```css
@media [미디어 타입] and ([조건]) {
  /* 조건이 참일 때 적용되는 CSS */
}
```

### 3.2 미디어 타입

| 타입 | 설명 | 주요 사용 |
|------|------|-----------|
| `all` | 모든 디바이스 (기본값) | 일반적인 스타일 |
| `screen` | 화면 (모니터, 폰, 태블릿) | 반응형 레이아웃 |
| `print` | 인쇄 미리보기 및 인쇄 | 인쇄용 CSS |
| `speech` | 스크린 리더 (음성) | 접근성 |

```css
/* 인쇄 시 불필요한 요소 숨기기 */
@media print {
  .nav, .ads, .footer { display: none; }
  body { font-size: 12pt; }
}
```

### 3.3 주요 미디어 특성(조건)

| 특성 | 설명 | 예시 |
|------|------|------|
| `min-width` | 뷰포트 너비가 이상일 때 | `(min-width: 768px)` |
| `max-width` | 뷰포트 너비가 이하일 때 | `(max-width: 767px)` |
| `min-height` | 뷰포트 높이가 이상일 때 | `(min-height: 600px)` |
| `orientation` | 화면 방향 | `(orientation: landscape)` |
| `hover` | 포인터 기기 호버 가능 여부 | `(hover: hover)` |
| `prefers-color-scheme` | 시스템 다크/라이트 모드 | `(prefers-color-scheme: dark)` |

### 3.4 논리 연산자

#### `and` — 두 조건 모두 만족

```css
/* 768px 이상이면서 1024px 이하인 경우 (태블릿 전용) */
@media screen and (min-width: 768px) and (max-width: 1024px) {
  .container { padding: 0 30px; }
}
```

#### `,` (쉼표) — OR 조건 (둘 중 하나 만족)

```css
/* 작은 화면 또는 인쇄 시 */
@media screen and (max-width: 480px), print {
  .sidebar { display: none; }
}
```

#### `not` — 조건 부정

```css
/* screen이 아닌 디바이스에서 */
@media not screen {
  body { font-family: serif; }
}
```

### 3.5 min-width vs max-width

이것이 모바일 퍼스트 vs 데스크탑 퍼스트의 핵심 차이입니다.

```css
/* ── min-width: 모바일 퍼스트 ──
   "이 너비 이상이면 적용" → 작은 화면이 기준 */
.container { width: 100%; }            /* 기본 (모바일) */
@media (min-width: 768px)  { .container { width: 750px; } }  /* 태블릿↑ */
@media (min-width: 1024px) { .container { width: 960px; } }  /* 데스크탑↑ */

/* ── max-width: 데스크탑 퍼스트 ──
   "이 너비 이하이면 적용" → 큰 화면이 기준 */
.container { width: 960px; }           /* 기본 (데스크탑) */
@media (max-width: 1023px) { .container { width: 750px; } }  /* 태블릿↓ */
@media (max-width: 767px)  { .container { width: 100%; } }   /* 모바일↓ */
```

## 4. 모바일 퍼스트 vs 데스크탑 퍼스트

### 4.1 전략별 장단점 비교

| 항목 | 모바일 퍼스트 | 데스크탑 퍼스트 |
|------|--------------|----------------|
| **기본 CSS** | 작은 화면 기준 | 큰 화면 기준 |
| **미디어쿼리 방향** | `min-width` (점차 확장) | `max-width` (점차 축소) |
| **성능** |  모바일 불필요한 CSS 없음 |  모바일에서 데스크탑 CSS 다운로드 |
| **SEO** |  Google 모바일 우선 색인 |  불리할 수 있음 |
| **개발 순서** | 핵심 기능 먼저 설계 | 풍부한 UI 먼저 설계 |
| **실무 권장** |  **권장** | △ 레거시 프로젝트에서 사용 |

### 4.2 코드 비교

```css
/* ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   모바일 퍼스트 (권장)
   기본 = 모바일, min-width로 점점 확장
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ */
.features {
  display: grid;
  grid-template-columns: 1fr;  /* 모바일: 1열 */
}

@media (min-width: 768px) {    /* 태블릿 이상 */
  .features {
    grid-template-columns: repeat(2, 1fr);  /* 2열 */
  }
}

@media (min-width: 1024px) {   /* 데스크탑 이상 */
  .features {
    grid-template-columns: repeat(3, 1fr);  /* 3열 */
  }
}


/* ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   데스크탑 퍼스트 (비권장)
   기본 = 데스크탑, max-width로 점점 축소
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ */
.features {
  display: grid;
  grid-template-columns: repeat(3, 1fr);  /* 데스크탑: 3열 */
}

@media (max-width: 1023px) {   /* 태블릿 이하 */
  .features {
    grid-template-columns: repeat(2, 1fr);  /* 2열 */
  }
}

@media (max-width: 767px) {    /* 모바일 이하 */
  .features {
    grid-template-columns: 1fr;  /* 1열 */
  }
}
```

> ** 권장** : 모바일 퍼스트를 사용하세요.  
> CSS는 위에서 아래로 적용되므로, 작은 화면 기본 스타일 → `min-width`로 확장하는 방식이  
> 코드 관리와 성능 모두에서 유리합니다.

## 5. 브레이크포인트(Breakpoint) 기준표

브레이크포인트는 레이아웃이 변경되는 뷰포트 너비 기준점입니다.

| 브레이크포인트 | 대상 기기 | 미디어 쿼리 | 일반적 레이아웃 |
|---------------|-----------|------------|----------------|
| `< 576px` | 소형 폰 (iPhone SE 등) | 기본 CSS | 1열, 전체 너비 |
| `≥ 576px` | 중형 폰 (갤럭시, 아이폰) | `@media (min-width: 576px)` | 1~2열 |
| `≥ 768px` | 태블릿 (iPad mini 등) | `@media (min-width: 768px)` | 2열, nav 표시 |
| `≥ 1024px` | 소형 노트북 | `@media (min-width: 1024px)` | 3열, 사이드바 |
| `≥ 1200px` | 데스크탑, 모니터 | `@media (min-width: 1200px)` | 최대 너비 컨테이너 |

```
320px  576px    768px      1024px        1200px
  |─────|─────────|──────────|─────────────|──────→
  폰     폰(중형)   태블릿      노트북         데스크탑
  1열    1열       2열         3열            3열+여백
```

## 6. 반응형 전환 패턴

### 6.1 Grid 컬럼 수 변경

```css
/* 카드 그리드: 1→2→3열 */
.card-grid {
  display: grid;
  grid-template-columns: 1fr;          /* 모바일: 1열 */
  gap: 16px;
}

@media (min-width: 768px) {
  .card-grid { grid-template-columns: repeat(2, 1fr); }  /* 태블릿: 2열 */
}

@media (min-width: 1024px) {
  .card-grid { grid-template-columns: repeat(3, 1fr); }  /* 데스크탑: 3열 */
}
```

### 6.2 flex-direction 전환

```css
/* 네비게이션: 세로 → 가로 */
.nav-list {
  display: flex;
  flex-direction: column;  /* 모바일: 세로 나열 */
  gap: 8px;
}

@media (min-width: 768px) {
  .nav-list {
    flex-direction: row;   /* 데스크탑: 가로 나열 */
    gap: 24px;
  }
}
```

### 6.3 이미지 반응형

```css
img {
  max-width: 100%;  /* 부모 요소를 초과하지 않도록 제한 */
  height: auto;     /* 비율을 유지하며 높이 자동 조절 */
  display: block;   /* 이미지 하단 공백 제거 */
}
```

### 6.4 폰트 크기 조정

```css
/* 클램핑: min ~ max 사이에서 유동적으로 크기 변경 */
h1 {
  font-size: clamp(1.8rem, 4vw, 3.5rem);
  /*         최소      권장    최대   */
}

/* 미디어 쿼리 방식 */
h1 { font-size: 1.8rem; }                        /* 모바일 */
@media (min-width: 768px)  { h1 { font-size: 2.4rem; } }  /* 태블릿 */
@media (min-width: 1024px) { h1 { font-size: 3.2rem; } }  /* 데스크탑 */
```

### 6.5 컨테이너 최대 너비 패턴

```css
.container {
  width: 100%;           /* 모바일: 전체 너비 */
  max-width: 1200px;     /* 데스크탑: 최대 1200px */
  margin: 0 auto;        /* 가운데 정렬 */
  padding: 0 16px;       /* 양옆 기본 여백 */
}

@media (min-width: 768px) {
  .container { padding: 0 24px; }
}

@media (min-width: 1024px) {
  .container { padding: 0 40px; }
}
```

## 7. Flex + Grid 혼합 설계 전략

실무에서는 Flexbox와 Grid를 **목적에 따라 혼합**하여 사용합니다.

```
페이지 레벨 레이아웃 (전체 구조)    → Grid
컴포넌트 내부 정렬 (버튼, 메뉴 등) → Flexbox
카드 갤러리 (2D 배치)               → Grid
단순 수평/수직 정렬                  → Flexbox
```

```css
/* Grid로 전체 레이아웃 설계 */
.page {
  display: grid;
  grid-template-areas:
    "header"
    "main"
    "footer";
}

/* Flex로 헤더 내부 정렬 */
.header {
  display: flex;
  justify-content: space-between;  /* 로고와 메뉴를 양 끝으로 */
  align-items: center;
}

/* Grid로 피처 섹션 카드 배치 */
.features {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
}

/* Flex로 카드 내부 요소 정렬 */
.card {
  display: flex;
  flex-direction: column;
  gap: 12px;
}
```

## 8. 종합 프로젝트 구조

이번 실습에서 구현할 랜딩 페이지의 전체 구조입니다.

```
index.html
├── <header>         ← 헤더 + 네비게이션 (Flexbox, sticky)
│   └── <nav>        ← 메뉴 (모바일: 세로 / 데스크탑: 가로)
├── <section#hero>   ← 히어로 배너 (Flex 중앙 정렬, 전체 화면 높이)
├── <section#features> ← 특징 카드 (Grid: 1→2→3열 반응형)
├── <section#about>  ← 회사 소개 (Grid: 1→2열 반응형)
├── <section#gallery> ← 갤러리 (Flexbox wrap)
├── <section#contact> ← 문의 폼 (CSS 폼 스타일링)
└── <footer>         ← 푸터 (Flexbox: 모바일 세로/데스크탑 가로)
```

### 차시별 기술 활용 현황

| 구역 | 적용 기술 | 학습 차시 |
|------|-----------|-----------|
| `<header>` | display:flex, sticky position | 6차시 |
| `<nav>` | flex-direction 전환 | 6차시 |
| `#hero` | flex 중앙 정렬, vh 단위 | 3, 6차시 |
| `#features` | display:grid, auto-fit, minmax | 7차시 |
| `#about` | grid 2열 레이아웃 | 7차시 |
| `#gallery` | flex-wrap | 6차시 |
| `#contact` | 폼 시맨틱 태그, CSS 스타일링 | 2, 3, 5차시 |
| `<footer>` | flex-wrap, column→row | 6차시 |
| 전체 | CSS 변수, 미디어 쿼리 | 3, 8차시 |
| 전체 | 시맨틱 HTML | 1차시 |

## 9. 마무리 체크리스트

### HTML 구조 점검

- [ ] 시맨틱 태그를 적절히 사용했는가? (`header`, `nav`, `main`, `section`, `footer`)
- [ ] `<meta charset="UTF-8">` 선언이 있는가?
- [ ] `<meta name="viewport" ...>` 선언이 있는가?
- [ ] 이미지에 `alt` 속성을 모두 작성했는가?
- [ ] 폼 요소에 `label`이 연결되어 있는가?

### CSS 스타일 점검

- [ ] CSS 선택자 충돌(specificity 문제)이 없는가?
- [ ] 모바일 기본 스타일이 먼저 작성되어 있는가? (모바일 퍼스트)
- [ ] `max-width` + `margin: 0 auto`로 컨테이너를 가운데 정렬했는가?
- [ ] 이미지에 `max-width: 100%`가 적용되어 있는가?
- [ ] CSS 변수(Custom Properties)로 색상을 일관성 있게 관리했는가?

### 반응형 동작 점검

- [ ] 576px, 768px, 1024px, 1200px 기준으로 레이아웃이 변경되는가?
- [ ] 가로 스크롤이 발생하지 않는가?
- [ ] 텍스트가 너무 작아지거나 겹치지 않는가?
- [ ] 버튼/링크의 터치 영역이 최소 44×44px 이상인가? (모바일 접근성)
