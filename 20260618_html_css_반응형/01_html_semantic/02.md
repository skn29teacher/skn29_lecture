# HTML5 핵심 태그 (텍스트·표·폼·미디어)

## 1. 학습 목표
* 텍스트의 성격과 용도에 부합하는 웹 표준 텍스트 태그를 선택하여 구분해 쓸 수 있습니다.
* 정보를 행과 열로 구조화하여 표시하는 `table` 태그의 명확한 골격을 잡고 셀 병합(`colspan`, `rowspan`)을 사용할 수 있습니다.
* 하이퍼링크(`<a href>`)와 이미지(`<img src>`) 삽입 시 접근성을 위한 필수 속성(`alt`)을 올바르게 기입합니다.
* 사용자의 입력을 받아 백엔드 서버로 전송하는 `form` 태그 및 `input`의 핵심 타입들을 이해하고, `label`의 `for` 연동을 통한 웹 접근성 표준 규격을 만족합니다.
* 미디어 영역의 구조적 정의를 돕는 `<figure>`, `<figcaption>`과 `<video>`, `<audio>` 등의 미디어 삽입 원리를 익힙니다.

---

## 2. 텍스트 태그 완전 정리
HTML은 단순히 텍스트 크기를 키우기만 하는 것이 아니라 문맥(Context)에 따라 의미가 강조되는 별도의 태그를 지원합니다.

| 태그 | 역할 | 사용 시 특징 |
|:---|:---|:---|
| `<p>` | 하나의 독립된 문단(Paragraph)을 나눕니다. | 상/하단에 기본 마진(Margin)이 자동으로 설정되는 Block 요소입니다. |
| `<span>` | 문장 내 특정 단어나 글자를 묶어 디자인을 적용합니다. | 줄바꿈이 없는 Inline 요소로 단순 그루핑 목적에 가장 적합합니다. |
| `<strong>` | **텍스트의 내용상 아주 중요함**을 의미론적으로 강조합니다. | 화면 낭독기에서도 음성의 톤이나 크기를 조절해 중요성을 전달합니다. |
| `<em>` | **텍스트를 구어적으로 강조**할 때 사용합니다. (이탤릭체 표시) | Emphasis의 약자로 단순 폰트 기울임이 아닌 구어적 강세를 부여합니다. |
| `<b>` / `<i>` | 시각적으로만 굵게(Bold) 하거나 기울이게(Italic) 합니다. | **의미적 강조가 전혀 없습니다.** 최신 표준에서는 `strong`, `em` 권장. |
| `<br>` | 줄바꿈(Line Break)을 수행합니다. | 줄 간격을 띄우는 용도로 남발하면 안 되며 문단 구분은 `<p>`를 써야 합니다. |
| `<hr>` | 이야기 주제가 바뀔 때 시각적인 구분선을 긋습니다. | Horizontal Rule의 약자로, 현대 웹 디자인에서는 구분선 뿐 아니라 주제 전환점 의미도 갖습니다. |

## 3. 목록(List) 태그 정리
목록을 그루핑하면 구조화가 명확해지며 CSS 메뉴 바나 게시판 리스트의 원형이 됩니다.

```html
<!-- 1. 비순서형 목록 (Unordered List) -->
<ul>
  <li>사과</li>
  <li>바나나</li>
</ul>

<!-- 2. 순서형 목록 (Ordered List) -->
<ol>
  <li>첫 번째 단계</li>
  <li>두 번째 단계</li>
</ol>

<!-- 3. 정의형 목록 (Description List) -->
<dl>
  <dt>용어</dt>
  <dd>용어에 대한 설명이 들어갑니다.</dd>
</dl>
```
* `<dt>`는 Definition Term(용어의 이름), `<dd>`는 Definition Description(용어의 상세 설명)을 의미합니다. 사전식 설명이나 정보-값 매칭 시 유용합니다.

## 4. 표(Table) 태그 구조의 시각화
표는 구조적 해석이 어렵기 때문에 스크린 리더 등 웹 접근성을 고려하여 `thead`, `tbody`, `tfoot` 등으로 명확히 구획하여 마크업해야 합니다.

### 1) 구조도
```
+------------------------------------------+  <- <table>
|                 제목 행                   |  <- <thead>  (th: 굵은 제목 셀)
+------------------------------------------+
|                 내용 행                   |  <- <tbody>  (td: 일반 데이터 셀)
|                 내용 행                   |
+------------------------------------------+
|                 합계 행                   |  <- <tfoot>
+------------------------------------------+
```

### 2) 셀 병합 속성
* **`colspan="N"`**: 열(Column)을 병합합니다. 가로로 인접한 N개의 셀을 하나로 합칩니다.
* **`rowspan="N"`**: 행(Row)을 병합합니다. 세로로 인접한 N개의 셀을 하나로 합칩니다.

## 5. 링크와 이미지
### 1) 하이퍼링크 `<a>`
* `href` 속성에 이동하고자 하는 목적지 URL을 정의합니다.
* `target="_blank"`를 사용하면 현재 탭이 아닌 새 탭에서 링크를 띄웁니다.
  * *보안 팁*: 새 창 이동 시 `rel="noopener noreferrer"` 속성을 함께 기입하여 부모 창 객체 조작 취약점을 방지합니다.

### 2) 이미지 `<img>`
* `<img src="이미지경로" alt="대체텍스트">`
* **`alt` 속성의 필수성**: 이미지가 로드 실패 시 깨진 그림 대신 대체 텍스트를 제공합니다. 또한 검색 엔진 봇에게 해당 이미지가 어떤 이미지인지를 이해시키며, 시각장애인용 화면 낭독기에서 이미지 대신 읽어 주어 필수적인 대체 정보 역할을 수행합니다. `alt`를 아예 지우는 것은 웹 접근성 위반입니다.

## 6. 폼(Form) 태그와 입력 양식
사용자 정보 입력, 파일 전송, 가입 폼 등은 백엔드와의 통신을 위한 가장 핵심적인 관문입니다.

### 1) `<form>` 속성
* `action`: 입력받은 양식의 데이터를 최종적으로 처리할 백엔드 서버의 URL 주소.
* `method`: 전송 방식 (`GET`: URL 파라미터에 값을 노출하여 전송, `POST`: HTTP Body 패킷 영역에 숨겨서 대용량/보안 전송).

### 2) `<label>`과 `<input>`의 연결
웹 접근성을 높이기 위해 입력창 옆의 글자(Label)를 누르더라도 자동으로 인풋창이 활성화(포커스)되도록 설정해야 합니다.

```html
<!-- label의 for와 input의 id를 일치시킵니다 -->
<label for="user-email">이메일 주소</label>
<input type="email" id="user-email" name="email">
```

### 3) 주요 `<input>` type 속성
* `text`: 일반 한 줄 텍스트 입력
* `password`: 입력 문자가 가려짐(`*` 또는 `●`)
* `email`: 메일 형식 검증 기능 자동 수행
* `tel`: 모바일 기기 접속 시 숫자 키패드 팝업 유도
* `radio`: 그룹 내 단 하나만 선택 가능 (`name` 속성값이 같아야 그룹으로 묶임)
* `checkbox`: 여러 개 다중 선택 가능
* `date`: 달력 형식의 날짜 선택 창 표출
* `submit`: 폼 데이터를 서버로 송신하는 전송 버튼

## 7. 멀티미디어 태그
* **`<figure>`와 `<figcaption>`**: 일러스트레이션, 다이어그램, 사진, 코드 리스트 등을 독립적인 설명 단위로 묶고 그 하단에 캡션(설명글)을 달아 가독성을 높입니다.
* **`<video>` / `<audio>`**: 별도의 플러그인(Flash 등) 없이도 자체 재생 제어 도구(`controls`), 루프 반복(`loop`), 자동재생(`autoplay`), 음소거(`muted`) 등의 속성을 제공합니다.

---

## 8. 핵심 키워드 정리
* **alt 속성**: 이미지 로딩이 불가하거나 스크린 리더를 읽을 때 제공하는 텍스트 대체 정보.
* **label-for 연결**: 레이블 영역 클릭 시 연결된 ID의 양식 창으로 포커싱을 인계하는 접근성 필수 규칙.
* **input type**: 백엔드로 데이터를 넘길 형식을 규정하고 적절한 기기 대응 입력을 돕는 속성.
* **table 구조**: 단순히 표 모양을 만드는 데 그치지 않고 `thead`, `tbody`로 구획을 분할하는 구조화 원칙.
