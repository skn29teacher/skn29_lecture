# CSS3 적용 방법과 선택자

## 1. 학습 목표
* CSS를 웹 페이지에 적용하는 3가지 방식의 차이점을 파악하고, 실무에서 외부 스타일시트(`.css`) 분리 적용 방식을 지향해야 하는 이유를 이해합니다.
* 전체 선택자, 태그 선택자, 클래스 및 ID 선택자의 정확한 구문 규칙을 습득합니다.
* 자손, 자식, 인접 형제, 일반 형제 선택자 등 결합 선택자(Combinators)를 조합하여 원하는 대상 요소를 입체적으로 조작합니다.
* 속성 선택자(`[attr=value]`) 및 다양한 마우스 반응/구조 가상 클래스(`:hover`, `:nth-child()`), 그리고 가상 요소(`::before`, `::after`)를 응용합니다.
* CSS 캐스케이딩(Cascading)의 핵심 원리인 우선순위 점수(Specificity)를 수학적으로 계산할 수 있고, 유지보수를 방해하는 `!important` 구문을 피해야 하는 사유를 파악합니다.

---

## 2. CSS를 적용하는 3가지 방법 비교

| 방식 | 작성 형태 | 장점 | 단점 | 실무 사용 권장도 |
|:---|:---|:---|:---|:---|
| **인라인 스타일 (Inline Style)** | `<div style="color: red;">` | 특정 단일 요소에 스타일을 가장 빠르게 임시로 적용 가능 | HTML과 CSS의 관심사 분리가 무너져 가독성이 저하되며 유지보수가 극도로 불가능함 | **매우 비권장** (JS 동적 변경 제외) |
| **내부 스타일시트 (Embedded Style)** | `<head>` 안에 `<style>` 태그 작성 | 별도의 파일 없이 해당 단일 HTML 문서의 모든 스타일 한 번에 관리 | 다수의 HTML 페이지가 존재할 때 중복 코드가 대량 발생하여 스타일 통제가 불가능함 | **소규모 샘플/단일 테스트** 시에만 제한 권장 |
| **외부 스타일시트 (External Link)** | `<link rel="stylesheet" href="style.css">` | 여러 HTML 문서에서 **동일한 단일 CSS 파일을 캐싱 및 공유**하므로 용량이 절감되고 전체 디자인을 한 번에 제어 가능 | 서버 요청 횟수가 늘어날 수 있음 (현대 브라우저 환경에서는 무시할 만함) | **실무 표준 (적극 권장)** |

---

## 3. 기본 선택자 (Basic Selectors)

* **전체 선택자 (`*`)**: 페이지 내의 모든 요소를 선택합니다. 주로 여백 초기화(CSS Reset) 시 적용됩니다.
* **태그 선택자 (`태그명`)**: 해당 태그명을 가진 모든 HTML 요소를 일괄 선택합니다. (예: `p { color: #333; }`)
* **클래스 선택자 (`.클래스명`)**: HTML에서 `class="..."` 속성을 부여한 요소를 복수 타깃으로 선택합니다. (예: `.btn { border-radius: 4px; }`)
* **ID 선택자 (`#ID명`)**: HTML 문서 내에서 고유해야 하는 `id="..."` 요소를 단일 타깃으로 선택합니다. (예: `#main-header { height: 80px; }`)

---

## 4. 결합 선택자 (Combinators)
각 요소의 포함 관계 및 순서적 형제 관계를 기반으로 대상을 정밀 타격합니다.

* **자손 선택자 (`A B`)**: A 요소 **내부에 중첩된 모든 하위** B 요소를 선택합니다. (단계 깊이에 상관없음)
* **자식 선택자 (`A > B`)**: A 요소 **바로 밑에 위치한 직속 자식**인 B 요소만 선택합니다. (중첩 단계 1단계만 해당)
* **인접 형제 선택자 (`A + B`)**: A 요소의 **바로 뒤에 인접해서 달라붙은 첫 번째 형제** B 요소 하나만 선택합니다.
* **일반 형제 선택자 (`A ~ B`)**: A 요소의 **뒤에 위치한 모든 형제** B 요소를 전부 선택합니다.

---

## 5. 속성 선택자 (Attribute Selectors)
클래스나 ID 외에 태그 내부의 속성값 조건에 대응하여 선택합니다.

* **`[href]`**: `href` 속성이 존재하는 모든 요소를 선택합니다.
* **`[type="text"]`**: `type` 속성값이 정확히 `"text"`인 요소만 선택합니다.
* **`[href^="https"]`**: `href` 속성이 `"https"`로 **시작하는** 요소를 선택합니다.
* **`[href$=".pdf"]`**: `href` 속성이 `".pdf"`로 **끝나는** 요소를 선택합니다.
* **`[class*="btn-"]`**: 클래스명에 `"btn-"`이라는 문자열이 **포함된** 모든 요소를 선택합니다.

## 6. 가상 클래스(Pseudo-Classes)와 가상 요소(Pseudo-Elements)

### 1) 주요 가상 클래스 (상태 및 구조 기반)
* **`:hover`**: 요소 위에 마우스 포인터가 올라갔을 때 적용합니다.
* **`:focus`**: 폼 인풋 양식 등이 키보드 탭이나 마우스 클릭으로 입력 활성화 상태가 되었을 때 적용합니다.
* **`:first-child`**: 부모의 첫 번째 자식 요소를 선택합니다.
* **`:last-child`**: 부모의 마지막 자식 요소를 선택합니다.
* **`:nth-child(N)`**: 부모의 N번째 자식 요소를 선택합니다.
  * `:nth-child(even)` (짝수 번째), `:nth-child(odd)` (홀수 번째), `:nth-child(3n)` (3의 배수 번째) 등 수식을 넣을 수 있어 표 스타일링 등에 자주 쓰입니다.

### 2) 주요 가상 요소 (문서 트리 외 가상 콘텐츠 생성)
* **`::before`**: 선택한 요소의 **가장 앞부분**에 가상의 자식 콘텐츠를 동적으로 추가합니다.
* **`::after`**: 선택한 요소의 **가장 뒷부분**에 가상의 자식 콘텐츠를 동적으로 추가합니다.
  * **필수 사항**: `::before`와 `::after` 사용 시에는 CSS 안에 빈 문자열이라도 좋으니 **`content: "";`** 속성을 반드시 선언해야 렌더링에 노출됩니다. 주로 아이콘, 뷸렛 기호, 데코레이션 선을 넣을 때 사용합니다.

## 7. 우선순위(Specificity) 점수 계산법
여러 CSS 규칙이 동일한 요소를 다르게 스타일링할 때, 브라우저는 우선순위 점수를 계산하여 가장 점수가 높은 스타일을 적용합니다.

### 1) 점수 등급표
1. **`!important`**: 99999점 (우선순위 체계를 교란하므로 실무 사용 극도 기피 대상)
2. **인라인 스타일 (Inline Style)**: 1000점 (HTML 내부에 적은 스타일)
3. **ID 선택자 (`#`)**: 100점
4. **클래스 선택자 (`.`), 속성 선택자, 가상 클래스**: 10점
5. **태그 선택자, 가상 요소**: 1점
6. **전체 선택자 (`*`)**: 0점

### 2) 계산 예시
* `div p` -> 태그 2개 = **2점**
* `div .card p` -> 태그 2개(2점) + 클래스 1개(10점) = **12점**
* `#main-nav .menu-item a` -> ID 1개(100점) + 클래스 1개(10점) + 태그 1개(1점) = **111점**
* `#main-nav .menu-item a:hover` -> ID 1개(100점) + 클래스 1개(10점) + 가상 클래스 1개(10점) + 태그 1개(1점) = **121점**

> [!WARNING]
> 점수가 같을 경우, **가장 마지막(아래쪽)에 선언된 스타일**이 캐스케이딩(Cascading) 원칙에 따라 앞서 기술된 스타일을 덮어씁니다.

---

## 8. 핵심 키워드 정리
* **specificity**: 선택자가 가지는 고유의 가중치 점수.
* **가상 클래스 (Pseudo-Classes)**: 동작 상태나 순서 조건에 반응하는 일종의 상태 필터.
* **가상 요소 (Pseudo-Elements)**: 마크업상 실제로 존재하지 않는 가짜 요소를 생성하여 추가하는 기법.
* **결합 선택자**: 포함 및 인접 관계를 나타내어 CSS 가독성과 정밀성을 향상시키는 방법.
