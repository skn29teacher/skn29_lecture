#  HTML5 문서 구조와 시맨틱 마크업

## 1. 학습 목표
* 웹 브라우저가 HTML 문서를 요청하고 화면에 그리기까지의 기본적인 파싱 및 렌더링 과정을 이해합니다.
* HTML 문서의 표준 구조와 필수적인 메타 데이터 태그들의 의미와 역할을 설명할 수 있습니다.
* 무의미하게 요소를 감싸는 `div` 위주의 구조화와 의미를 내포한 시맨틱 태그(Semantic Tags) 구조화의 차이를 비교하고, 이에 따른 검색 엔진 최적화(SEO) 및 웹 접근성 향상 효과를 이해합니다.
* `h1`부터 `h6`까지의 제목 태그 규칙 및 문서 개요(Outline) 생성 원칙을 준수하여 뼈대 설계를 할 수 있습니다.

---

## 2. 브라우저 렌더링 맛보기 (요청부터 화면 표시까지)
웹 사용자가 브라우저 주소창에 URL을 입력하면 다음과 같은 과정을 거쳐 화면에 결과물이 렌더링됩니다.

```
[사용자 요청] -> (서버로 HTTP Request 전송)
     ↓
[서버 응답]   -> (HTML, CSS, JS 파일 등 HTTP Response 반환)
     ↓
[구문 분석]   -> 1. HTML 파싱 ──> DOM 트리 생성
(Parsing)        2. CSS 파싱  ──> CSSOM 트리 생성
     ↓
[렌더 트리]   -> DOM + CSSOM 결합 (화면에 보이지 않는 display: none 요소 등은 제외)
(Render Tree)
     ↓
[레이아웃]    -> 각 노드의 정확한 위치와 크기 계산 (Reflow)
(Layout)
     ↓
[페인팅]      -> 계산된 값을 바탕으로 화면에 실제 픽셀로 변환 (Repaint)
(Paint)
```

* **HTML 파싱**: 브라우저가 위에서 아래로 한 줄씩 HTML 코드를 읽으면서 기계가 이해할 수 있는 트리 구조인 **DOM(Document Object Model) 트리**를 구축합니다.
* **렌더링**: 완성된 DOM 트리와 CSSOM(CSS Object Model) 트리가 결합하여 사용자 화면에 실제로 그려지게 됩니다. 잘못 작성된 마크업(예: 닫는 태그 누락 등)은 브라우저가 임의로 해석하여 오류를 복구하려 하므로 비효율적이고 비정상적인 출력을 야기할 수 있습니다.

## 3. HTML5 문서의 표준 구조 (뼈대 파헤치기)
가장 표준적인 HTML5 문서 구조는 아래와 같이 선언 및 구성됩니다.

```html
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>문서 제목</title>
</head>
<body>
  <!-- 화면에 나타나는 실제 콘텐츠가 들어갑니다 -->
</body>
</html>
```

### 1) `<!DOCTYPE html>`
* **역할**: 현재 문서가 표준 **HTML5** 문서임을 브라우저에 알리는 선언문입니다.
* **중요성**: 생략되거나 잘못 선언될 경우, 브라우저는 하위 호환성을 유지하기 위해 **쿼크 모드(Quirks Mode)**로 렌더링을 시도하며, 이로 인해 최신 CSS/레이아웃이 정상적으로 작동하지 않을 수 있습니다.

### 2) `<html lang="ko">`
* **역할**: 웹 문서의 주 언어가 한국어(`ko`)임을 나타냅니다.
* **장점**: 화면 낭독기(스크린 리더)를 사용하는 시각 장애인 사용자가 접속했을 때 알맞은 언어 발음(엔진)으로 읽어 주도록 도우며, 검색 엔진이 해당 국가나 언어 기반 검색 시 적절하게 색인화하도록 기여합니다.

### 3) `<head>` 영역
* 브라우저 화면에 직접 렌더링되지는 않지만, 문서의 메타데이터(정보), 캐릭터셋 인코딩 설정, 뷰포트 설정, 외부 리소스(CSS, JS 등) 링크 정보 등을 포함합니다.
* **`<meta charset="UTF-8">`**: 문자 인코딩 방식으로 전 세계 모든 문자를 표현할 수 있는 UTF-8을 지정하여 한글 깨짐 현상을 방지합니다.
* **`<meta name="viewport" content="width=device-width, initial-scale=1.0">`**: 반응형 웹 디자인을 지원하기 위해 필수적입니다. 기기의 가로 해상도 크기(`width=device-width`)에 맞게 웹 페이지의 너비를 설정하고, 초기 확대/축소 배율을 1배(`initial-scale=1.0`)로 고정하여 화면이 모바일이나 태블릿에서 비정상적으로 작게 표현되는 것을 방지합니다.
* **`<title>`**: 브라우저 탭에 표시되는 제목이자 검색 엔진 결과 페이지에서 가장 돋보이는 제목 텍스트로 활용됩니다.

### 4) `<body>` 영역
* 실제 웹 브라우저 뷰포트에 표시될 텍스트, 이미지, 링크, 동영상 등의 모든 visual 요소가 작성되는 본문 영역입니다.

## 4. 시맨틱 마크업(Semantic Markup)이 필요한 이유
'시맨틱(Semantic)'이란 **"의미가 통하는"** 또는 **"의미론적인"**이라는 뜻을 갖습니다. 구조화되지 않은 일반 상자인 `<div>`만으로 도배된 레이아웃 대신, 각각의 역할에 맞는 이름을 가진 태그들을 사용하는 마크업 방식을 의미합니다.

###  왜 시맨틱 마크업을 사용해야 할까요?
1. **검색 엔진 최적화 (SEO)**: 검색 엔진 로봇이 웹 페이지를 크롤링할 때 어떤 부분이 헤더이고 본문인지, 가장 중요한 주요 콘텐츠 영역이 어디인지 정확하게 분류하여 검색 노출 우선순위 결정에 반영합니다.
2. **웹 접근성(Web Accessibility)**: 시각 장애인이 화면 낭독기(스크린 리더)를 통해 웹 서핑을 할 때, 시맨틱 태그는 중요한 탐색 지점(Landmark)을 제공합니다. 사용자는 메뉴(`<nav>`)나 핵심 본문(`<main>`)으로 바로 건너뛰어 청취할 수 있습니다.
3. **개발자 협업 및 유지보수 가독성**: 코드만 보더라도 특정 영역이 전체 페이지의 꼬리말인지, 사이드바인지 직관적으로 이해할 수 있어 유지보수 비용이 대폭 감소합니다.

## 5. 핵심 HTML5 시맨틱 태그 정리

| 태그 | 의미 및 역할 | 주의사항 및 특징 |
|:---|:---|:---|
| `<header>` | 문서나 특정 섹션의 머리말 영역 (주로 로고, 검색창, 내비게이션 포함) | 문서 전체뿐 아니라 `<article>` 등 개별 섹션 내에도 포함될 수 있음 |
| `<nav>` | 다른 페이지나 문서 내부의 다른 영역으로 연결되는 링크 링크 집합 (내비게이션 바, 목차) | 주요 링크에만 사용 (푸터의 단순 약관 링크 등에는 생략 권장) |
| `<main>` | 문서의 핵심 및 고유 콘텐츠 영역 | 문서 내 **단 하나만** 존재해야 함 (IE 미지원 대비로 필요 시 role="main" 명시하기도 함) |
| `<section>` | 문서에서 독립적인 장(Chapter)이나 테마별 콘텐츠 단락을 묶어주는 영역 | 일반적으로 내부에 반드시 제목 태그(`<h2>`~`<h6>`)를 포함하는 것이 권장됨 |
| `<article>` | 뉴스 기사, 블로그 포스트, 포럼 글처럼 그 자체로 독립적인 배포/재사용이 가능한 콘텐츠 | 독립적으로 다른 사이트에 퍼가더라도 문맥상 흐름이 자연스러운 완전한 정보 단락에 사용 |
| `<aside>` | 주요 본문 흐름 외의 주변부 정보 콘텐츠 (광고, 카테고리 목록, 스크롤 탑 버튼 등) | 본문 흐름을 방해하지 않는 주변 콘텐츠에 사용 |
| `<footer>` | 문서 또는 특정 섹션의 바닥글 영역 (저작권 표시, 제작 정보, 패밀리 사이트 등) | 주소나 연락처는 푸터 내에 `<address>` 태그와 함께 사용하는 경우가 많음 |

## 6. 제목 태그(`<h1>` ~ `<h6>`)의 계층 구조 규칙
제목 태그는 브라우저가 웹 문서의 계층(Outline) 구조를 해석할 때 척도가 됩니다.

* **원칙 1: 한 페이지당 `<h1>`은 가급적 하나만 권장됩니다.**
  * 전체 페이지의 대제목이나 로고 영역에 주로 배치됩니다.
* **원칙 2: 계층 구조 순서를 건너뛰어서는 안 됩니다.**
  * `<h1>` 다음에 바로 `<h3>`이 와서는 안 되며, 반드시 부모-자식 흐름에 맞게 `<h2>`를 거쳐 `<h3>`으로 내려가야 합니다. (올바른 순서: `h1` -> `h2` -> `h3` -> `h2` ...)
* **원칙 3: 크기를 줄이기 위한 목적으로 제목 태그 계층을 낮춰 잡아서는 안 됩니다.**
  * 디자인적인 폰트 크기 변경은 CSS의 `font-size` 속성을 통해 처리해야 합니다.

## 7. `div` / `span` vs 시맨틱 태그 사용 기준

* **`div` (Block 요소)**: 의미 없이 구역을 묶을 때만 사용합니다. CSS 레이아웃 스타일링(그루핑, 간격 주기)이나 JS 조작을 위한 타깃 컨테이너로 삼을 때 적합합니다.
* **`span` (Inline 요소)**: 텍스트 줄바꿈 없이 본문 내부의 특정 구절, 단어에만 개별 스타일을 입히거나 스크립트 처리를 하고자 할 때 감싸는 요소입니다.
* **시맨틱 태그**: 의미가 존재하는 단락이나 콘텐츠 영역은 **무조건** 해당 시맨틱 태그로 작성하고, 이를 감싸거나 스타일적으로 보완하는 껍데기 역할에만 `div`를 사용합니다.

## 8. 핵심 키워드 정리
* **DOCTYPE**: 브라우저에게 HTML5 문서 규격 준수를 명시하는 선언문.
* **viewport**: 모바일/태블릿 등의 화면 비율과 배율을 제어하는 메타 정보.
* **시맨틱 태그**: 의미론적 태그로 기계와 인간 모두에게 콘텐츠 구조를 명확히 이해시킴.
* **문서 개요 (Outline)**: 제목 태그(`h1~h6`) 및 섹셔닝 태그를 기준으로 형성되는 문서의 계층적 구조.
