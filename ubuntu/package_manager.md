# 패키지 매니저(apt)를 통한 관리

## 1. 학습 목표
* 데비안(Debian)/우분투 배포판의 표준 설치 규격인 `apt` 명령어의 동작 개념을 이해합니다.
* 실제 필요한 개발 도구를 설치하고 시스템 라이브러리를 동기화합니다.
* 패키지 검색, 버전 확인, 업그레이드, 완전 제거 등 실무 운영 흐름 전체를 익힙니다.

---

## 2. 관련 이론

### 2-1. 패키지 매니저란?

패키지 매니저는 운영체제 공급사가 보장하는 공식 저장소(Repository)에서 소프트웨어를 안전하게 내려받고, 종속 라이브러리를 자동으로 해결하며, 설치·업데이트·삭제를 일관되게 관리해주는 시스템 도구입니다.

**직접 설치 vs 패키지 매니저 비교**:

| 항목 | 직접 설치 (wget/curl) | 패키지 매니저 (apt) |
|------|----------------------|-------------------|
| 의존성 처리 | 수동 | 자동 |
| 보안 검증 | 없음 | 공식 서명 검증 |
| 업데이트 | 수동 재설치 | `apt upgrade` |
| 제거 | 파일 직접 삭제 | `apt remove` |
| 추천 상황 | 최신 버전, 비공식 패키지 | 안정성이 중요한 운영 환경 |

### 2-2. apt 명령어 구조

```
apt [동작] [패키지명] [옵션]
```

| 동작 | 설명 |
|------|------|
| `update` | 원격 저장소의 패키지 인덱스를 최신으로 동기화 (설치 아님!) |
| `upgrade` | 설치된 패키지를 최신 버전으로 일괄 업그레이드 |
| `install` | 패키지 설치 |
| `remove` | 패키지 제거 (설정 파일은 유지) |
| `purge` | 패키지와 설정 파일 모두 완전 제거 |
| `autoremove` | 더 이상 필요 없는 의존성 패키지 자동 제거 |
| `search` | 패키지 이름/설명 검색 |
| `show` | 패키지 상세 정보(버전, 의존성, 설명) 확인 |
| `list --installed` | 현재 설치된 패키지 목록 확인 |

### 2-3. apt update vs apt upgrade 차이

초보자가 가장 많이 헷갈리는 부분입니다.

```
apt update   → "어떤 패키지가 있는지 목록만 새로 받아옴" (실제 설치/변경 없음)
apt upgrade  → "목록 기준으로 설치된 패키지들을 실제로 최신 버전으로 올림"
```

**순서가 중요합니다**:
```bash
apt update    # 1단계: 목록 갱신 (항상 먼저)
apt upgrade   # 2단계: 실제 업그레이드
```

`update` 없이 `install`하면 오래된 버전이 설치되거나 "패키지를 찾을 수 없음" 오류가 납니다.

### 2-4. 주요 실무 도구 패키지

| 패키지 | 용도 | 실무 사용 예 |
|--------|------|------------|
| `curl` | URL 요청/파일 다운로드 | API 테스트, 파일 다운로드 |
| `wget` | 파일 다운로드 | 백그라운드 다운로드, 재시도 자동화 |
| `vim` | 터미널 텍스트 편집기 | 서버에서 직접 설정 파일 수정 |
| `nano` | 초보자용 텍스트 편집기 | vim 익숙하지 않을 때 대안 |
| `lsof` | 열린 파일/포트 조회 | 포트 충돌 진단 |
| `net-tools` | `netstat` 등 네트워크 도구 포함 | 네트워크 상태 확인 |
| `htop` | 인터랙티브 프로세스 모니터 | CPU/메모리 실시간 모니터링 |
| `tree` | 디렉토리 구조 시각화 | 프로젝트 구조 파악 |
| `unzip` | zip 파일 압축 해제 | 배포 패키지 해제 |
| `git` | 버전 관리 | 소스 코드 클론, 배포 |

### 2-5. vim 기본 사용법 (실무 필수)

vim은 서버에서 직접 파일을 수정할 때 거의 필수적으로 씁니다.

| 모드 | 진입 방법 | 설명 |
|------|----------|------|
| 일반 모드 | 기본 상태 / `Esc` | 커서 이동, 복사/붙여넣기 |
| 입력 모드 | `i` 또는 `a` | 텍스트 직접 입력 |
| 명령 모드 | `:` | 저장, 종료 등 명령 실행 |

**자주 쓰는 명령**:
```
i         → 현재 커서 위치에서 입력 시작
Esc       → 입력 모드 종료 → 일반 모드로
:w        → 저장
:q        → 종료 (변경사항 없을 때)
:wq       → 저장 후 종료
:q!       → 저장 없이 강제 종료
/검색어    → 텍스트 검색 (n으로 다음 결과)
dd        → 현재 줄 삭제
yy        → 현재 줄 복사
p         → 붙여넣기
```

## 3. 실습 단계 및 명령어

### 실습 1: 로컬 패키지 인덱스 동기화
```bash
# 신규 도커 컨테이너는 기본 패키지 인덱스가 비어 있으므로 반드시 먼저 실행
apt update
```

### 실습 2: 실무 도구 패키지 설치
```bash
# curl, lsof, vim 3가지를 동시에 설치
# -y 옵션: 중간에 "계속 하시겠습니까?" 질문에 자동으로 Y 응답
apt install -y curl lsof vim
```

### 실습 3: 설치 확인
```bash
curl --version
lsof -v
vim --version | head -n 1
```

### 실습 4: 패키지 검색 및 정보 확인
```bash
# 패키지 이름 검색 (설치 여부 무관)
apt search tree

# 패키지 상세 정보 확인 (버전, 의존성, 설명)
apt show curl

# 현재 설치된 패키지 목록 확인
apt list --installed

# 특정 패키지가 설치됐는지 확인
apt list --installed | grep curl
```

### 실습 5: vim으로 파일 편집 체험
```bash
# vim으로 파일 열기
vim /root/workspace/test_edit.txt

# 순서:
# 1. i 눌러 입력 모드 진입
# 2. 텍스트 입력: "Hello from vim!"
# 3. Esc 눌러 일반 모드로
# 4. :wq 입력 후 Enter로 저장 후 종료

cat /root/workspace/test_edit.txt
```

### 실습 6: 패키지 제거
```bash
# 패키지 제거 (설정 파일은 남김)
apt remove curl -y

# 설정 파일까지 완전 제거
apt purge curl -y

# 불필요한 의존성 패키지 정리
apt autoremove -y
```

## 4. 자주 발생하는 오류와 해결

| 오류 메시지 | 원인 | 해결법 |
|------------|------|--------|
| `Unable to locate package 패키지명` | 인덱스 미동기화 또는 패키지 이름 오류 | `apt update` 후 재시도, 패키지명 재확인 |
| `E: Could not get lock /var/lib/dpkg/lock` | 다른 apt 프로세스가 실행 중 | 잠시 기다리거나 `kill` 후 재시도 |
| 설치 후 명령어를 찾을 수 없음 | 경로 문제 | `which 명령어` 로 경로 확인, `hash -r` 실행 |
| 의존성 충돌 오류 | 패키지 버전 충돌 | `apt -f install` 로 의존성 복구 시도 |

## 5. 서버 운영 관점의 실전 시나리오

### 시나리오 A: 배포 후 누락 라이브러리 설치
```bash
# 이미지 처리 기능에서 오류 발생
# ImportError: libpng.so.16: cannot open shared object file

# 누락된 시스템 라이브러리 설치
apt update
apt install -y libpng-dev

# Python 이미지 처리 라이브러리 재설치
pip install Pillow
```

### 시나리오 B: 서버에서 직접 설정 파일 수정
```bash
# nginx 설정 파일을 서버에서 직접 수정
vim /etc/nginx/nginx.conf

# 수정 후 설정 유효성 검사
nginx -t

# 설정 반영
nginx -s reload
```

### 시나리오 C: 도커 이미지 빌드 시 패키지 설치 자동화
```dockerfile
# Dockerfile 예시
FROM ubuntu:22.04

# 패키지 설치를 한 레이어로 묶어 이미지 크기 최소화
RUN apt update && apt install -y \
    curl \
    vim \
    git \
    && apt clean \
    && rm -rf /var/lib/apt/lists/*
```

> **팁**: Dockerfile에서는 `apt update`와 `apt install`을 반드시 같은 `RUN` 명령에 묶어야 합니다. 별도 레이어로 분리하면 캐시 문제로 오래된 인덱스로 설치될 수 있습니다.
