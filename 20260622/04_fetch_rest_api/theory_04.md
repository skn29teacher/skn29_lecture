# Fetch API 기초와 REST API 개념

## 학습 목표
*   REST API의 핵심 개념을 레스토랑 주문 시스템 비유를 통해 쉽고 직관적으로 이해합니다.
*   자바스크립트 내장 Fetch API의 작동 원리를 배달 앱 비유를 바탕으로 학습합니다.
*   서버 응답 규격인 JSON의 정의와 response.json()을 통한 자바스크립트 객체 개봉 흐름을 파악합니다.
*   JSONPlaceholder의 단일 유저 데이터(GET /users/1)를 호출하여 화면의 단일 카드에 일대일 매핑하는 심플한 비동기 예제를 구현합니다.

## 1. REST API 개념 (비유: 레스토랑 음식 주문 시스템)

### 1) REST API란 무엇인가
*   **RE**presentational **S**tate **T**ransfer API의 약자로, 웹 서버의 자원(데이터)을 이름으로 구분하여 주고받는 대중적인 네트워크 통신 아키텍처 스타일입니다.
*   네트워크 주소창에 `/users/1`과 같이 명확한 명사형 주소를 사용하여 통신하자는 전 세계적인 약속입니다.

### 2) 레스토랑 비유로 알아보는 통신 구조
*   **Client (손님)**: 스마트폰이나 PC 화면을 보며 마우스를 클릭하는 우리 브라우저입니다.
*   **Server (주방)**: 요리 재료(데이터베이스)를 가지고 손님의 요청에 따라 실제로 데이터를 요리해주는 컴퓨터입니다.
*   **REST API (메뉴판)**: 주방에 어떻게 주문할지 주소와 요리법을 정리해둔 소통 규칙입니다.
    *   **GET (조회)**: "1번 요리(데이터) 테이블에 가져다주세요." (데이터 읽기 요청)
    *   **POST (생성)**: "주방에 신규 주문 요리 하나 새로 등록해 주세요." (새로운 데이터 쓰기 요청)
    *   **PUT (수정)**: "가져다주신 1번 요리를 새 요리로 통째로 교체해 주세요." (기존 데이터 전체 덮어쓰기 요청)
    *   **DELETE (삭제)**: "1번 테이블 요리 다 먹었으니 싹 치워주세요." (데이터 삭제 요청)
*   **Resource (요리 종류 / URL)**: `/users`, `/posts` 처럼 우리가 얻고자 하는 실제 자원의 주소입니다.
    *   예: `GET /users/1` -> "1번 사용자(user)의 정보 요리를 조회하겠다."

```text
 [ 손님 (Browser) ]        [ 배달 / 메뉴판 (REST API) ]        [ 주방 (Server) ]
+-------------------+     +----------------------------+     +-------------------+
|                   |     |  GET /users/1 (1번 조회)   |     |                   |
|   화면 카드 렌더  | <== |  POST /users  (신규 생성)  | <== |  데이터베이스에서  |
|                   |     |  DELETE /users/1 (삭제)    |     |  데이터 가공 조리 |
+-------------------+     +----------------------------+     +-------------------+
```

## 2. Fetch API 기초 (비유: 배달 대행 앱 배달원)

### 1) Fetch API란 무엇인가
*   웹 브라우저에 기본적으로 내장된 비동기 네트워크 통신용 표준 자바스크립트 함수입니다.
*   별도의 라이브러리(Axios 등) 설치 없이 `fetch()` 호출 한 줄만으로 서버에 데이터를 요청할 수 있어 현대 웹 개발에서 가장 널리 쓰입니다.

### 2) 백엔드(FastAPI)와 프론트엔드의 통신 장벽: CORS(Cross-Origin Resource Sharing)
*   **교차 출처 자원 공유(CORS)**는 브라우저 보안 정책입니다.
*   실습용 프론트엔드 HTML 페이지는 로컬 정적 서버 포트인 `http://127.0.0.1:3000`에서 로드되고, API 서버는 `http://127.0.0.1:8000`에서 가동됩니다.
*   **포트 번호가 다르므로 브라우저는 보안 위협으로 간주하여 기본적으로 API 차단 요청을 보냅니다.**
*   이를 해결하기 위해, FastAPI 백엔드 코드 내부에서 `CORSMiddleware` 설정을 통해 명시적으로 브라우저 도메인을 허용해주어야 비동기 통신이 완료될 수 있습니다.

### 3) 배달 앱 비유로 알아보는 Fetch의 비동기 라이프사이클
1.  **`fetch('http://127.0.0.1:8000/api/users/1')` 호출 (배달 접수 및 배달원 출발)**:
    *   자바스크립트 엔진이 배달원(Fetch)에게 로컬 주방(FastAPI 서버 URL) 주소로 주문을 다녀오라고 비동기 지시를 내립니다.
    *   배달원이 출발하자마자 제어권을 즉시 반환하므로, 우리 코드는 멈추지 않고 아래 줄로 내려가 화면에 "데이터 조회 중..." 스피너를 먼저 띄웁니다.
2.  **서버 응답 수령 (배달 상자 도착)**:
    *   배달원이 주방에서 갓 조리된 음식 데이터 상자(Response 객체)를 들고 브라우저로 복귀합니다.
3.  **`response.json()` 개봉 (배달 상자 언박싱)**:
    *   수령한 상자는 원시 텍스트 형태(JSON)로 밀봉되어 있습니다.
    *   이 상자를 가공하여 자바스크립트가 먹을 수 있는 진짜 요리(Object 객체)로 개봉하는 메서드가 바로 `response.json()`입니다. 이 개봉 작업 역시 비동기로 처리됩니다.
4.  **`then(data => ...)` (요리 시식 및 화면 렌더링)**:
    *   개봉이 완전히 끝나 반환된 실제 데이터(이름, 이메일 등)를 가지고 브라우저 화면의 텍스트창에 쏙쏙 대입하여 사용자에게 보여줍니다.

```javascript
// Fetch API의 가장 쉽고 기본형태인 단일 리소스 GET 통신 코드
fetch("http://127.0.0.1:8000/api/users/1") // 1. 로컬 FastAPI 주방 주소로 배달원 출발
  .then(response => {
    return response.json(); // 2. 수령한 데이터 배달 상자(JSON) 개봉
  })
  .then(user => {
    // 3. 개봉된 실제 사용자 데이터로 화면 1:1 매핑
    console.log("사용자 이름:", user.name);
    console.log("사용자 이메일:", user.email);
  });
```
