단계별 실습 흐름

  1. Step 1 [Promise 상태 전이]: Geolocation API를 활용해 위치 권한 동의에 따라 Pending -> Fulfilled / Rejected 상태가 어떻게 바뀌는지 추적합니다.
  2. Step 2 [then/catch/finally]: 파일 업로드 실무 모사를 통해 리소스 성공/실패 여부와 관계없이 로딩 상태를 최종 해제하는 .finally()의 역할을 배웁니다.
  3. Step 3 [Promise 체이닝과 return의 원리]: 주문서 작성 -> 결제 요청 -> 알림 발송을 단계별로 이으며, 체인 중간에서 반드시 새로운 Promise를 return 해야 하는 원리를 배웁니다.
  4. Step 4 [Promise.all 병렬 최적화]: 세 개의 독립 데이터를 연달아 기다리는 비효율을 Promise.all을 통해 동시에 호출하여 대기 속도를 3배 개선합니다.
