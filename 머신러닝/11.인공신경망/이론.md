
## 일반 퍼셉트론(Perceptron) 먼저 이해하기
- 수학공식:
  - z = w^T x + b
  - y_hat = 1 if z >= 0 else 0
  - w := w + η * (y - y_hat) * x
  - b := b + η * (y - y_hat)
- 공식 풀이:
  - z는 입력 x를 가중합한 "판단 점수"입니다.
  - y_hat은 임계값(0) 기준으로 0/1을 결정한 최종 예측입니다.
  - (y - y_hat)는 예측 오차이며, 오차가 0이면 업데이트가 없습니다.
  - 오차가 있으면 오차 방향으로 w, b를 조정해 다음 예측을 개선합니다.
- 샘플 적용(AND 데이터 한 점):
  - 초기값: w=[0, 0], b=-0.3, η=0.2
  - 샘플: x=[1, 1], y=1
  - 계산: z = 0\*1 + 0\*1 - 0.3 = -0.3 -> y_hat=0
  - 오차: y - y_hat = 1 - 0 = 1
  - 업데이트:
    - w' = [0,0] + 0.2\*1\*[1,1] = [0.2, 0.2]
    - b' = -0.3 + 0.2\*1 = -0.1
  - 해석: 같은 x=[1,1]에 대한 점수 z가 커져 정답(1)에 가까워집니다.

## [이론] 다층 퍼셉트론(MLP) 학습의 핵심 알고리즘과 수식
- 다층 퍼셉트론의 학습은 "예측값과 정답의 차이를 줄이도록 w와 b를 조금씩 바꾸는 과정"입니다.
즉, 한 번에 정답을 맞히는 것이 아니라, 틀린 방향을 보고 다음 예측을 더 좋게 만드는 반복 과정입니다.
1) MLP 학습의 기본 흐름
- 순전파(Forward): 입력 x가 층을 지나 예측값 ŷ를 만듭니다.
- 손실 계산(Loss): 실제값 y와 예측값 ŷ의 차이를 수치화합니다.
- 역전파(Backpropagation): 손실이 줄어드는 방향으로 각 파라미터의 영향도를 계산합니다.
- 업데이트(Update): 계산된 방향으로 w와 b를 조금 조정합니다.
2) 대표 손실 함수
- 분류에서 자주 쓰는 손실:
  - 이진 분류: Binary Cross-Entropy
  - 다중 분류: Cross-Entropy
- 회귀에서 자주 쓰는 손실:
  - MSE = (1/m) * Σ(y - ŷ)^2
- 직관:
  - 손실이 클수록 모델이 많이 틀렸다는 뜻입니다.
  - 학습은 손실을 줄이는 방향으로 진행됩니다.
3) 역전파의 핵심 아이디어
- 체인 룰(chain rule)을 이용해 "각 층이 손실에 얼마나 기여했는가"를 뒤에서 앞으로 계산합니다.
- 대표적인 표현:
  - ∂L/∂w
  - ∂L/∂b
- 의미:
  - L: 손실
  - w, b: 학습할 파라미터
- 직관:
  - 최종 오차를 처음 층까지 거슬러 올라가며 책임을 나눠 갖는 방식입니다.  
4) 다층 퍼셉트론 학습 전략
- 가중치 초기화: 너무 크거나 작지 않게 시작
- 활성화 함수 선택: ReLU/sigmoid/tanh
- 학습률(learning rate): 너무 크면 발산, 너무 작으면 느림
- 배치 크기(batch size): 한 번에 몇 개 샘플을 보고 업데이트할지 결정
- 에폭(epoch): 전체 데이터를 몇 번 반복해서 볼지 결정
- 정규화(regularization): 과적합을 줄이기 위한 장치  


보조 수식:
- 선형층: z = W x + b
- 활성화: h = φ(z)
- 손실: L(y, ŷ)
- 파라미터 업데이트: w := w - η * ∂L/∂w
- 변수 의미:
  - η: 학습률
  - ∂L/∂w: 손실을 줄이기 위한 기울기
### A) 순전파와 손실: 왜 틀렸는지를 먼저 본다
- 상황:
  - 입력 x가 들어오면 MLP가 ŷ를 예측합니다.
  - 실제값 y=1, 예측값 ŷ=0.7이라고 가정합니다.
- 계산:
  - 오차 = y - ŷ = 1 - 0.7 = 0.3
  - MSE 한 샘플 기준 제곱 오차 = 0.3^2 = 0.09
- 해석:
  - 틀린 정도를 수치로 바꾸면, 그 다음에 얼마나 고쳐야 할지 판단할 수 있습니다.  
### B) 역전파: 뒤에서 앞으로 책임을 나눈다
- 상황:
  - 2층 신경망이 있다고 합시다.
  - 마지막 출력층에서 오차가 발생했습니다.
- 직관:
  - 오차는 출력층만의 문제가 아니라, 앞선 층의 계산이 누적된 결과입니다.
  - 그래서 손실의 책임을 체인 룰로 앞층까지 전달합니다.
- 해석:
  - 역전파는 "누가 얼마나 틀렸는지"를 미분으로 분배하는 작업입니다.  
### C) 학습률: 너무 크면 튀고, 너무 작으면 느리다
- 상황:
  - 현재 w=2, 기울기 = 0.5, 학습률 η=0.1 이라고 합시다.
- 계산:
  - 새 w = 2 - 0.1*0.5 = 1.95
- 다른 경우:
  - η=1.0이면 새 w = 2 - 1.0*0.5 = 1.5
- 해석:
  - 학습률이 너무 크면 한 번에 너무 많이 움직여 오히려 불안정할 수 있습니다.
  - 학습률이 너무 작으면 개선이 아주 느립니다.  
### D) 배치 크기와 에폭
- 상황:
  - 데이터 100개를 학습한다고 가정합니다.
- 미니배치 예시:
  - batch size = 10이면 10개씩 잘라 10번 업데이트합니다.
- 에폭 예시:
  - epoch = 3이면 전체 100개 데이터를 3번 반복해서 봅니다.
- 해석:
  - 배치는 한 번에 보는 양, 에폭은 전체를 몇 번 도는지입니다.  
### E) 활성화 함수가 왜 필요한가
- 상황:
  - 층마다 단순 선형 계산만 반복하면 전체가 결국 하나의 선형식처럼 됩니다.
- 직관:
  - 비선형 활성화가 있어야 층을 깊게 쌓는 의미가 생깁니다.
- 해석:
  - ReLU는 음수를 0으로 눌러 패턴을 단순화하고,
  - sigmoid는 0~1 사이로 눌러 확률 해석에 도움을 줍니다.  

- 한눈에 비교:

| 개념 | 의미 | 실무에서 보는 포인트 |
|---|---|---|
| 순전파 | 입력으로 예측 생성 | 현재 모델 성능 확인 |
| 손실 | 틀린 정도 측정 | 학습 목표 함수 |
| 역전파 | 기울기 계산 | 어떤 파라미터를 얼마나 바꿀지 결정 |
| 학습률 | 한 번에 움직이는 폭 | 안정성/속도 균형 |
| 배치 크기 | 한 번에 보는 샘플 수 | 메모리/노이즈/속도 균형 |
| 에폭 | 전체 반복 횟수 | 충분히 학습했는지 판단 |


### F) BCE(Binary Cross-Entropy)를 손으로 계산해 보기
- 상황:
  - 이진 분류에서 실제값 y=1, 예측확률 ŷ=0.8
- 계산:
  - BCE = -( y*log(ŷ) + (1-y)*log(1-ŷ) )
  - BCE = -( 1*log(0.8) + 0*log(0.2) ) = -log(0.8) ≈ 0.223
- 비교:
  - 같은 y=1인데 ŷ=0.2라면 BCE = -log(0.2) ≈ 1.609
- 해석:
  - 정답 클래스 확률을 낮게 주면 손실이 크게 증가합니다.
  - 그래서 분류에서는 BCE가 "확률 예측의 질"을 더 직접적으로 벌점화합니다.
### G) 왜 grad_out = ŷ - y 인가 (코드 연결)
- 상황:
  - 출력층 logit을 z, 확률을 ŷ = sigmoid(z), 손실을 BCE라고 둡니다.
- 핵심 결과:
  - dL/dz = ŷ - y
- 해석:
  - 출력층에서 오차 신호는 "예측확률 - 실제값"으로 단순화됩니다.
  - 그래서 통합 실습 코드의 `grad_out = y_hat - y`가 바로 이 식입니다.
