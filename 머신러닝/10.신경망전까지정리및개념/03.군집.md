## [이론] 군집화(Clustering)의 핵심 알고리즘과 수식
군집화는 정답 라벨이 없는 데이터들을 서로 비슷한 것끼리 묶는 문제입니다.
실무 예시로는 고객을 구매 패턴별로 나누거나, 뉴스 기사들을 주제별로 묶거나, 비슷한 행동을 하는 사용자를 그룹으로 분리하는 일이 있습니다.

핵심 아이디어는 간단합니다.
- 이 데이터끼리는 얼마나 비슷한가?
- 비슷한 것끼리 모이면 어떤 그룹이 자연스럽게 생기는가?
- 그 그룹의 중심은 어디인가?

학습 흐름(이 교안의 순서):
1. 군집화와 거리 개념을 먼저 이해합니다.
2. K-평균의 목적함수와 반복 구조를 이해합니다.
3. DBSCAN에서 밀도와 노이즈의 개념을 이해합니다.
4. 계층적 군집화에서 linkage와 덴드로그램을 이해합니다.
5. 용어 정리와 핵심 증명으로 개념을 고정합니다.
6. 마지막에 코드로 손계산 결과를 검산합니다.

1) 군집화의 기본 개념
- 군집화는 정답 클래스가 주어진 분류와 다릅니다.
- 분류는 "이건 스팸인가?"처럼 이미 정해진 라벨을 맞히는 문제입니다.
- 군집화는 라벨이 없고, 데이터의 구조를 먼저 찾아 그룹을 만드는 문제입니다.
- 즉 "이 데이터는 어디와 가장 비슷한가?"를 먼저 묻습니다.

2) 거리 기반 생각
- 가장 많이 쓰는 생각은 거리입니다.
- 대표 수식(유클리드 거리): d(p, q) = sqrt(Σ(p_i - q_i)^2) = ||p - q||_2
- 변수 의미:
  - p, q: 비교하는 두 데이터
  - p_i, q_i: i번째 특성값
  - Σ: 모든 특성 차이 제곱을 합침
  - sqrt: 제곱된 스케일을 원래 거리 스케일로 복원
- 거리 함수(metric)로 쓰려면 최소한 음수가 아니고, 대칭이고, 자기 자신과의 거리가 0이어야 합니다.

3) K-평균 군집화
- K-평균은 미리 군집 개수 K를 정하고 시작합니다.
- 각 점을 가장 가까운 중심에 배정한 뒤, 중심을 다시 평균으로 갱신합니다.
- 핵심은 "배정(assign) -> 중심 재계산(update)"를 반복하는 것입니다.
- K-평균의 목적함수(= inertia, SSE)는 다음과 같습니다.
  - J = Σ ||x_i - μ_{c_i}||^2
  - x_i: i번째 데이터
  - c_i: i번째 데이터가 배정된 군집 번호
  - μ_{c_i}: 배정된 군집의 중심
- 중심은 각 군집의 SSE를 최소화하는 평균입니다.
- 중심 수식: μ_k = (1/n_k) * Σ x_i
- 변수 의미:
  - μ_k: k번째 군집 중심
  - n_k: 그 군집에 속한 데이터 개수
  - x_i: 군집에 속한 각 데이터

4) 계층적 군집화
- 계층적 군집화는 처음에는 각 데이터가 하나의 군집이라고 보고 시작합니다.
- 그다음 가장 비슷한 두 군집을 합치면서 트리처럼 계층을 만듭니다.
- 결과는 나무 구조처럼 보이며, 원하는 높이에서 잘라 군집 수를 결정할 수 있습니다.
- 중요한 생각은 "처음부터 K를 딱 고정하지 않아도 된다"는 점입니다.

5) DBSCAN
- DBSCAN은 밀도 기반 군집화입니다.
- eps 반경 안에 들어오는 이웃 수가 min_samples 이상이면 그 점을 core point로 봅니다.
- core point를 따라 밀도 연결(density-connected)된 점들을 하나의 군집으로 묶습니다.
- core point 주변에 붙어 있지만 자기 이웃 수가 부족한 점은 border point, 어느 core에도 연결되지 않으면 noise로 봅니다.
- 군집 개수 K를 미리 정하지 않아도 되고, 비구형 군집과 이상치 분리에 강합니다.

군집화와 분류의 차이:
- 분류는 이미 정해진 라벨을 맞춥니다.
- 군집화는 라벨이 없어서 먼저 그룹을 찾습니다.
- 그래서 군집화는 탐색, 분류는 판정에 가깝습니다.

## [풀이] 수식에 담긴 직관적인 의미
이 파트는 코드보다 먼저 이해하는 구간입니다.
순서는 "상황 이야기 -> 수식 대입 -> 손계산 검증 -> 해석"입니다.

### A) d(p, q) = sqrt(Σ(p_i - q_i)^2) (가까운 것끼리 묶기)
- 수식 설명:
  - p, q: 두 데이터
  - p_i, q_i: 각 특성값
  - 차이를 제곱해서 더한 뒤 제곱근을 취하면 유클리드 거리가 됩니다.

- 실제 예시:
  - 상황: 고객 두 명의 행동 패턴이 비슷한지 비교합니다.
  - 고객 A = (주간 방문 2회, 평균 체류 10분)
  - 고객 B = (주간 방문 3회, 평균 체류 12분)
  - 거리 계산:
    - sqrt((2-3)^2 + (10-12)^2)
    - = sqrt(1 + 4)
    - = sqrt(5)
    - ≈ 2.236
  - 해석: 숫자가 작을수록 더 비슷합니다.

- 직관 시각화(텍스트):
```text
[입력]
고객 A = (2, 10)
고객 B = (3, 12)

      ↓

[거리 계산]
d(A, B) = sqrt((2-3)^2 + (10-12)^2)
        = sqrt(5) ≈ 2.236

      ↓

[해석]
거리 값이 작다 -> 두 고객은 비슷하다
```

- 한눈에 보는 표:

| 고객 | 주간 방문 | 평균 체류 | 거리 |
|---|---:|---:|---:|
| A | 2 | 10 | - |
| B | 3 | 12 | sqrt(5) ≈ 2.236 |

- 해석:
  - 두 고객은 방문 횟수와 체류 시간이 비슷하므로 같은 그룹에 묶일 가능성이 높습니다.

### B) K-평균 군집화 (중심을 잡고, 다시 모으기)
- 수식 설명:
  - K-평균은 목적함수 J = Σ ||x_i - μ_{c_i}||^2 (제곱거리합)을 최소화합니다.
  - 중심 μ_k는 군집 안 데이터들의 평균이며, 이 값이 해당 군집 SSE를 가장 작게 만듭니다.

- 왜 하필 평균인가:
  - K-평균은 "거리의 합"이 아니라 "거리의 제곱합(SSE)"을 줄입니다.
  - 한 군집 C에서 f(μ)=Σ_{x∈C} ||x-μ||^2 를 최소화하면 해가 산술평균 μ=(1/|C|)Σx가 됩니다.
  - 그래서 update 단계는 항상 "군집 평균으로 중심 갱신"이 됩니다.

- 아주 작은 예시(1차원, 제곱거리 기준):
  - 한 군집에 1, 2, 8이라는 점이 있다고 합시다.
  - 중심을 2로 두면 SSE는 (1-2)^2 + (2-2)^2 + (8-2)^2 = 1 + 0 + 36 = 37입니다.
  - 중심을 3으로 두면 SSE는 (1-3)^2 + (2-3)^2 + (8-3)^2 = 4 + 1 + 25 = 30입니다.
  - 중심을 4로 두면 SSE는 (1-4)^2 + (2-4)^2 + (8-4)^2 = 9 + 4 + 16 = 29입니다.
  - 평균은 (1+2+8)/3 = 3.667이며, 이때 SSE는 약 28.667로 (2, 3, 4)보다 더 작습니다.
  - 참고: 절대거리합 Σ|x-μ|을 최소화하는 값은 평균이 아니라 중앙값입니다.

- 2차원 예시:
  - 점이 (1,1), (2,1), (8,8), (9,8)처럼 두 무리로 나뉘어 있으면,
    각 무리의 중심은 그 무리의 평균 좌표가 됩니다.
  - 왼쪽 무리의 평균은 (1.5, 1), 오른쪽 무리의 평균은 (8.5, 8)입니다.
  - 중심이 데이터의 가운데에 놓일수록 양쪽 점들과의 제곱거리합(SSE)이 균형 있게 줄어듭니다.

- 실제 예시:
  - 상황: 편의점 고객을 두 그룹으로 나눕니다.
  - 데이터가 다음과 같다고 가정합니다.
    - P1=(1,1)
    - P2=(2,1)
    - P3=(8,8)
    - P4=(9,8)
  - K=2라고 두고 시작합니다.
  - 초기 중심을 대충 μ1=(1,1), μ2=(8,8)로 잡으면,
    - P1, P2는 μ1 쪽이 더 가깝고
    - P3, P4는 μ2 쪽이 더 가깝습니다.
  - 다시 중심을 평균으로 계산하면,
    - μ1 = ((1+2)/2, (1+1)/2) = (1.5, 1)
    - μ2 = ((8+9)/2, (8+8)/2) = (8.5, 8)
  - 결론: 비슷한 점끼리 모이고, 그 평균이 새 중심이 됩니다.

- 한 줄 직관:
  - 평균은 군집 안 점들을 가장 무리 없이 대표하는 위치입니다.
  - 그래서 K-평균의 중심은 계속 평균 쪽으로 이동합니다.

- K-평균의 핵심 흐름:
  1. 중심을 정합니다.
  2. 각 점을 가장 가까운 중심에 보냅니다.
  3. 각 군집의 평균으로 중심을 다시 계산합니다.
  4. 목적함수 J가 더 이상 줄지 않으면 종료합니다(지역 최적해).

- 직관 시각화(텍스트):
```text
[1] 초기화
초기 중심 선택: μ1, μ2, ..., μK

      ↓

[2] 배정(Assignment)
각 점을 가장 가까운 중심에 할당

      ↓

[3] 갱신(Update)
각 군집의 평균으로 중심 재계산

      ↓

[4] 종료 판정
목적함수 J가 더 이상 줄지 않으면 종료
줄어들면 [2]로 돌아가 반복
```

- 한눈에 보는 표:

| 점 | 좌표 | 가장 가까운 중심 |
|---|---|---|
| P1 | (1,1) | μ1 |
| P2 | (2,1) | μ1 |
| P3 | (8,8) | μ2 |
| P4 | (9,8) | μ2 |

- 해석:
  - K-평균은 "중심에 더 가까운 쪽으로 계속 끌려가는 구조"입니다.
  - 중심은 군집의 대표 얼굴처럼 동작합니다.
  - 더 정확히는, 대표 얼굴이 군집의 평균 위치가 되도록 스스로 조정됩니다.

### C) 계층적 군집화 (가까운 것부터 차곡차곡 합치기)
- 수식/개념 설명:
  - 각 점을 처음엔 하나의 군집으로 봅니다.
  - 가장 가까운 두 군집을 합칩니다.
  - 이때 "가까움"의 정의(linkage)를 먼저 정합니다.
    - single linkage: 두 군집 점들 사이 최소 거리
    - complete linkage: 두 군집 점들 사이 최대 거리
    - average linkage: 두 군집 점들 사이 평균 거리
    - Ward linkage: 병합 후 SSE 증가량이 최소인 쌍 선택
  - 이 과정을 반복하면 트리 구조가 만들어집니다.

- 실제 예시:
  - 상황: 책 4권을 주제별로 묶습니다.
  - 책 A와 B는 인공지능 입문서, C와 D는 수학 입문서라고 가정합니다.
  - A와 B의 거리가 가장 가깝고, C와 D도 가깝다면,
    - 먼저 A-B가 묶이고
    - 그다음 C-D가 묶입니다.
  - 마지막에는 (A,B)와 (C,D) 두 덩어리가 멀리 떨어진 상태로 남습니다.

- 해석:
  - 계층적 군집화는 한 번에 군집 수를 정하기보다, 묶이는 과정을 기록합니다.
  - 그래서 결과를 나무(덴드로그램)처럼 보고, 원하는 높이에서 잘라 군집 수를 결정할 수 있습니다.

- 직관 시각화(텍스트):
```text
시작:   {A}   {B}   {C}   {D}
          ↓
1단계:  {A,B} {C}   {D}
          ↓
2단계:  {A,B} {C,D}
          ↓
3단계:  {A,B,C,D}

절단(cut) 예시:
- 2단계 높이에서 자르면 -> 군집 2개: {A,B}, {C,D}
```

- 한눈에 보는 표:

| 단계 | 묶임 | 의미 |
|---|---|---|
| 1 | A + B | 가장 가까운 두 데이터 결합 |
| 2 | C + D | 다음으로 가까운 두 데이터 결합 |
| 3 | (A,B) + (C,D) | 전체 구조 완성 |

- 실무 감각:
  - K-평균은 중심이 분명해서 빠르게 쓸 수 있습니다.
  - 계층적 군집화는 구조를 자세히 보고 싶을 때 유리합니다.
  - "처음부터 몇 개 군집인지 모르겠다"면 계층적 군집화가 더 자연스럽습니다.

### D) DBSCAN (밀도 높은 곳만 군집으로)
- 수식/개념 설명:
  - eps: 한 점 주변을 어디까지 이웃으로 볼지 정하는 반경입니다.
  - N_eps(p) = {q | d(p, q) <= eps}: p의 eps-이웃 집합입니다.
  - |N_eps(p)| >= min_samples 이면 p는 core point가 됩니다.
  - core point에서 시작해 core에 도달 가능한 점들을 계속 확장하면 하나의 군집이 됩니다.
  - 밀도가 부족해 어느 군집에도 연결되지 못한 점은 noise로 남습니다.

- 실제 예시:
  - 상황: 지도 위의 방문 좌표를 보고 "사람이 몰린 구역"만 찾습니다.
  - 점이 다음과 같다고 가정합니다.
    - P1=(0,0)
    - P2=(0.2,0.1)
    - P3=(-0.1,0.1)
    - P4=(5,5)
    - P5=(5.1,5.1)
    - P6=(4.9,5.0)
    - P7=(8,1)
  - eps=0.35, min_samples=3으로 두면:
    - P1, P2, P3는 서로 충분히 가까워 cluster 1이 됩니다.
    - P4, P5, P6도 서로 가까워 cluster 2가 됩니다.
    - P7은 어느 군집에도 붙지 못해 noise가 됩니다.

- 해석:
  - DBSCAN은 중심을 계산하지 않습니다.
  - 대신 "밀도가 높은 곳"을 먼저 찾고, 그 밀도 덩어리를 군집으로 인정합니다.
  - 그래서 구불구불한 모양이나 타원형, 비정형 군집에도 잘 맞습니다.

- 직관 시각화(텍스트):
```text
[점 배치]
P1  P2  P3         P4  P5  P6              P7

       ↓ eps와 min_samples로 밀도 판정

[판정]
P1,P2,P3 -> cluster 1
P4,P5,P6 -> cluster 2
P7 -> noise
```

- 한눈에 보는 표:

| 점 | 좌표 | eps 이웃 수 | 판정 |
|---|---|---:|---|
| P1 | (0,0) | 3 | core -> cluster 1 |
| P2 | (0.2,0.1) | 3 | core -> cluster 1 |
| P3 | (-0.1,0.1) | 3 | core -> cluster 1 |
| P4 | (5,5) | 3 | core -> cluster 2 |
| P5 | (5.1,5.1) | 3 | core -> cluster 2 |
| P6 | (4.9,5.0) | 3 | core -> cluster 2 |
| P7 | (8,1) | 1 | noise |

- 실무 감각:
  - DBSCAN은 군집 개수를 미리 몰라도 됩니다.
  - 이상치(noise)를 따로 분리하기 좋습니다.
  - 대신 eps와 min_samples 설정이 결과를 크게 바꿉니다.

실무적으로는:
- 군집화의 개념: 라벨 없는 데이터의 구조를 찾는 첫 단계
- K-평균 군집화: 대용량 데이터의 빠른 그룹화에 강함
- DBSCAN: 노이즈가 섞인 비정형 군집 탐지에 강함
- 계층적 군집화: 군집 구조를 트리로 해석하는 데 강함

## [용어 정리] 빠짐없이 이해하기
| 용어 | 설명 | 실무 포인트 |
|---|---|---|
| 샘플(sample, point) | 하나의 관측 데이터 x_i | 고객 1명, 문서 1개처럼 군집화의 기본 단위 |
| 특성(feature) | 샘플을 설명하는 변수 축 p_i | 스케일 차이가 크면 거리 왜곡이 발생 |
| 거리(metric) | 두 샘플 간 차이를 수치화한 함수 d(p,q) | 어떤 거리를 쓰는지에 따라 군집 결과가 달라짐 |
| 유클리드 거리 | d(p,q)=sqrt(Σ(p_i-q_i)^2) | 연속형/등방성 데이터에서 기본 선택 |
| 군집(cluster) | 서로 유사한 샘플의 집합 | 군집 내부는 가깝고 군집 간은 멀어야 해석이 쉬움 |
| 중심(centroid) | 군집의 대표점 μ_k | K-평균에서는 산술평균으로 계산 |
| K | 만들 군집 개수 | 너무 작으면 과소분할, 너무 크면 과대분할 |
| 배정(assignment) | 각 샘플을 가장 가까운 중심에 할당 | K-평균의 E-step 역할 |
| 갱신(update) | 배정 결과로 중심 재계산 | K-평균의 M-step 역할 |
| SSE / inertia | Σ||x_i-μ_{c_i}||^2 목적함수 값 | 작을수록 군집 내부 응집도가 큼 |
| 수렴(convergence) | 반복해도 배정/중심이 더 바뀌지 않는 상태 | K-평균은 전역 최적이 아닌 지역 최적에 수렴 |
| 초기화(initialization) | 시작 중심을 잡는 절차 | k-means++로 초기화 민감도를 완화 |
| eps | DBSCAN에서 이웃을 판단하는 반경 | 반경이 너무 작으면 군집이 잘게 쪼개짐 |
| min_samples | DBSCAN에서 core point를 만들 최소 이웃 수 | 너무 크면 군집이 noise로 빠질 수 있음 |
| core point | eps 이웃이 min_samples 이상인 점 | DBSCAN 군집 확장의 시작점 |
| border point | core point 주변에 있지만 자기 이웃은 부족한 점 | 군집 경계에 붙는 점 |
| noise | 어떤 군집에도 속하지 않는 점 | 이상치 탐지에 활용 가능 |
| linkage | 군집 간 거리 정의 방식 | single/complete/average/Ward 선택이 결과를 좌우 |
| 덴드로그램(dendrogram) | 병합 과정을 트리로 표현한 그림 | 절단 높이(cut height)로 군집 수를 사후 결정 |
| cut height | 덴드로그램을 자르는 기준 높이 | 높이를 낮추면 군집 수 증가, 높이면 군집 수 감소 |

## [증명] 핵심 용어가 수식에서 왜 맞는가
### 1) 유클리드 거리는 왜 "거리"로 쓸 수 있는가
- d(p,q)=||p-q||_2 는 항상 0 이상입니다(제곱합의 제곱근).
- d(p,q)=d(q,p) 입니다(차이 제곱은 순서와 무관).
- d(p,p)=0 이고 p≠q면 d(p,q)>0 입니다.
- 삼각부등식 d(p,r)≤d(p,q)+d(q,r)은 L2 노름 성질로 보장됩니다.
- 따라서 유클리드 거리는 군집화의 거리 함수(metric) 조건을 만족합니다.

### 2) K-평균 중심이 평균이 되는 이유(핵심 미분 증명)
- 한 군집 C={x_1,...,x_n}가 고정되었다고 두고 f(μ)=Σ||x_j-μ||^2 를 최소화합니다.
- 전개하면 f(μ)=Σ(x_j^T x_j) - 2μ^TΣx_j + nμ^Tμ 입니다.
- μ로 미분하면 ∇f(μ)= -2Σx_j + 2nμ 입니다.
- ∇f(μ)=0 이 되려면 μ=(1/n)Σx_j 입니다.
- 즉 중심 갱신식 μ_k=(1/n_k)Σx_i 는 "정의"가 아니라 SSE 최소화의 해입니다.

### 3) K-평균 반복이 왜 수렴하는가(단조 감소)
- 배정 단계: 각 x_i를 가장 가까운 μ_k에 보내면 해당 항 ||x_i-μ_{c_i}||^2 가 최소가 되어 J가 증가하지 않습니다.
- 갱신 단계: 각 군집에서 평균으로 μ_k를 바꾸면 (위 증명에 의해) 군집별 SSE가 최소가 되어 J가 증가하지 않습니다.
- 따라서 한 번의 assign->update 이후 J는 단조 비증가합니다.
- 가능한 배정 경우의 수는 유한(K^n)하므로 결국 배정이 반복되지 않는 지점에서 멈춥니다(지역 최적해).

### 4) 계층적 군집화가 왜 "계층" 구조가 되는가
- 시작 시점에는 n개 샘플이 n개 군집을 이룹니다(분할).
- 매 단계는 두 군집을 합치기만 하고 쪼개지 않습니다.
- 따라서 단계 t+1의 각 군집은 단계 t 군집들의 합집합으로 표현됩니다(포함 관계 유지).
- 이 포함 관계가 누적되면 중첩(nested) 구조가 되고, 이것이 덴드로그램입니다.
- 덴드로그램을 임의 높이에서 자르면 항상 서로 겹치지 않는 유효한 군집 분할을 얻습니다.

### 5) DBSCAN이 왜 노이즈와 비정형 군집에 강한가
- DBSCAN은 중심점이 아니라 지역 밀도만 보고 군집을 확장합니다.
- eps 안의 이웃 수가 min_samples 이상인 core point만 군집 확장을 시작할 수 있습니다.
- 밀도가 이어지지 않으면 확장이 멈추므로, 성긴 점들은 noise로 남습니다.
- 이 방식은 원형 중심을 가정하지 않기 때문에, 길게 늘어진 군집이나 굴곡진 군집도 그대로 잡아낼 수 있습니다.

### 1) 거리 계산과 K-평균 중심 갱신 직접 구현
```python

from math import sqrt

points = [(1, 1), (2, 1), (8, 8), (9, 8)]
centers = [(1, 1), (8, 8)]

def euclidean(p, q):
    #### [ 빈칸: 이 부분을 완성하세요 ]
    return sqrt(sum((pi - qi) ** 2 for pi, qi in zip(p, q)))

assignments = []
for point in points:
    distances = [euclidean(point, center) for center in centers]
    #### [ 빈칸: 이 부분을 완성하세요 ]
    assignments.append(distances.index(min(distances)))

cluster_0 = [point for point, label in zip(points, assignments) if label == 0]
cluster_1 = [point for point, label in zip(points, assignments) if label == 1]

def mean_center(cluster):
    #### [ 빈칸: 이 부분을 완성하세요 ]
    x_mean = sum(point[0] for point in cluster) / len(cluster)
    y_mean = sum(point[1] for point in cluster) / len(cluster)
    return (x_mean, y_mean)

new_centers = [mean_center(cluster_0), mean_center(cluster_1)]
print("assignments:", assignments)
print("new_centers:", new_centers)
```

### 2) DBSCAN의 밀도 판정 직접 구현
```python
from math import sqrt

points = [(0, 0), (0.2, 0.1), (-0.1, 0.1), (5, 5), (5.1, 5.1), (4.9, 5.0), (8, 1)]
eps = 0.35
min_samples = 3

def euclidean(p, q):
    #### [ 빈칸: 이 부분을 완성하세요 ]
    return sqrt(sum((pi - qi) ** 2 for pi, qi in zip(p, q)))

def region_query(idx):
    #### [ 빈칸: 이 부분을 완성하세요 ]
    return [j for j, point in enumerate(points) if euclidean(points[idx], point) <= eps]

labels = [-1] * len(points)  # -1: unvisited, -2: noise
cluster_id = 0

for i in range(len(points)):
    if labels[i] != -1:
        continue

    neighbors = region_query(i)
    if len(neighbors) < min_samples:
        labels[i] = -2
        continue

    labels[i] = cluster_id
    seeds = [j for j in neighbors if j != i]

    while seeds:
        j = seeds.pop()
        if labels[j] == -2:
            labels[j] = cluster_id
        if labels[j] != -1:
            continue

        labels[j] = cluster_id
        j_neighbors = region_query(j)
        if len(j_neighbors) >= min_samples:
            for n in j_neighbors:
                if n not in seeds:
                    seeds.append(n)

    cluster_id += 1

print("dbscan labels:", labels)
```

## [결과] 이 알고리즘의 한계와 실무 적용 포인트
- 군집화 한계:
  - 정답 라벨이 없어서 결과 해석이 항상 쉬운 것은 아닙니다.
  - 거리 척도와 특성 스케일에 민감합니다.
- K-평균 한계:
  - K를 미리 정해야 합니다.
  - 초기 중심에 따라 결과가 달라질 수 있습니다.
  - 원형에 가까운 군집에 잘 맞고, 길쭉하거나 복잡한 군집에는 약할 수 있습니다.
- DBSCAN 한계:
  - eps와 min_samples 선택에 민감합니다.
  - 밀도 차이가 큰 데이터에서는 한 군집이 잘게 쪼개지거나 합쳐질 수 있습니다.
  - 군집 내부 밀도가 매우 다르면 해석이 어려울 수 있습니다.
- 계층적 군집화 한계:
  - 데이터가 많아지면 계산이 무거워질 수 있습니다.
  - 한 번 병합하면 되돌리기 어렵습니다.

실무 적용 포인트:
- 라벨이 없을 때 먼저 데이터 구조를 파악하는 데 군집화가 유용합니다.
- K-평균은 고객 세분화, 이미지 색상 압축, 빠른 그룹화에 자주 쓰입니다.
- DBSCAN은 지도 데이터, 이상치 탐지, 복잡한 형태의 군집 찾기에 자주 쓰입니다.
- 계층적 군집화는 그룹 간 관계를 트리로 해석해야 할 때 유리합니다.
