# SGD (Stochastic Gradient Descent) 최적화 알고리즘 분석

---

### 전문 용어 사전 설명
본격적인 분석에 앞서, 핵심 용어들을 정의합니다.

*   **배치 (Batch):** 모델 업데이트를 위해 한 번에 사용하는 데이터의 묶음입니다. 전체 데이터를 다 쓰면 'Full Batch'라고 합니다.
*   **에포크 (Epoch):** 전체 학습 데이터셋을 한 번 모두 훑는 과정을 의미합니다.
*   **지역 최솟값 (Local Minimum):** 전체에서 가장 낮은 지점은 아니지만, 주변보다 낮아서 알고리즘이 갇히기 쉬운 웅덩이 같은 지점입니다.
*   **노이즈 (Noise):** 데이터 한 개만 보고 움직일 때 발생하는 일시적인 방향의 흔들림입니다. SGD에서는 이 흔들림이 지역 최솟값을 탈출하는 동력이 됩니다.

---

### 1. 정의 및 배경 설명
**SGD(Stochastic Gradient Descent)** 는 전체 데이터셋의 평균 기울기를 계산하는 대신, 무작위로 선택된 단 하나의 데이터 샘플($(i)$)에 대한 기울기만을 활용하여 매개변수($\theta$)를 즉각적으로 업데이트하는 최적화 방식입니다.

---

### 2. 경사 하강법(GD) vs 확률적 경사 하강법(SGD) 비교

두 알고리즘의 결정적 차이를 분석합니다.

| 비교 항목 | 경사 하강법 (GD) | 확률적 경사 하강법 (SGD) |
| :--- | :--- | :--- |
| **데이터 사용량** | **전체 데이터셋** (All Data) | **무작위 샘플 1개** (One Sample) |
| **계산 속도** | 느림 (데이터가 많을수록 연산량 폭증) | **매우 빠름** (실시간 업데이트 가능) |
| **학습 경로** | 안정적이고 매끄러운 직선 형태 | **지그재그이며 변동이 심함 (노이즈)** |
| **수렴 특성** | 지역 최솟값(Local Minimum)에 갇힘 | 노이즈 덕분에 **지역 최솟값 탈출 유리** |

#### [수식적 차이점]
*   **GD (Batch Gradient Descent)**: 전체 $m$개 데이터의 평균 기울기를 구함
    $$\theta_{t+1} = \theta_t - \eta \cdot \frac{1}{m} \sum_{i=1}^{m} \nabla_{\theta} L(\theta_t, x^{(i)}, y^{(i)})$$
*   **SGD (Stochastic Gradient Descent)**: 단일 샘플 $i$의 기울기만 즉시 적용
    $$\theta_{t+1} = \theta_t - \eta \nabla_{\theta} L(\theta_t, x^{(i)}, y^{(i)})$$

---

### 3. 수식 유도 과정 (데이터 대입 증명)

초기값 $\theta=2.0$, 학습률 $\eta=0.1$, 샘플 데이터 $x^{(i)}=1, y^{(i)}=1$ 상황에서 증명을 수행합니다.

1.  **손실 함수 설정 (MSE 기준)**:
    $$L(\theta, x^{(i)}, y^{(i)}) = \frac{1}{2}(\theta \cdot x^{(i)} - y^{(i)})^2$$

2.  **기울기 계산 (미분)**:
    $\theta$에 대해 손실 함수를 미분하여 기울기 공식을 얻습니다.
    $$\nabla_{\theta} L = (\theta \cdot x^{(i)} - y^{(i)}) \cdot x^{(i)}$$
    *   *데이터 대입*: $(2.0 \cdot 1 - 1) \cdot 1 = \mathbf{1.0}$

3.  **최속 강하 적용**:
    기울기의 반대 방향($-1.0$)에 학습률($0.1$)을 곱해 이동량을 결정합니다.
    $$\text{이동량} = -0.1 \cdot 1.0 = \mathbf{-0.1}$$

4.  **최종 업데이트 완성**:
    $$\theta_{t+1} = 2.0 + (-0.1) = \mathbf{1.9}$$

---

### 4. 수치적 증명 (Dry Run)

| 단계 | 계산식 | 결과 | 비고 |
| :--- | :--- | :--- | :--- |
| **Step 1** | 오차 측정: $2.0 \cdot 1 - 1$ | $1.0$ | 정답보다 1.0만큼 큼 |
| **Step 2** | 기울기 산출: $1.0 \cdot 1$ | $1.0$ | 현재 위치에서의 기울기 값 |
| **Step 3** | 최종 갱신: $2.0 - (0.1 \cdot 1.0)$ | **1.9** | 정답 방향으로 가중치 수정 |

---

### 5. 코드 구현 및 데이터 증명
```python
# GD와 SGD의 차이를 보여주는 간단한 시뮬레이션
theta = 2.0        # 초기 가중치
x_i, y_i = 1.0, 1.0 # 무작위 샘플 (SGD는 이것만 사용)
eta = 0.1          # 학습률

# 1. SGD 방식의 기울기 계산 (단일 샘플 사용)
grad_sgd = (theta * x_i - y_i) * x_i

# 2. 매개변수 업데이트
theta_next = theta - eta * grad_sgd

print(f"SGD 계산 기울기: {grad_sgd}")
print(f"업데이트된 가중치: {theta_next}") # 결과: 1.9
```

**[최종 증명 결론]**
분석 결과, GD는 정교하지만 연산 비용이 너무 커서 실전 딥러닝에는 부적합한 반면, **SGD는 단일 샘플의 정보를 빠르게 반영하여 연산 효율을 극대화**합니다. 또한 SGD 특유의 지그재그 움직임(노이즈)은 복잡한 손실 함수 평면에서 더 나은 최적점을 찾는 데 기여함을 수식과 논리로 증명하였습니다.
