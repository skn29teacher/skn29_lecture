# 6. 완성: 엔드 투 엔드 신경망 훈련 파이프라인

지금까지 모듈 1~5에서 퍼셉트론, 다층 구조, 손실 함수, 경사하강법, 역전파의 원리를 낱낱이 해부했습니다. 이번 마지막 모듈에서는 그 모든 퍼즐 조각을 하나로 조립하여, **데이터 준비부터 학습, 평가까지 현업에서 사용하는 완전한 파이토치 훈련 파이프라인**을 구축합니다.

## 6.1. 파이토치 학습 파이프라인 전체 구조

```
┌───────────────────────────────────────────────────────────────────────┐
│                    PyTorch 학습 파이프라인 전체 흐름                     │
├───────────────────────────────────────────────────────────────────────┤
│                                                                       │
│  1단계: 데이터 준비                                                    │
│     └── Dataset → DataLoader (미니배치 자동 분할)                      │
│                                                                       │
│  2단계: 모델 정의                                                      │
│     └── nn.Module 클래스 상속 → __init__ + forward 메서드               │
│                                                                       │
│  3단계: 손실 함수 선택                                                  │
│     └── nn.MSELoss (회귀) / nn.CrossEntropyLoss (분류)                 │
│                                                                       │
│  4단계: 옵티마이저(최적화기) 선택                                       │
│     └── optim.SGD / optim.Adam 등                                     │
│                                                                       │
│  5단계: 학습 루프 (에폭 반복)                                           │
│     ┌──────────────────────────────────────────────┐                  │
│     │  for epoch in range(에폭수):                  │                  │
│     │    for batch in DataLoader:                   │                  │
│     │      ① Forward:  예측 = model(입력)           │                  │
│     │      ② Loss:     손실 = criterion(예측, 정답)  │                  │
│     │      ③ Zero:     optimizer.zero_grad()       │                  │
│     │      ④ Backward: loss.backward()             │                  │
│     │      ⑤ Step:     optimizer.step()            │                  │
│     └──────────────────────────────────────────────┘                  │
│                                                                       │
│  6단계: 평가                                                           │
│     └── with torch.no_grad(): 정확도(Accuracy) 측정                    │
│                                                                       │
└───────────────────────────────────────────────────────────────────────┘
```

## 6.2. 핵심 구성 요소 설명 (현업 기준으로 단계별)

### 6.2.1 `nn.Module`: 신경망의 설계도
파이토치에서 모든 신경망은 `nn.Module`을 상속받아 만듭니다.

```python
class MyNetwork(nn.Module):
    def __init__(self):
        super().__init__()
        # 여기에 레이어(부품) 선언
        self.layer1 = nn.Linear(입력크기, 은닉크기)
        self.layer2 = nn.Linear(은닉크기, 출력크기)
    
    def forward(self, x):
        # 여기에 순전파 흐름 작성 (모듈 2에서 배운 그 흐름!)
        x = torch.relu(self.layer1(x))  # 은닉층
        x = self.layer2(x)              # 출력층
        return x
```

**왜 `__init__`과 `forward`를 나누나?**

1. `__init__`: 학습해야 할 파라미터(레이어)를 등록하는 단계
2. `forward`: 입력이 들어왔을 때 계산 흐름을 실행하는 단계

파이토치는 `forward()` 계산 그래프를 기준으로 `backward()`에서 미분 경로를 자동 생성합니다.

> [!TIP]
> `forward()`는 "순전파 설계도", `backward()`는 "모듈 5에서 유도한 체인룰 자동 실행기"라고 이해하면 정확합니다.

### 6.2.2 `Dataset`/`DataLoader`: 미니배치 자동 관리자
전체 데이터를 한 번에 넣으면 메모리/일반화 측면에서 비효율적입니다.  
`DataLoader`는 데이터를 지정한 배치 크기로 나눠 순차 공급합니다.

| 용어 | 의미 |
|:---:|:---|
| **Epoch (에폭)** | 전체 데이터를 한 번 완주하는 사이클 |
| **Batch Size** | 한 번에 공부할 데이터 묶음 크기 (보통 16, 32, 64) |
| **Iteration** | 한 에폭에서 미니배치 1개를 처리하는 1회 학습 스텝 |

### 6.2.3 손실 함수(`criterion`)는 무엇을 기준으로 고르나?

1. **회귀(숫자 예측)**: `nn.MSELoss()`  
   - 왜: 예측값과 정답값의 거리(제곱 오차)를 직접 최소화
2. **이진 분류**: `nn.BCEWithLogitsLoss()`  
   - 왜: 시그모이드+로그손실을 수치 안정적으로 묶어 제공
3. **다중 분류(클래스 3개 이상)**: `nn.CrossEntropyLoss()`  
   - 왜: 소프트맥스 기반 분류 손실을 표준 형태로 제공

### 6.2.4 옵티마이저(`optimizer`)는 무엇이 다른가?

1. `SGD`: 가장 기본. 단순하고 해석이 쉬움
2. `Adam`: 학습률을 파라미터별로 자동 조절해 초반 수렴이 빠른 편

입문/교육에서는 SGD로 원리를 익히고, 실무에서는 Adam을 자주 사용합니다.

### 6.2.5 학습 루프 5단계의 의미 (한 줄씩 이유 포함)

| 단계 | 코드 | 모듈에서 배운 것 |
|:---:|:---:|:---|
| ① Forward | `pred = model(x)` | 모듈 2: 순전파 계산 흐름 |
| ② Loss | `loss = criterion(pred, y)` | 모듈 3: 손실 함수 |
| ③ Zero Grad | `optimizer.zero_grad()` | 이전 루프의 기울기 잔여값 초기화 (누적 방지) |
| ④ Backward | `loss.backward()` | 모듈 5: 오차역전파 (연쇄 법칙) |
| ⑤ Step | `optimizer.step()` | 모듈 4: 경사하강법 ($$W_{new} = W_{old} - \eta \cdot \nabla L$$) |

> [!IMPORTANT]
> **③ Zero Grad를 빠뜨리면?**
> 파이토치는 기울기를 호출할 때마다 누적(accumulate)합니다. zero_grad()를 호출하지 않으면 이전 배치의 기울기가 현재 배치에 합산되어 엉뚱한 방향으로 학습합니다. 반드시 backward() 전에 zero_grad()를 호출해야 합니다.

아래처럼 읽으면 가장 안전합니다.

1. **Forward**: 현재 파라미터로 예측 계산
2. **Loss**: 예측이 얼마나 틀렸는지 수치화
3. **Zero Grad**: 이전 배치 기울기 초기화
4. **Backward**: 현재 배치 기준 기울기 계산
5. **Step**: 계산된 기울기로 파라미터 업데이트

순서가 바뀌면 학습이 잘못되거나 멈춥니다.

## 6.3. 코드 연계 실습
세트로 제공되는 `module_6_pipeline.ipynb`에서 다음을 실행합니다.

### 6.3.1 XOR 실습: End-to-End로 연결해서 보기

1. 데이터 준비: XOR 입력/정답 텐서 생성
2. 모델 정의: 2층 MLP 구성 (`Linear-ReLU-Linear`)
3. 손실/옵티마이저 설정: `BCEWithLogitsLoss` + `Adam`(또는 SGD)
4. 학습 루프 실행: 5단계(Forward→Loss→Zero→Backward→Step) 반복
5. 평가: 임계값(예: 0.5) 기준으로 0/1 분류 확인

핵심은 "모듈 2(순전파), 4(업데이트), 5(역전파)"가 한 루프로 결합된다는 점입니다.

### 6.3.2 Iris 실습: 미니배치 분류 파이프라인

1. 데이터 전처리: 특성/레이블 분리, train/valid 분할
2. `Dataset` 생성 후 `DataLoader` 구성 (`shuffle=True`)
3. 모델 정의: 입력 4차원, 출력 3클래스
4. 손실 함수: `CrossEntropyLoss` (정답은 클래스 인덱스)
5. 학습 + 검증 루프: 에폭마다 손실/정확도 기록

### 6.3.3 그래프 해석 기준

1. 훈련 손실이 내려가면: 학습이 진행 중
2. 훈련만 내려가고 검증이 정체/악화되면: 과적합 신호
3. 둘 다 거의 안 변하면: 학습률, 모델 크기, 입력 스케일 점검
