## 3. 🛠️ 데이터 전처리 (가장 중요한 챕터!) 🛠️

우리가 받은 데이터는 숫자, 텍스트가 섞여 있고 어떤 곳은 빈칸(결측치)으로 되어 있습니다. 이상치가 섞여있거나 단위가 제각각이기도 합니다. 모델은 가공되지 않은 순생 데이터와 이상치에 매우 취약하기 때문에 우리는 이 데이터를 오븐에 굽기 좋도록 깨끗한 숫자 배열로 정제해 주어야 합니다. 이것을 **전처리(Preprocessing)**라고 부릅니다.

### 3.1 비정상 데이터 필터링: 이상치(Outliers) 탐지와 제거
머신러닝 알고리즘은 평균 같은 통계치에 크게 의존하기 때문에, 누군가 실수로 입력한 "방 1만 개짜리 1인 가구" 같은 아주 극단적인 이상치(Outlier) 한두 개가 모델 전체의 학습 잣대를 엉망으로 붕괴시킬 수 있습니다.
* **해결책**: 통계적 기법이나 `IsolationForest` 같은 전용 탐지 알고리즘을 사용하여 정상 범주를 벗어난 패턴을 가진 불청객 데이터를 학습 전에 색출하여 아예 삭제해 버립니다. 쓰레기를 넣으면 정답도 쓰레기가 나온다(GIGO)는 원칙에 대응하기 위한 가장 중요한 첫 단추입니다.

### 3.2 꼬리가 긴 데이터의 함정: 로그(Log) 변환의 마법
주택 데이터의 '수입(Income)'이나 '인구수(Population)' 등의 히스토그램을 그려보면 가운데가 볼록한 종 모양이 아니라 한쪽으로 꼬리가 매우 길게 늘어진 비대칭 모양(Heavy-tailed)을 띠는 경우가 많습니다. 대부분의 알고리즘은 **가운데가 두툼한 정규분포 형태**를 가장 좋아하며, 꼬리가 길면 일부 소수 데이터에 과도한 집착을 하게 되어 학습 효율이 극도로 떨어집니다.
* **해결책 (로그 변환)**: 값들에 `np.log1p` (자연로그 함수)를 씌워주면, 저 멀리 퍼진 꼬리 값들의 스케일이 수학적으로 압축되면서 마법처럼 데이터가 예쁜 종 모양(정규분포)으로 뭉치기 시작합니다! (주의: 데이터에 값 0이 있으면 $\log(0)$ 연산 시 무한대 에러가 나므로, 실무진들은 데이터 전체에 안전하게 1을 먼저 더하고 로그를 씌워주는 `log1p`를 표준 규격으로 씁니다.)

### 3.3 수치형 데이터 기본 정제 (결측치와 스케일링)

> [!WARNING] **결측치는 어떻게 할까? (Imputation)**
> 1. 해당 구역(행)을 아예 삭제한다.
> 2. 해당 빈칸이 있는 특성(열) 전체를 지워버린다.
> 3. **어떤 값(0, 평균, 중간값 등)으로 채워 넣는다.**  
> ⇒ Scikit-Learn의 `SimpleImputer`를 사용하면 훈련 데이터의 중간값을 계산해 모든 빈칸을 쉽게 덮어씌울 수 있습니다!

> [!WARNING] **단위가 다른데 어떡하나요? (Scaling)**
> 방 개수는 1~10 단위이고, 중간 소득은 10000~50000 단위이면, 머신러닝 모델은 무조건 단위가 큰 특성을 더 중요하게 생각하는 치명적 멍청함이 있습니다.
> 
> * **StandardScaler (표준화)**: 모든 숫자를 평균 0, 분산 1로 줄 바꿈을 시킵니다. 알고리즘이 특정 변수에 압도되지 않게 막아줍니다.

### 3.4 범주형 데이터 인코딩 (Categorical Pipeline)

바다와 가까우면 `NEAR OCEAN`, 섬이면 `ISLAND` 처럼 글자로 된 데이터는 그냥 넣으면 에러가 납니다.

* **OrdinalEncoder**: "나쁘다=0", "보통=1", "좋다=2" 처럼 순서가 있는 변수에 적합합니다.
* **OneHotEncoder**: 하지만 `NEAR OCEAN`과 `ISLAND`는 순서 우열이 없습니다. 이를 0과 1, 2로 변환하면 멍청한 알고리즘은 "아, ISLAND(2)가 NEAR OCEAN(1)보다 크고 두 배 더 중요하구나!" 라고 착각합니다. 그래서 카테고리별로 0/1 스위치 열을 새로 만들어주는 워터게이트(Water Gate) 방식인 **원-핫 인코딩**을 사용합니다.

> [!TIP] **나만의 변환기도 만들 수 있다? (Custom Transformer)**
> 사이킷런이 제공하는 것에 그치지 않고, `BaseEstimator`, `TransformerMixin`을 상속받아 나만의 변환기를 선언하면, `fit()`과 `transform()` 메서드가 포함되어 사이킷런의 공식 파이프라인에 조립할 수 있습니다.

### 3.5 종합 파이프라인 조립 (`Pipeline` & `ColumnTransformer`)
이 복잡한 과정을 `Pipeline` (컨베이어 벨트) 안에 넣어 연속으로 일어나게 하고, `ColumnTransformer`를 이용해 수치형 컨베이어 파이프와 범주형 컨베이어 파이프를 합쳐 최종 데이터 조립 시스템을 완성시킵니다. 

> [!IMPORTANT] **🚨 파이프라인의 절대 원칙: 훈련에는 `fit_transform`, 테스트에는 반드시 `transform`! 🚨**
> 머신러닝 초보자들이 가장 많이 하는 치명적인 실수가 바로 "테스트 데이터에도 `fit()`을 호출하는 것"입니다. 
> 
> 파이프라인 변환기의 동작은 두 단계로 나뉩니다.
> * **`fit` (학습)**: "데이터를 훑어보고 변환 규칙(특성의 평균값, 중간값 등)을 파악하고 기억해라!"
> * **`transform` (변환)**: "기억해둔 규칙을 잣대로 삼아서 데이터를 진짜로 변환해라!"
> 
> **왜 테스트 데이터는 `fit`을 하면 안 될까요? (데이터 누수, Data Leakage의 재앙)**
> 모의고사(훈련 데이터)를 가지고 스케일링을 한다고 가정해 봅시다. 훈련 데이터의 국어 점수 평균이 50점이었습니다. 파이프라인은 "아, 앞으로 모든 학생의 점수에서 50점을 빼서 기준을 맞춰야지" 라고 **`fit(학습)`**을 통해 못박습니다. 그리고 훈련 데이터를 50점씩 빼서 **`transform(변환)`**합니다.
> 
> 이제 실제 수능 시험(테스트 데이터)의 시간입니다. 이때 테스트 데이터에도 새롭게 `fit`을 해버리면 어떻게 될까요? 컴퓨터가 "어? 수능 치는 애들의 국어 평균은 60점이네? 그럼 이번엔 60점을 빼야지!" 하고 자기 마음대로 새로운 자를 꺼내 들어 기준을 바꿔버립니다.
> 
> 이렇게 되면 모델이 열심히 50점을 기준으로 학습해놓은 뇌의 지식과, 60점을 기준으로 들어오는 수능 데이터가 완전히 어긋나버립니다! 
> 
> **결론**: 우리가 만든 파이프라인은 훈련 데이터(`X_train`)를 만날 때는 **`fit_transform()`**을 호출하여 잣대를 만들고 변환시킵니다. 하지만 실전 테스트 데이터(`X_test`)나 새로운 데이터를 만날 때는, 절대 새로운 잣대를 만들지 말고 훈련 때 쓰던 파이프라인 그대로 **`transform()` (모델 분류기는 `predict()`)만** 호출해야 합니다!
> 
> **💡 보너스 팁: 그럼 `score()` 함수는 써도 될까요?**
> 네, **당연히 쓰셔도 됩니다!** 사이킷런 파이프라인의 **`pipeline.score(X_test, y_test)`**를 호출하면, 이 함수는 내부적으로 파이프라인의 `fit`을 절대로 건드리지 않습니다. 오직 조용히 `pipeline.predict(X_test)` (결국 내부에서 안전한 `transform`만 작동함)를 실행해서 예측값을 뽐아낸 뒤, 우리가 준 진짜 정답 `y_test`와 비교하여 알아서 채점(분야에 따라 R² 점수나 정확도 등)해주는 **아주 안전하고 편리한 단축키**입니다. 예측과 평가를 코딩 한 줄에 끝내주므로 적극 권장합니다.

## 4. 회귀 모델의 종류와 작동 원리 (심층 이론 및 증명)

앞서 깨끗하게 전처리된 파이프라인 데이터를 바탕으로, 이제 주택 가격을 맞출 **'학습 알고리즘(모델)'**을 선택해야 합니다. 
"제대로 된 이론 없이 코딩만 하면 수박 겉핥기"가 됩니다. 각 모델의 핵심 수학적 원리와 특징, 한계를 증명 코드와 함께 깊이 있게 파헤쳐 봅시다. 전공자가 아니더라도 수식의 의미를 이해할 수 있도록 쉽게 풀어 설명합니다!

### 4.1 선형 회귀 (Linear Regression)
* **이론적 배경 및 수식**: 
  선형 회귀는 데이터들 사이를 꿰뚫고 지나가는 **'단 하나의 최적의 초평면(직선)'**을 긋는 모델입니다.
  $$ \hat{y} = \theta_0 + \theta_1 x_1 + \theta_2 x_2 + \dots + \theta_n x_n $$
  여기서 $\hat{y}$는 예측된 주택 가격, $x$는 특성(방 갯수, 소득 등), $\theta$(세타)는 모델이 학습해야 할 **가중치(Weight)**입니다.
  
  그럼 컴퓨터는 어떤 $\theta$가 '최적'인지 어떻게 알까요? 바로 **평균 제곱 오차(MSE, Mean Squared Error)**를 최소화하는 수학적 최적점을 찾는 것입니다.
  $$ MSE = \frac{1}{m} \sum_{i=1}^{m} (\text{실제정답}_i - \text{예측값}_i)^2 $$
  선형 회귀는 정규 방정식(Normal Equation)이나 경사 하강법을 통해 이 MSE 손실 함수가 곡선의 가장 바닥(최솟값)에 도달하는 $\theta$ 조합을 수학적으로 계산합니다.

* **한계 (과소적합, Underfitting)**: 현실의 캘리포니아 집값은 '방이 1개 늘면 집값이 정확히 10만 달러 오른다'는 식의 직선 하나로 설명하기엔 패턴이 너무 복잡(비선형)합니다. 모델의 수식 자체가 너무 단순해서, 컴퓨터가 수학적 최적해를 완벽히 구했음에도 불구하고 훈련 데이터조차 제대로 맞추지 못합니다.
* **💡 코드 증명: 정말 선형 회귀는 한계가 명확할까?**
```python
from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_squared_error

# 선형 회귀 모델 훈련
lin_reg = LinearRegression()
lin_reg.fit(housing_prepared, housing_labels)

# 훈련 데이터에 대한 오차(RMSE) 계산
housing_predictions = lin_reg.predict(housing_prepared)
lin_rmse = mean_squared_error(housing_labels, housing_predictions, squared=False)
print(f"선형 회귀의 예측 오차: 약 ${lin_rmse:,.0f}")
# 결과 증명: 오차가 무려 $68,000 이상 나옵니다. 엄청난 '과소적합'입니다!
```

### 4.2 의사결정 나무 회귀 (Decision Tree Regressor)
* **이론적 배경 및 수식 (CART 알고리즘)**:
  직선으로 선을 긋는 것이 한계라면, 데이터를 '질문'을 통해 박스 형태로 계속 쪼개면 어떨까요? "소득이 3.0보다 큰가? (Yes/No)"
  의사결정 나무를 훈련시키는 대표적인 **CART (Classification and Regression Tree)** 알고리즘은 데이터를 두 그룹으로 쪼갤 때 다음의 비용 함수(Cost Function)가 **가장 작아지는** 특성($j$)과 분할 기준선($s$)을 찾아냅니다.
  
  $$ Cost(j, s) = \frac{N_{left}}{N} MSE_{left} + \frac{N_{right}}{N} MSE_{right} $$
  
  (전체 데이터 $N$개 중 왼쪽 그룹에 속한 데이터수 $N_{left}$ 비중) $\times$ (왼쪽 그룹 내의 오차 $MSE$) 에 오른쪽 그룹의 오차를 더한 값입니다. 즉, "칼질을 한 번 했을 때 쪼개진 두 그룹 내부의 오차들의 합이 가장 최소화되는 칼질 지점"을 끝없이 반복해서 탐욕적(Greedy)으로 찾아 내려가는 알고리즘입니다.

> [!NOTE] **🌳 분할의 핵심 기준: 수치를 맞출 땐 오차(MSE), 분류를 할 땐 불순도(지니/엔트로피)**
> 현재 2장은 '집값(숫자)'을 예측하므로 $MSE$를 척도로 가지를 쳤지만, 만약 추후에 개와 고양이를 나누는 **'분류(Classification)'** 나무를 기른다면 어떻게 쪼갤까요? 이때 칼질의 기준이 되는 것이 바로 **불순도(Impurity)**입니다. 하나의 무리(노드) 안에 고양이와 개가 마구 섞여있으면 불순도가 높고, 한 품종만 모이면 불순도가 0(순수)이 됩니다.
> 
> **1. 지니 불순도 (Gini Impurity, $G_i$)**
> $$ G_i = 1 - \sum_{k=1}^{n} p_{i,k}^2 $$
> * **개념**: "이 안에서 아무나 맹검으로 두 마리를 연속으로 뽑았을 때, 서로 다른 종이 뽑힐 확률"입니다.
> * **특징**: 이 계산법은 컴퓨터가 연산하기 매우 빠르기 때문에, 사이킷런 의사결정 나무(`DecisionTreeClassifier`)의 **기본(Default) 분할 잣대**로 쓰입니다.
> 
> **2. 엔트로피 (Entropy, $H_i$)**
> $$ H_i = -\sum_{k=1}^{n} p_{i,k} \log_2(p_{i,k}) $$
> * **개념**: 통계물리학과 정보이론에서 등장하는 '무질서도'를 측정합니다. 방 안이 온갖 쓰레기로 뒤죽박죽 섞여 무질서할수록 엔트로피 수치가 폭발합니다.
> * **특징**: 지니 불순도와 만들어내는 나무의 모양은 거의 비슷하지만, 나무를 아주 미세하게 더 양옆 밸런스 있게(균형 잡히게) 쪼개려는 성향이 있습니다. 하지만 $\log$ 계산 때문에 지니 불순도보다 연산이 조금 더 느립니다.
>
> 💡 **심화 요약**: 의사결정 뱡향타는, 연속형 정답(회귀)일 때는 $MSE$가, 범주형 정답(분류)일 때는 저 **지니 계수**가 가장 크게 줄어드는 쪽을 향해 미친듯이 달려갑니다!

> [!NOTE] **🍰 쉽고 재밌는 용어 사전: 탐욕적(Greedy) 알고리즘이란?**
> CART 알고리즘을 설명할 때 쓰인 '탐욕적'이란 무슨 뜻일까요?
> 
> **갈림길에 선 강아지**를 상상해 보세요. 
> "지금 이 갈림길에서 왼쪽 길에는 간식이 1개 있고, 오른쪽 길에는 간식이 2개 있네? 그럼 당장 눈앞에 간식이 더 많은 오른쪽으로 가자!"
> 우측 도로 끝에 간식 100개가 숨겨진 거대한 공장이 있을지라도, 강아지는 그것을 볼 수 없으니 **오직 '지금 당장 이 순간'에 가장 이득이 되는 선택만 덥석 무는 성향**이 있습니다. 이렇게 다음 스텝은 생각하지 않고 즉각적인 최적의 선택만 반복하는 방식을 컴퓨터 공학에서는 '탐욕적(Greedy) 알고리즘'이라고 부릅니다.
> 
> **머신러닝에서의 뜻**: 나무가 가지를 칠 때 "지금 특성 A로 쪼개는 것이 나중을 보면 별로일지 몰라도, 일단 **지금 당장 오차(Cost)를 제일 강력하게 줄여주니까** 단물만 빨고 보자!" 하고 직진해 버린다는 뜻입니다. 계산 속도가 엄청나게 빠르지만, 가끔은 전체 판쇄를 보지 못해 완벽한 '신의 한 수(최적해)'를 놓칠 수도 있는 한계를 갖습니다.

> [!NOTE] **🎲 [미니 실습] 동물 병원 데이터로 직접 해보는 '불순도 계산'과 '탐욕적 분할'**
> 이 복잡해보이는 수학과 알고리즘이 실제로 어떻게 동작하는지 강아지/고양이 데이터 5마리를 직접 만들어서 수학적으로 쪼개봅시다!
> 
> * **우리의 목표**: 몸무게(kg) 특성 하나만 보고 이 동물이 '개(Dog)'인지 '고양이(Cat)'인지 분류하는 의사결정 나무 만들기
> * **우리의 데이터**: A(2kg, Cat), B(4kg, Cat), C(5kg, Dog), D(7kg, Dog), E(8kg, Dog)
> 
> **Step 1. 칼질 전(루트 노드)의 전체 혼잡도 계산하기**
> 현재 첫 번째 방에 5마리가 다 모여있습니다. (고양이 2마리, 개 3마리)
> * 고양이 비율($p_{cat}$) = 2/5 = 0.4
> * 개 비율($p_{dog}$) = 3/5 = 0.6
> 
> 👉 **지니 불순도 계산**: $G_{root} = 1 - (0.4^2 + 0.6^2) = 1 - (0.16 + 0.36) = 1 - 0.52 = 0.48$
> 👉 **엔트로피 계산**: $H_{root} = -(0.4 \times \log_2(0.4) + 0.6 \times \log_2(0.6)) \approx 0.971$
> *(지니 지수는 0.5에, 엔트로피는 1.0에 가까울수록 서로 다른 데이터가 완전 반씩 섞인 최악의 혼잡 상태입니다. 지금 방 안은 꽤 개판이군요!)*
> 
> **Step 2. 탐욕적(Greedy) 알고리즘의 무차별 칼질 테스트**
> 컴퓨터는 몸무게를 2, 4, 5, 7, 8 순서대로 쭉 세워놓고, 사이사이에 가상의 칼집을 내어 불순도가 얼마나 줄어들지 모두 시뮬레이션 해봅니다.
> 가능한 칼질 포인트: ① 3kg 기준(A와 B사이) ② 4.5kg 기준(B와 C사이) ③ 6kg 기준(C와 D사이)...
> 
> **Step 3. 결과 확인과 탐욕적 선택 (4.5kg 기준 칼질 테스트의 경우)**
> 알고리즘이 "몸무게 4.5kg 이하인가?" 라는 질문으로 데이터를 촥! 쪼갰습니다.
> 
> * **왼쪽 방 (몸무게 $\le$ 4.5kg)**: A(2kg, Cat), B(4kg, Cat)
>   * 고양이 2마리, 개 0마리 ➡️ 고양이 비율 1.0!
>   * **왼쪽 방 지니 불순도**: $1 - (1.0^2 + 0^2) = 0$ (오염도 0, 완전 순수함!)
> * **오른쪽 방 (몸무게 > 4.5kg)**: C(5kg, Dog), D(7kg, Dog), E(8kg, Dog)
>   * 고양이 0마리, 개 3마리 ➡️ 개 비율 1.0!
>   * **오른쪽 방 지니 불순도**: $1 - (0^2 + 1.0^2) = 0$ (오염도 0, 완전 순수함!)
> 
> **탐욕적 알고리즘의 최종 결론**
> "오옷! 두 번째 칼질 기점인 '4.5kg'에서 쪼갰더니, 왼쪽 방의 불순도도 0이 되고 오른쪽 방의 불순도도 0이 돼서 전체 혼잡도(Cost)가 아예 소멸해 버렸어! 
> 이 뒤에 6kg, 7.5kg 칼질 기준을 써보면 어쩌면 더 좋은 결과가 있을지도 모르지만 더 이상 따져보거나 고민하지 않아! **지금 당장 눈앞의 불순도를 가장 강력하게 0으로 만들어주는 '4.5kg' 분할**을 이번 스텝의 최고의 칼질로 덥석 확정하고 다른 노드로 직진한다!"
> 
> 바로 이렇게 **현재 상태에서 가장 불순도를 크게 떨어뜨리는 선택지만 덥석 물고 앞만 보고 달리는 성향**이 의사결정 나무가 사용하는 탐욕적 알고리즘(Greedy Algorithm)의 가장 핵심적인 동작 원리입니다!

* **한계 (과대적합, Overfitting)**: 제한(깊이 제약 등) 없이 알고리즘이 계속 칼질을 하도록 놔두면 어떻게 될까요? "소득이 3.2초과고, 방이 2개고, 해안선이고..." 결국엔 **단 1채의 집만 들어가는 수만 개의 박스**로 데이터를 쪼개버립니다. 그럼 각 박스의 평균값과 실제 집값이 똑같으므로 훈련 그룹 안의 오차($MSE_{node}$)는 무조건 0이 됩니다. 하지만 새로운 집 데이터가 들어오면 만들어둔 극단적인 조건들 때문에 엉뚱한 박스에 들어가서 예측이 처참하게 망합니다! 이것이 **치명적인 과대적합**입니다.
* **💡 코드 증명: 정말 훈련 오차가 0이 나올까?**
```python
from sklearn.tree import DecisionTreeRegressor

tree_reg = DecisionTreeRegressor(random_state=42)
tree_reg.fit(housing_prepared, housing_labels)

housing_predictions = tree_reg.predict(housing_prepared)
tree_rmse = mean_squared_error(housing_labels, housing_predictions, squared=False)
print(f"의사결정 나무의 훈련 오차: ${tree_rmse:,.0f}") 
# 결과 증명: 오차가 $0 이 나옵니다!! 천재일까요? NO! 
# 훈련 데이터의 예외적인 노이즈까지 통째로 다외워버린 심각한 '과대적합' 상태입니다.
```

### 4.3 랜덤 포레스트 (Random Forest Regressor)
* **이론적 배경 및 수식 (배깅, Bagging)**:
  의사결정 나무가 "훈련 데이터에만 완벽히 적응해버리는 외골수(과대적합)"가 되는 문제를 어떻게 풀까요? 수학에서는 이를 **집단 지성(Ensemble)** 으로 해결합니다!
  
  **배깅(Bootstrap Aggregating)** 기법은 원본 훈련 데이터셋에서 무작위로 복원 추출하여 조금씩 다른 서브 데이터 셋을 100개 만듭니다. 그리고 이 100개의 데이터셋 위에서 의사결정 나무를 각기 다르게 100그루 키워 냅니다. (이것이 숲, Forest입니다!)
  
  $$ \hat{y}_{RF} = \frac{1}{B} \sum_{b=1}^{B} \hat{y}_{tree\_b} $$
  
  최종 예측 시에는 100그루의 나무($B=100$)가 내놓은 예측값($\hat{y}_{tree}$)들을 كلها 더해서 **평균(Average)** 을 냅니다. 
  통계학적으로, 100개의 각기 다른 편향(Bias)을 가진 개별 예측기들이 모여 평균을 내게 되면, 모델의 분산(Variance, 데이터가 달라질 때 결과가 널뛰는 정도)이 100배 가까이 줄어듭니다! 외골수 성향이 둥글게 깎여나가고 보편적으로 강력한 정답에 도달하는 것입니다.

* **장점**: 선형 회귀보다 강력하고, 단일 나무보다는 과대적합을 획기적으로 막아주는 현대 머신러닝의 가장 훌륭한 범용 모델 중 하나입니다.
* **💡 코드 증명: 숲을 이루면 훈련 오차가 어떻게 변할까?**
```python
from sklearn.ensemble import RandomForestRegressor

forest_reg = RandomForestRegressor(n_estimators=100, random_state=42)
forest_reg.fit(housing_prepared, housing_labels)

housing_predictions = forest_reg.predict(housing_prepared)
forest_rmse = mean_squared_error(housing_labels, housing_predictions, squared=False)
print(f"랜덤 포레스트의 훈련 오차: 약 ${forest_rmse:,.0f}")
# 결과 증명: 오차가 $18,000 수준으로 나옵니다. 선형회귀($68,000)보다 월등히 똑똑하며, 
# 단일 트리의 암기 오류($0)에 빠지진 않고 적절한 분산을 유지하는 것을 증명했습니다.
```

## 5. 모델 튜닝 및 평가

### 왜 튜닝을 할까요?
모델(RandomForest 등)마다 설정할 수 있는 조작키(하이퍼파라미터)가 수 십 개입니다. 이 조작키를 어떻게 맞추느냐에 따라 성능이 극과 극입니다.

* **그리드 탐색 (GridSearchCV)**: 사용자가 후보 조작키 값들을 몇 개 던져주면 (예: a는 1,2,3 / b는 10,20), 이 3x2=6가지 조합을 교차 검증을 통해 전부 노가다로 테스트해 보고 가장 좋은 세팅을 알려줍니다.
* **랜덤 탐색 (RandomizedSearchCV)**: 경우의 수가 너무 많아질 때는 그냥 무작위로 여러 번 뽑아서 최고를 찾습니다. 컴퓨터 자원을 크게 절약합니다.

### 최종 평가
가장 튜닝이 잘 된 모델(Best Estimator)을 뽑아, 그동안 한 번도 건드리지 않았던 **테스트 세트(Test Set)**에 넣고 단 한 번의 시험을 칩니다. 이 결과가 진짜 실전 실력입니다.
