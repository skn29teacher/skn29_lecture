Fetch API 심화: CRUD 및 이벤트 시스템

실제 SQLite3 데이터베이스와 연동되는 방명록을 조회, 생성, 수정, 삭제하는 과정을 2일차 이벤트 핵심 지식과 결합하여 구현합니다.

1. 방명록 작성 및 실시간 목록 관리 (인라인 CRUD)

새로운 글을 작성하면 페이지 새로고침 없이 리스트에 추가되며(e.preventDefault), 목록에 있는 아이템의 삭제와 인라인 수정을 처리할 때 부모 컨테이너에 단 하나의 리스너를 다는 이벤트 위임을 적용합니다. 또한, 수정/삭제 시 이벤트 전파 방지(e.stopPropagation)를 활용해 행 선택 버블링 이벤트를 차단합니다.

2. response.ok를 활용한 404 및 500 에러 포착

서버 내부의 가상 500 에러 경로를 호출하거나 존재하지 않는 404 주소를 불러들일 때, response.ok 분기 처리 구조로 안정적인 예외 흐름을 포착합니다.
