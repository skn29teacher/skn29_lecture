5차시: Fetch CRUD & 에러 제어 점진적 라이브 코딩

※ 실습 시작 전 백엔드 서버(python backend/main.py)가 구동 중이어야 합니다.

5차시 단계별 실습 흐름

  1. Step 1 [READ - 방명록 목록]: 로컬 백엔드 서버의 방명록 데이터 API를 fetch해 와서 리스트를 채웁니다.
  2. Step 2 [CREATE - 방명록 작성]: POST 메서드 옵션을 실습합니다. 작성자 글자가 2자 미만일 시 FastAPI 백엔드가 던지는 400 Bad Request 에러 정보를 화면에 파싱합니다.
  3. Step 3 [DELETE - 방명록 삭제]: DELETE 요청을 수행하고, 자식 삭제 버튼을 클릭할 때 카드 행 전체 클릭이 오작동하는 현상을 막기 위한 e.stopPropagation()의 필연성을 공부합니다.
  4. Step 4 [HTTP 에러 처리]: 일반 fetch는 HTTP 오류(404, 500)가 발생해도 네트워크 전송 자체는 성공했으므로 catch 블록으로 안 갑니다. response.ok를 이용해 수동 throw하는 예외 설계를 실습합니다.
