# 웹 서버-클라이언트 아키텍처와 연결 테스트

## 1. 클라이언트-서버 모델 (Client-Server Model)
웹 서비스의 가장 기본적인 아키텍처는 **클라이언트(Client)** 와 **서버(Server)** 가 요청(Request)과 응답(Response)을 주고받는 구조입니다.

- **클라이언트**: 사용자의 입력을 받아 필요로 하는 데이터나 자원을 서버에 요청하는 브라우저나 클라이언트 프로그램입니다.
- **서버**: 클라이언트의 요청을 받아 비즈니스 로직을 처리하거나 데이터베이스를 조회하여 그 결과를 다시 클라이언트에 돌려주는 컴퓨터 또는 프로그램입니다.
- **HTTP 프로토콜**: 웹 상에서 클라이언트와 서버가 서로 규칙을 지켜 데이터를 주고받기 위한 통신 규약입니다.

## 2. 자바스크립트 비동기 프로그래밍과 Promise
과거 웹은 페이지 전체가 리로드되는 방식을 사용했으나, 현대 웹 애플리케이션은 화면 전환 없이 데이터를 갱신하기 위해 **비동기 통신** 을 활용합니다.

- **비동기 통신**: 요청을 보낸 뒤 응답을 받을 때까지 멈춰 있지 않고 다음 코드를 계속 실행하는 방식입니다.
- **Promise**: 자바스크립트에서 비동기 연산의 완료 또는 실패와 그 결과값을 나타내는 객체입니다. 성공(resolve)과 실패(reject) 상태를 가지며, `.then()`과 `.catch()`를 통해 비동기 처리 결과를 연결해 나갈 수 있습니다.
- **Fetch API**: 웹 브라우저에 내장된 비동기 HTTP 요청 전송 기능으로, Promise를 반환하여 사용하기 쉽고 직관적입니다.

## 3. FastAPI 백엔드 맛보기
파이썬 생태계의 대표적인 고성능 웹 프레임워크인 **FastAPI** 를 사용하여 서버를 구동합니다. FastAPI는 들어오는 HTTP 요청을 받아 라우터에 매핑된 파이썬 함수를 실행하고 그 결과를 JSON 형태로 변환하여 반환해 줍니다.

## Django로의 연계 지점
향후 Django 프레임워크를 배울 때도 브라우저(클라이언트)가 URL을 통해 요청을 전달하면 Django 서버가 이를 받아 적절한 응답을 주는 동일한 클라이언트-서버 메커니즘을 사용합니다. Django 역시 WSGI 게이트웨이를 거쳐 프레임워크 내부로 HTTP 요청이 도달한다는 기본 흐름을 인지하는 것이 핵심입니다.
