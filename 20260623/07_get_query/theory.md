# HTTP 요청 구조 분석 - GET 메서드와 쿼리 스트링

## 1. HTTP 요청 메시지 구조 개요
클라이언트가 서버로 전송하는 HTTP 요청(Request) 패킷은 텍스트 형식으로 이루어져 있으며 다음과 같이 구성됩니다.

```text
[요청 라인] -> HTTP 메서드 (GET) / 요청 URI (경로 + 쿼리 스트링) / HTTP 버전
[요청 헤더] -> Host, User-Agent, Accept 등 클라이언트 및 목적지 정보 (Key: Value)
[빈 라인]   -> 헤더와 바디 영역을 나누기 위한 빈 줄 (\r\n)
[요청 바디] -> 전송할 데이터 본문 (GET 요청의 경우 보통 비어 있음)
```

## 2. HTTP GET 메서드와 쿼리 스트링 (Query String)
- **GET 메서드**: 서버로부터 지정된 리소스의 정보를 얻어오기(Retrieve) 위해 사용됩니다. 데이터를 단순 조회하는 성격을 띠기 때문에 요청 바디를 사용하지 않고 URL 뒤에 데이터를 이어 전송합니다.
- **쿼리 스트링**: URL 끝에 물음표(`?`)를 시작으로 붙는 매개변수 리스트입니다. `키=값` 형식으로 작성되며 여러 개일 때는 앤드(`&`)로 연결합니다.
  - 예: `/api/users?name=홍길동&limit=5`

## 3. 자바스크립트 Fetch API와 GET 요청
Fetch API에서 GET 요청을 날릴 때, 브라우저는 URL에 붙은 쿼리 스트링을 파싱하여 HTTP 요청 패킷의 시작 라인(요청 URI)에 삽입합니다.

---

## Django로의 연계 지점
향후 Django 프레임워크 학습 시, 뷰(View) 클래스 또는 함수 내부에서 GET 요청의 쿼리 스트링 데이터를 다룰 때 `request.GET` 딕셔너리를 활용하게 됩니다.
- 예: `search_name = request.GET.get('name')`
FastAPI에서 `Query` 매개변수를 선언해 인자를 취하는 방식이 Django의 `request.GET.get()` 방식과 개념적으로 일대일 대응된다는 점을 이해하면 전환이 무척 쉬워집니다.
