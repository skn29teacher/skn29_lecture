# 📄 산출물 1: 인공지능 데이터 전처리 결과서

## 1. 프로젝트 개요

### 1-1. 비즈니스 배경
- 현재 고객 이탈로 인해 발생하는 비즈니스 문제를 서술
- 예: "통신사 고객의 월평균 이탈률 X%, 이탈 고객 유치 비용은 신규 유치의 5배"

### 1-2. 비즈니스 목표
- 해결하고자 하는 핵심 문제를 명확히 기술
- 예: "이탈 가능성이 높은 고객을 사전에 식별하여 선제적 리텐션 전략 수립"

### 1-3. 머신러닝 활용 계획
| 항목 | 내용 |
|------|------|
| 문제 유형 | 이진 분류 (이탈=1 / 유지=0) |
| 예측 대상 | 향후 N개월 내 이탈 여부 |
| 활용 방안 | 이탈 위험군 고객 식별 → 마케팅 팀 전달 → 맞춤 혜택 제공 |
| 기대 효과 | 이탈률 X% 감소, 연간 매출 유지 |

### 1-4. 성공 기준 (Success Criteria)
- 모델 성능 기준: Recall ≥ 0.80 (이탈 고객 탐지율 우선)
- 비즈니스 기준: 이탈 예측 정확도가 현재 경험적 판단 대비 향상

## 3. 탐색적 데이터 분석 (EDA)

### 3-1. 기초 통계량
- df.info(), df.describe() 결과 요약 표 삽입
- 수치형 변수: 평균, 중앙값, 표준편차, 최소/최대값
- 범주형 변수: 고유값 수, 최빈값

### 3-2. 결측값 분석
| 특성명 | 결측 수 | 결측률(%) | 처리 방안 |
|--------|---------|-----------|----------|
| TotalCharges | 11 | 0.15% | 중앙값 대체 |
| ... | | | |

### 3-3. 이상값 분석
- 박스플롯 또는 IQR 기법으로 탐지
- 이상값으로 판단된 특성 및 건수 기술
- 처리 방법 (제거 / 클리핑 / 유지) 및 근거 명시

### 3-4. 특성별 분포 분석
- 수치형 특성: 히스토그램, 이탈 여부에 따른 분포 차이
- 범주형 특성: 막대 그래프, 이탈률 차이

### 3-5. 상관관계 분석
- 수치형 특성 간 상관계수 히트맵
- 타겟(Churn)과 각 특성 간의 상관관계
- 다중공선성 의심 특성 식별 (|r| > 0.8)

### 3-6. EDA 주요 발견사항 (Insight)
- 발견 1: tenure(가입기간)가 짧을수록 이탈률 높음
- 발견 2: MonthlyCharges가 높을수록 이탈 경향
- 발견 3: Contract 유형(월간 계약)에서 이탈률 집중
- → 이후 특성 공학(Feature Engineering)에 반영할 인사이트 정리

| 특성명 | 처리 방법 | 근거 |
|--------|----------|------|

| ... | | |

### 4-2. 이상값 처리
| 특성명 | 탐지 방법 | 처리 방법 | 처리 전 범위 | 처리 후 범위 |
|--------|----------|----------|------------|------------|

### 4-3. 인코딩 (범주형 → 수치형)
| 특성명 | 인코딩 방법 | 변환 결과 | 선택 근거 |
|--------|------------|----------|----------|
| gender | Label Encoding | Male→0, Female→1 | 이진 변수 |
| Contract | One-Hot Encoding | 3개 더미 변수 생성 | 명목형, 3개 범주 |
| ... | | | |

### 4-4. 특성 공학 (Feature Engineering)
- 생성한 새 특성 목록과 생성 근거
  - 예: `charge_per_month = TotalCharges / tenure` (기간 대비 청구액 밀도)
  - 예: `high_risk_flag` = (tenure < 6) & (MonthlyCharges > 70)

### 4-5. 불균형 데이터 처리
| 방법 | 적용 여부 | 처리 전 비율 | 처리 후 비율 |
|------|----------|------------|------------|
| SMOTE (오버샘플링) | O / X | 이탈:유지=X:Y | X:Y |
| class_weight 조정 | O / X | | |
| 언더샘플링 | O / X | | |

### 4-6. 스케일링
| 방법 | 적용 특성 | 선택 근거 |
|------|----------|----------|
| StandardScaler | tenure, MonthlyCharges, TotalCharges | 정규분포 근사 특성 |
