```markdown
- 예: "통신사 고객의 월평균 이탈률 X%, 이탈 고객 유치 비용은 신규 유치의 5배"

### 1-2. 비즈니스 목표
- 해결하고자 하는 핵심 문제를 명확히 기술
- 예: "이탈 가능성이 높은 고객을 사전에 식별하여 선제적 리텐션 전략 수립"

| 기대 효과 | 이탈률 X% 감소, 연간 매출 유지 |

### 1-4. 성공 기준 (Success Criteria)
- 모델 성능 기준: Recall ≥ 0.80 (이탈 고객 탐지율 우선)
- 비즈니스 기준: 이탈 예측 정확도가 현재 경험적 판단 대비 향상
```

> 이탈 예측은 **이탈 고객을 놓치는 것(False Negative)** 이 더 큰 손실이므로 Recall을 우선 지표로 설정합니다.

```markdown
|------|----------|

### 1-3. 실험 계획
- 각 모델에 기본 하이퍼파라미터로 베이스라인 성능 측정
- 최고 성능 모델에 하이퍼파라미터 튜닝 적용
- 최종 모델 선정 기준: Test Recall ≥ 0.80 & F1-Score 최고
```

import pickle, json
import pandas as pd

# 모델 및 스케일러 로드
with open('best_model.pkl', 'rb') as f:
    model = pickle.load(f)
with open('scaler.pkl', 'rb') as f:
    scaler = pickle.load(f)
with open('feature_list.json', 'r') as f:
    features = json.load(f)

# 신규 고객 데이터 예측
new_customer = pd.DataFrame([{...}])  # 특성값 입력
new_customer_scaled = scaler.transform(new_customer[features])

prediction = model.predict(new_customer_scaled)
probability = model.predict_proba(new_customer_scaled)[:, 1]

print(f"이탈 예측: {'이탈' if prediction[0]==1 else '유지'}")
print(f"이탈 확률: {probability[0]:.2%}")
